# 中国股票标准化Workflow Prompts

## 覆盖范围
- **沪深主板** (60xxxx / 00xxxx)
- **创业板** (30xxxx)
- **科创板** (68xxxx)
- **北交所** (8xxxxx / 4xxxxx)
- **港股通** (xxxx.HK)
- 新三板：数据受限，有具体标的时单独搜索

## 卖方信息使用规则
- **可用**：事实和数据（产品出货量/客户名单/产能数字/行业格局/管理层guidance原话）
- **不用**：卖方EPS预测/目标价/评级/观点/"看好看空"/财务预测模型
- 15/15卖方看多 → 仅用于判断priced in，不支持买入

## 1. 全盘扫描报告（发现）

```
全市场扫描（A股沪深主板+创业板+科创板+北交所+港股通）。部署40个agent覆盖所有主要板块（半导体/新能源/医药/军工/消费/化工/金融/制造/TMT/有色/汽车/食饮等），每个板块2-4个agent。每个agent的任务：找该板块中PEG最优+供给侧有结构约束的标的，给出一句话thesis+PEG+催化剂日期+bear case。港股通标的与A股同行业标的直接对比（同一公司A/H价差纳入考量）。卖方研报仅取事实数据，不取观点和财务预测。

全部agent返回后（不许分批出报告），做跨板块横向对比：
- 按PEG排名
- 按催化剂时间线排名
- 按供给约束强度排名

最终输出：Top 10 conviction排序表（标的/PEG/一句话thesis/催化剂/SABCT评级/建议仓位%），附建仓计划。
```

## 2. 买方PM深度研究（验证）

```
对以下标的进行买方PM级深度研究：[标的列表]

每只标的先定义核心决策问题（买/不买取决于什么），然后开5个agent：
- Agent 1 供给侧Edge验证：别人给不了的东西是什么？能持续多久？
- Agent 2 Kill Shot搜索：什么能一票否决？专门搜负面信息
- Agent 3 定价检验：市场已知道什么？我看到了什么市场没看到的？
- Agent 4 催化剂锁定：什么时候证明我对？不兑现怎么办？
- Agent 5 建仓裁决：PEG+仓位+止损+明确的买/不买决定

全部返回后，输出每只标的的verdict（买/不买/等待）和conviction评级，附执行卡片（建仓价区间/止损线/出场条件/仓位/催化剂日期）。
```

## 3. 持仓复盘+调仓执行（执行）

```
读portfolio_state.json，对现有A股持仓逐一审查：
1. 每只持仓回答：thesis还成立吗？催化剂兑现了吗？conviction还够A-吗？
2. 不达A-门槛的标的→列出清仓清单
3. Thesis弱化的标的→列出减仓/清仓建议+具体理由
4. 表现好的标的→是否加仓？仓位是否匹配conviction？

审查完毕后：
- 输出调仓计划表（标的/动作/股数/理由）
- 如果有释放资金→结合最近研究结果推荐建仓标的
- 目标：保持80%+仓位，所有持仓conviction≥A-

确认后直接执行。
```
