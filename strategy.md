## 2. 账户规则

### 美股
| 规则 | 细节 |
|------|------|
| 交易制度 | T+0（日内可多次买卖同一标的） |

- **催化剂日**: 日内可交易；**非催化日**: 仅开盘/收盘窗口操作

## 3. 仓位管理

### 信心等级与仓位

| 等级 | 仓位范围 | 建仓条件 |
|------|---------|---------|
| A级（核心持仓） | 10–15% | 自主深度研究完成 + 催化剂日期明确 + bear case <15% |
| B级（重点持仓） | 5–10% | thesis清晰 + 有明确催化剂 |
| C级（观察仓） | 2–5% | 初步thesis + 验证中 |
| 交易性仓位 | 2–5% | 短期事件驱动，有明确退出条件 |

### 组合级硬约束

| 约束 | 上限/下限 | 违反时行动 |
|------|---------|---------|
| 单只仓位 | ≤15% | 超额部分减至12%，当日执行 |
| 单板块仓位 | ≤30% | 新增该板块需先平仓 |
| 现金比例 | **≥20%（硬规则）** | 低于20%前停止一切新建仓 |
| 单日新建仓 | ≤3只 | 第4只等次日 |
| C级标的总仓位 | ≤15% | 新C级标的先减旧的 |
| 最大持仓数量 | ≤12只 | 超过12只必须淘汰一只 |

### 止损规则
- **每笔建仓时即设硬止损**（无止损不建仓）
- **组合级止损**: 单日亏损>3% → 当日停止新建仓，次日恢复
- **止损触发后**: 同一交易日不回买同一标的
- **计划外操作**: 执行前先写thesis到`daily-reviews/YYYY-MM-DD.md`，不"先买后研究"

## 4. 选股框架

### 第一步: 供给侧优先（F2框架）
看物理约束（交期/产能/技术矛盾），不看行业集中度。每只标的先问：**谁控制供给、供给瓶颈在哪**。

### 第二步: 催化剂筛选
- 催化剂**必须在30天内**（本周期到06/18截止）
- 说不出催化剂日期 = 不建仓
- 事件落地当天不动手，等1周资金验证

### 第三步: Bear case压力测试
**Bear case downside >20% = 无条件不建仓**（无例外，无讨论）

### A股选股信号（可靠性排序）

| 优先级 | 信号类型 | 操作 |
|-------|---------|------|
| 1 | 美股龙头映射（NVDA/AAPL隔夜大涨后次日A股） | 次日开盘前半小时评估 |
| 2 | 政策信号滞后5–10个交易日布局 | 政策公布后等情绪高峰过去 |
| 3 | 产业大会前1–2周布局 | 大会前不追，会后第一波回调买 |
| 4 | 板块轮动确认（放量、资金净流入） | 量价配合才进 |
| 5 | 业绩催化剂（预告/业绩超预期） | 预告发布后等一天确认 |
| 6 | 行业数据（月度销售/PMI细项） | 数据方向明确才动 |

### A股8条错误模式（负面筛选）

| 编号 | 错误模式 | 具体表现 | 对策 |
|------|---------|---------|------|
| E1 | 节后追高 | 节前停牌，复牌直接高开8%+ | 等首日高开回落再看 |
| E2 | 政策当天追 | 政策发布当日直接冲涨停 | 政策后第5个交易日之后布局 |
| E3 | 板块热点追尾 | 板块连续上涨5天以上才买 | 第3天开始观察，第1-2天才是入场窗口 |
| E4 | 概念无业绩 | 纯概念股无实际收入 | 验收：业绩贡献>10%总营收才可建仓 |
| E5 | 替代品陷阱 | 买不了A就买B（低相关） | 买不了A就等，不买低相关的B |
| E6 | 先买后研究 | 看到涨先建仓再想thesis | 无thesis不建仓 |
| E7 | 止损后回买 | 当日止损后当日重买 | 止损后至少隔1个交易日才可重评 |
| E8 | 追涨停 | 涨停板追入 | 涨停后等次日回调，不追板 |

### 美股选股规则

| 规则 | 细节 |
|------|------|
| 共识反向信号 | 15/15卖方看多 = priced in = 从做多universe排除（F15框架） |
| 排除条件 | Bear case >20% / 无30天内催化剂 / 卖方全覆盖看多 |
| 信息来源优先级 | watchlist已有研究 > market-pulse时事 > 卖方报告 |
| 做空Universe | 只从`watchlist_config.json`的`us_short_candidates`选标的 |

## 5. 进场/出场检查表

### 进场检查表（每笔必填，不填不执行）

```
标的:
方向: 多/空
thesis（一句话，说不清楚就不建仓）:
供给侧约束是什么:
催化剂+明确日期:
入场价:
止损价 / 止损幅度%:
目标价1 / 目标价2:
R/R比（目标涨幅÷止损幅度，<1.5不建仓）:
仓位大小 / 信心等级（A/B/C）:
Bear case（具体，不写"竞争加剧"这种废话）:
Bear case downside%（必须≤20%）:
为什么现在买（timing理由）:
计划偏差记录（与strategy不一致的地方）:
```

### ABCD下跌分类——下跌先分类再行动

| 类型 | 定义 | 对应行动 |
|------|------|---------|
| A类 | 大盘系统性下跌，个股跟跌 | 持仓不动，等市场企稳 |
| B类 | 板块轮动卖出，非基本面 | 持仓不动，观察1–2天 |
| C类 | 个股技术回调，无新闻 | 跌至止损线才触发，不提前止损 |
| D类 | **基本面恶化，thesis被证伪** | **立即止损，不等待** |

### 出场5条规则

1. **D类下跌**（thesis被证伪）: 立即止损，无商量
2. **止损触发**: 无条件执行，同日不回买
3. **目标达成**: 分批出（50%在目标价1，50%用trailing stop锁剩余）
4. **2周无催化**: 重新评估是否继续持有，原则上考虑退出
5. **再平衡**: 单只超15% → 减仓至12%，当日执行

## 6. 每日4-window流程（固定执行顺序）

> 5-window旧方案已废弃。现行4-window方案。

| Window | 时间 | 账户 | 必做事项 |
|--------|------|------|---------|
| W1 | **10:00 BJT** | A股 | 早盘分析（见下方5维度） |
| W2 | **16:00 BJT** | A股 | 收盘结算 + 5维度独立研究 |
| W3 | **22:00 BJT** (10:00 ET) | 美股 | 早盘分析（见下方5维度） |
| W4 | **04:00+1 BJT** (16:00 ET) | 美股 | 收盘结算 + 5维度独立研究 + **全面复盘** |

### W1: A股早盘（10:00 BJT）
1. 用`yf`命令查A股持仓实时价格（不用昨日收盘价代替）
2. 检查隔夜美股对A股的映射信号（美股大涨/大跌→对应A股板块）
3. 核对止损线——有触发风险的标的列出
4. 执行预案中已触发的If-Then操作（价格在范围内才动）
5. 扫描当日A股重要公告/新闻（20分钟内）

### W2: A股收盘结算（16:00 BJT）
1. 记录收盘价，更新`portfolio_state.json`中A股持仓current_price
2. 计算当日A股账户P&L，与沪深300对比
3. 执行5维度独立研究（详见第7节）
4. 写`daily-reviews/YYYY-MM-DD.md`的A股部分

### W3: 美股早盘（22:00 BJT / 10:00 ET）
1. 用`yf`命令查美股持仓实时价格
2. 检查盘前变动（BMO财报、重要新闻）
3. 核对止损线，执行预案中触发的If-Then操作
4. 催化剂日可进行日内交易；非催化日仅开盘窗口操作

### W4: 美股收盘 + 全面复盘（04:00+1 BJT / 16:00 ET）
1. 记录收盘价，更新`portfolio_state.json`中美股持仓current_price
2. 计算当日美股账户P&L，与SPY对比
3. 执行5维度独立研究（详见第7节）
4. **完成全面复盘**（A股+美股汇总，写daily-reviews）
5. 为次日A股开盘准备映射信号（美股涨跌→A股哪些板块受影响）
6. 更新catalyst_calendar_30d中已过期事件的状态

## 7. 5维度研究框架（每次收盘session必做）

> W2（A股收盘）+ W4（美股收盘）各做一轮5维度。每轮耗时约30–45分钟。

| 维度 | 内容 | 产出 |
|------|------|------|
| **D1: 新标的深度研究** | 从watchlist_config.json选1个未持仓标的，做完整10步研究。A股收盘侧重部署候选；美股收盘侧重新覆盖 | 进场检查表 or 排除理由 |
| **D2: 做空候选研究** | 建立/更新1个做空thesis（仅美股）。用watchlist的us_short_candidates | 做空检查表 or 维持watchlist |
| **D3: 持仓催化剂深挖** | 选1只持仓，主动找反面证据（非验证正面thesis）。用"如果我是空头，我会怎么攻击这个标的" | 更新portfolio_state.json中的bear_case字段 |
| **D4: 宏观环境研判** | 当日宏观数据/央行信号对持仓的影响。3步走：数据→判断→对watchlist的含义 | 写入daily-reviews宏观部分 |
| **D5: 新主题/新机会发现** | 扫描当日新闻/数据，找未覆盖的主题或个股机会。宁可发现后不行动，也不错过发现 | 加入watchlist_config.json or 记录到daily-reviews |

## 8. 行为铁律（防止重蹈覆辙）

### 6大铁律

| 编号 | 铁律 | 违反后果 |
|------|------|---------|
| L1 | **不全仓** | 单只最大15%，现金最低20%，违反立即强制减仓 |
| L2 | **不先买后研究** | thesis未写完不执行交易，无论价格怎么动 |
| L3 | **赢了不加码** | 同标的第二轮size ≤ 第一轮建仓量 |
| L4 | **不系统性偏多** | 每日核对总部署vs计划，不因乐观情绪超配 |
| L5 | **下跌先分类** | 按ABCD分类后再行动，不情绪止损 |
| L6 | **不替代品陷阱** | 买不了A就等，不买低相关的B |

### 新增行为模式（#7–9，从历史教训提取）

| 编号 | 模式 | 规则 |
|------|------|------|
| L7 | **价格必须实时查询** | **所有价格用`yf price TICKER`实时查**。禁止用记忆或旧文件中的价格（思源¥42→¥190教训：用记忆价格会建错仓位）|
| L8 | **模拟盘与真实交易独立** | 模拟盘决策不参考用户真实仓位，有独立的P&L和风控系统 |
| L9 | **计划偏差强制记录** | 任何偏离strategy的操作，必须先记录"为什么偏离"到daily-reviews，再执行 |

### 自我约束Checklist（操作前过一遍）

- [ ] 价格是刚查的实时价格，不是记忆值
- [ ] thesis一句话能说清楚
- [ ] 止损线已设好，在portfolio_state.json里
- [ ] bear case downside ≤ 20%
- [ ] 操作后现金仍 ≥ 20%
- [ ] 操作后单只仍 ≤ 15%
- [ ] 如果是计划外操作，thesis已写到daily-reviews

## 9. If-Then预承诺格式

**核心规则: 没有预承诺 = 没有操作。禁止临场发挥。**

### 标准格式

```
IF [条件] AND [日期/时间]
THEN [动作] [数量] [价格约束]
ELSE [替代动作 or 不动]
```

### 示例（当前已有预承诺）

```
IF HSAI Q1财报 Beat（收入>$90M + 机器人份额>20%）AND 2026-05-19盘前
THEN 加仓335股HSAI @ 市价开盘30分钟内（若跳空>10%等1小时）
ELSE 如果In-line则守仓1周不动 | Miss则止损上移至$19.50

IF NVDA Q1 FY2027财报 Beat + 强指引 AND 2026-05-28盘后
THEN 维持80股不追仓（15/15共识满，无alpha）
THEN A股05-29：晶方放量>3% → 加1000股@市价

IF AAPL WWDC超预期 AND 2026-06-08
THEN 加25股AAPL @ 市价 → 仓位升至15%
ELSE 功能延期/不及预期 → 维持10%不动

IF ADBE Q2财报 FCF>38% + 收入加速 AND 2026-06-11盘后
THEN 加16股ADBE @ 市价 → 仓位升至12%
ELSE AI威胁加剧 + 增速下滑 → 评估减至5%
```

## 10. 文件系统

| 文件 | 用途 | 读写频率 |
|------|------|---------|
| `portfolio_state.json` | **唯一真相源**。所有持仓、价格、P&L、催化剂日历 | 每个window必读，有交易必写 |
| `strategy.md` | 你正在读的这个。操作手册，不随意改动 | Session开始读一遍 |
| `watchlist_config.json` | 选股池。所有可买标的、仓位上限、stop参数 | D1/D2研究时读 |
| `daily-reviews/YYYY-MM-DD.md` | 每日复盘。A股+美股+宏观+5维度产出 | 每日W4写完整版 |
| `research-notes/` | 中长期研究笔记（battle plan、行业研究） | 需要时读 |
| `scripts/` | 自动化脚本（价格更新、P&L计算） | 需要时调用 |

### 文件更新规则
- `portfolio_state.json`: 每次交易后立即更新，不滞后
- `daily-reviews/`: 文件名格式`YYYY-MM-DD.md`，每日一个，W4完成
- 所有当日价格从`yf`实时查询后写入，不手动估算
