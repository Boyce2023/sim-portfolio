## §1 Core Rules R1-R8

**R1 唯一真相源**: 价格/仓位/P&L只从 `portfolio_state.json` 读取。禁止记忆推算。

**R2 仓位硬顶与建仓门槛**:

| 等级 | 仓位上限 | 最大止损 | 建仓资格 |
|------|---------|---------|---------|
| S | 50% | -20% | 是 |
| A+ | 35% | -18% | 是 |
| A | 25% | -15% | 是 |
| A- | 20% | -12% | 是（最低门槛） |
| B+ | 15% | -12% | **否，进观察池** |
| B | 12% | -12% | **否，进观察池** |
| B- | 10% | -12% | **否，进观察池** |
| C/D | — | — | 排除 |

**建仓最低门槛：SABCT ≥ A-。** B+以下只进观察池，不花钱。

持仓数不约束（v9.2 06-23用户令：集中度靠SABCT单仓上限保证，不靠持仓数——数量少≠集中度高）。S≤1只。

**R3 无thesis不建仓**: 说不出"一句话thesis（含结构性优势+催化剂+止损价）"= 不买。

**R4 止损硬止损**: 震荡市-12%，牛市-15%，熊市-10%（详见§6 Regime表）。跌至止损线当日执行，不分析不犹豫不等反弹。同日不回买。

**R5 If-Then盘中不可修改**: 预承诺在非交易时间写入。盘中只执行。想改→收盘后改→次日生效。

**R6a PEG优先**: 估值用PEG(PE/增长率)。PE 66x但增长70%→PEG<1→不算贵。禁止单独用PE判断贵贱。

**R6b 供需唯一锚**: 不看卖方目标价。

**R6c 龙头崩=板块终**: 板块龙头单日跌≥5%→当天=主线行情结束。同板块所有持仓次日开盘评估出场，不等自身止损线。

**R7 产品级分析**: 研究时看公司实际做什么（产品/产能/客户），不看sector label。"AI"不是thesis，具体产品供给约束才是。

**R8 仓位-Conviction对齐**: 催化剂≤2周的仓位>催化剂>2月的。持仓>15%必须能说清楚catalyst timeline。

## §2 Research & Entry

### 研究流程（先研究，后建仓）

```
Step 1: 发现标的
  来源: UASS扫描（发现工具）/ 产业链追踪 / 新闻
  → 进入研究pipeline，不直接建仓

Step 2: 研究（产品级）
  ① 公司实际做什么？产品/产能/市占率
  ② 结构性优势？同级供应商几家？壁垒持续几年？
  ③ 催化剂是什么？日期/机制/可验证性

Step 3: 估值检查
  PEG + 供需紧缺度。不用卖方目标价。

Step 4: 催化剂质量分级（决定conviction上限）
  α级: 特定日期+明确机制（财报日/政策窗口/旺季开始）→ 无上限
  β级: 日期范围+方向明确（月度数据窗口/季节性）→ conviction上限A级
  γ级: 主题驱动/无日期（"AI"/"国产替代"）→ 不满足A-建仓门槛，归观察池

Step 5: SABCT评级
  ≥ A- → 可建仓（R2表sizing）
  < A- → 观察池（不花钱，等conviction升级）

Step 6: 建仓
  设止损(-12%硬止损或等级上限取较小值)
  写一句话thesis → execute_trade.py
```

### UASS的角色（发现工具，非建仓入口）

- UASS扫描 = 每天/每周看"昨天什么在动"，发现有意思的板块和公司
- 扫描结果触发研究，不触发建仓
- 涨停板不是买入信号，是"某个thesis在被市场定价"的信号
- 发现有意思的票 → 进Step 2研究pipeline → 研究完+SABCT≥A- → 才建仓

**板块发散（UASS的最大价值）**: 涨停板热门板块 → 映射上下游 → 找同链条滞涨票 → 进研究pipeline

### 分批建仓

| 阶段 | 比例 | 条件 |
|------|------|------|
| 探针 | 40% | 研究完+SABCT≥A-即进 |
| 确认 | 60% | 探针日收盘未跌>5%+thesis未变 |
| 放弃 | 卖探针 | 跌>8%或新利空证伪thesis |

S级+催化剂≤3天+放量可一次性全仓。

## §3 Exit System v2.0

**核心原则: No Conviction Exemption** — Conviction决定仓位SIZE，不决定是否执行止损/止盈。A+也要止损，A-也要止盈。

### 6条退出规则

**X1 Trailing Stop（动态追踪止盈/损）**:

```
浮盈阶段     止损线                  说明
< 5%         入场价 × 0.88          还没赚→常规-12%硬止损
5% ~ 15%     峰值价 × 0.95          小赚→回撤5%止盈
15% ~ 30%    峰值价 × 0.92          中赚→回撤8%
> 30%        峰值价 × 0.90          大赚→回撤10%
```
峰值=持仓期间最高收盘价。触及当日执行，不分析不犹豫。

**X2 Catalyst Expiry T+5**: 催化剂日期过后5个交易日，thesis未兑现→清仓。不给"下一个催化剂"的借口。

**X3 T11 Mechanical（暴力拉升=减半）**: 单日>8%或2-3日累>15% → **自动减半**，无conviction豁免。
- 催化剂还没到 → 减半留底仓
- 催化剂正在发生 → 卖进强势
- 催化剂已过 → 清大部分
- **Claude自主义务**: 不等用户说，主动提"止盈窗口"
- **沪电教训**: +11%应减半锁利¥39K，反而加仓最终亏¥10K
- **历史执行率**: 仅14%，v2.0设为mechanical无例外

**X4 Track B Degradation（2日规则）**: Track B评级连续2天下降（如B→C或A-→B-）→ 减半或清仓。市场信号退潮时研究再好也扛不住。

**X5 Time Stop 20天**: 建仓后20个交易日thesis无进展（无催化剂兑现、无供给验证、价格未动）→ 清仓释放资金。资金有机会成本。

**X6 Thesis证伪（E1升级）**: 供需反转/催化剂不成立/竞争格局恶化 → 48h清仓。硬数据证伪不等反弹。

### 其他退出规则（保留）

**龙头崩（R6c）**: 板块龙头单日跌≥5% → 次日开盘全板块评估出场。

### 下跌分类（60秒内判断）

| 类型 | 判别 | 行动 |
|------|------|------|
| A | 沪深300跌≥1.5%，无个股新闻 | Hold |
| B | 指数稳，无新闻，跌<5% | Hold 1-2天观察 |
| C | 有新闻但不改thesis | 减0-30%观察 |
| D | 硬数据证伪thesis（X6触发） | 48h清仓 |

**催化剂前T-5天**: 已盈利持仓减至≤15%。

**T+1约束**: 当日买入不可卖出。

## §5 Behavioral Guards

**L10 关注点漂移**: "别的涨了"想换仓 → 强制两问：①thesis变了吗？②是因为看了别的想卖？→ ②是 → 关闭行情等1小时。

**L12 浮亏检查**: "现在价格我会新买吗？" → 是=持有，否=减仓。

**L13 追涨三行检查**: ①理由不是"涨了" ②跌20%割不割 ③今天才知道→不买。

**研究完才买，不是买了再研究。**

| # | 信号 | 牛市(+1) | 中性(0) | 熊市(-1) |

| 1 | 成交量vs20日均 | >+30% | ±30% | <-30% |

### Regime→参数调整

| 参数 | 牛市 | 震荡 | 熊市 |

| 止损 | -15% | -12% | -10% |
| 单只上限 | R2分级上限（S≤50%/A+≤35%/A≤25%/A-≤20%） | R2分级上限 | ≤15% |
| 现金底线 | 无 | ≥20% | ≥40% |
| γ催化剂 | 观察池 | 观察池 | 禁止 |
| 持仓数 | ≤8 | ≤6 | ≤4 |

熊市只做α级催化剂，现金≥40%。供给收缩是跨regime有效的唯一策略。

## §8 Lessons

### 赚钱的pattern（v12.0实盘验证: 思源A+/巨化A+/新易盛A-，+10-14%）
- 研究驱动：知道公司做什么、为什么涨、何时涨
- 供给侧结构约束（配额制/物理瓶颈）是最强的护城河
- 催化剂有日期：旺季/政策窗口/财报
- SABCT≥A-才进：高conviction入场才能拿得住

### 亏钱的pattern（v12.0实盘验证: 南风B-/宁波能源B-/英杰B-，-4%到-7%）
- 扫描驱动：涨停板看到，研究不足就买
- 主题驱动（"AI概念"/"储能"）没有具体公司护城河
- SABCT<A-（B+/B/B-）但还是买了
- 买了再研究，不是研究完再买

### 核心教训
- 系统越复杂越差：v9.x简单研究驱动→v11.0复杂UASS扫描，结果越来越差
- Track B是发现工具，不是建仓入口
- 扫描结果给研究方向，不给建仓理由
- 最开始的样子就是最好的：研究员，不是扫描仪
