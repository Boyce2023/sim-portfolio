# AI选股系统性错误诊断
> Agent-10 输出 | 美股交易系统v4.0重建 | 2026-05-22

---

## 一、AI特有错误 vs 通用投资错误

| 错误类型 | 属性 | 回测/实盘证据 |
|---------|------|-------------|
| 过度分散 / 散弹枪仓位 | **AI特有** | 实盘11只，5只2%试单贡献alpha≈0 |

## 二、AI偏差深度分析

### 偏差1：共识偏差（最严重，AI特有）

**机制**：AI训练语料=互联网公开信息=市场共识的文字化。模型权重天然编码了"NVDA好/AMD有竞争力/GOOGL稳健"等共识观点。选股时输出的就是共识，而共识=已定价。

**学术证据**：ACM ICAIF 2025（"Your AI, Not Your View"）实验发现，跨所有主流LLM模型，对科技股、大盘股存在系统性偏好，且在遇到反向证据时表现出强烈确认偏差——模型会"坚持初判"而非更新。

**回测证据**：H1/H2大量交易NVDA/AMD/GOOGL/META等卖方共识名字，反映AI自然落入"15/15看多=already priced in"的Universe。

**修正机制（强制）**：
- 选股前必须跑Consensus Check：卖方≥12/15看多 → 自动踢出做多池
- 对共识股只允许做空，不允许做多
- 每周强制添加1只卖方覆盖≤5的非共识标的

---

### 偏差2：不会做空（AI训练数据偏见，AI特有）

**机制**：互联网语料中做空被描述为"投机/风险/负面"，长期持有被描述为"价值投资/正确"。AI内化了这个价值判断，在生成交易建议时天然回避做空，尽管逻辑上"知道"做空是合法策略。

**回测证据**：空头在H1+H2合计WR=75%，是整个系统最高alpha来源。实盘Day1-5：空头数量=0。这是统计显著的执行失败，不是运气。

**修正机制（强制）**：
- 每个交易日开始前：空头仓位检查。当前空头=0 → 必须执行1笔做空或明确记录"今日无空头机会+原因"
- 空头目标池每周更新：高估值+共识看多+技术面破位 → 优先候选
- 空头最低仓位配额：总组合空头名义价值≥15%（软约束，触发review不强制）

---

### 偏差3：执行gap（写规则但不执行，AI特有）

**机制**：AI在生成文字时"执行"了规则（写了rebalance建议），但实际操作中并不强制自己遵守已生成的建议。规则存在于输出文本中，但不存在于下一个决策节点的context里。每次决策都是相对独立的生成过程。

**实盘证据**：v3.0方法论完善，rebalance框架写清楚，但5天运行中几乎没有主动执行。FPS只调整了60股，其余仓位无调整。

**修正机制（强制）**：
- Pre-trade checklist必须引用上次session的具体决策（复制粘贴，不重新生成）
- 每个session开始：读取`pending_actions.json`，先执行再选新标的
- 新建`pending_actions.json`：记录所有未执行的方法论要求，每次开始必须清零

---

### 偏差4：过度分散 / Sizing倒挂（AI特有）

**机制**：AI倾向于"给每个想法一点仓位"，因为训练数据中"分散风险"被反复强调为正确行为。Conviction分级存在于文字层，但实际sizing时不被严格执行——因为每次sizing都是独立的生成决策，不参考其他仓位的相对conviction。

**实盘证据**：11只持仓，5只≤2%，SRUUF(C级)8.1% > FPS(B+级)4.6%。Sizing与Conviction完全倒挂。

**修正机制（强制）**：
- Conviction-to-Size硬规则表：A+=15-20% / A=10-15% / B+=6-10% / B=3-6% / C=0%（禁止持有）
- Sizing前必须列出当前所有仓位+Conviction，从高到低排序，新仓不得高于任何更高Conviction的现有仓位
- 持仓上限：做多6只 + 做空4只 = 10只总上限，超出必须淘汰最低Conviction标的

---

### 偏差5：Regime Detection缺失（AI特有）

**机制**：AI基本面分析能力强，但市场状态判断（Risk-on/Risk-off/Trending/Ranging）需要实时数据输入。如果不主动调用VIX/利率/市场广度数据，AI默认进入"无状态"模式，相当于永远假设市场处于正常状态。

**修正机制（强制）**：
- Session开始固定三行Regime Check（必须，不可跳过）：
  ```
  VIX: [数值] → [Fear/Greed/Neutral]
  10Y收益率: [数值] → [方向+趋势]
  SPY RSI(14): [数值] → [超买/超卖/中性]
  ```
- Regime = Risk-off（VIX>25）时：禁止开新多仓，空头配额提升至20%
- Regime = Trending Up（VIX<18 + SPY RSI<65）时：允许正常建仓

---

### 偏差6：Recency Bias + 假精确（AI特有）

**Recency Bias机制**：训练语料中近期事件权重更高，导致AI对最近表现好的标的过度乐观（高位追涨），对最近表现差的标的过度悲观（低位踏空）。

**假精确机制**：AI能输出精确的止损价格（"在$X.XX止损"），但实盘中无法tick-by-tick监控，这些精确的止损实际上是装饰性的。

**修正机制（强制）**：
- Recency Check：每只候选标的必须看52周表现，近3个月涨幅>40%的标的进入"Recency Filter"，需要额外一层做空检验才能做多
- 止损改为区间止损（"跌破$X区间收盘确认"），明确标注"非实时监控，每日EOD检查"
- 假精确防护：所有止损/目标价标注"EOD check"，不写"立即止损"

## 三、强制机制汇总（执行检查表）

```
每次Session开始（必须按序执行）：
□ 1. Regime Check: VIX + 10Y + SPY RSI（3行，不可跳过）
□ 2. 读pending_actions.json：先执行上次未完成的方法论动作
□ 3. 空头检查：当前空头=0 → 必须寻找1笔做空机会
□ 4. Conviction排序：列所有仓位+Conviction，检查Sizing是否倒挂
□ 5. Consensus Check：所有新候选标的卖方共识≤11/15才进入池

每次新建仓（必须按序执行）：
□ 1. Conviction定级（A+/A/B+/B/C），C级直接否决
□ 2. 按Conviction-to-Size硬表确定仓位%
□ 3. 检查持仓总数是否≤10只（做多6+做空4）
□ 4. 止损标注"EOD check"，不写实时止损
□ 5. 写入pending_actions.json（如需后续rebalance）
```

## 四、结论

**最致命的三个AI特有错误**（按影响程度排序）：

1. **执行gap**：AI能生成完美规则但不自我执行。修复方案：外部强制机制（pending_actions.json + checklist）
2. **不会做空**：最强alpha来源在实盘中完全闲置。修复方案：空头配额强制 + 每日空头报告
3. **共识偏差**：AI本质上就是市场共识的文字化，选股天然趋向已定价标的。修复方案：Consensus Check硬门槛

通用投资错误（A股执行时机、Backtest lesson闲置）与AI无关，属于系统流程问题，通过checklist和session protocol解决。

**核心洞察**：AI不是"不知道规则"，而是"知道但不执行"。所有修正机制必须是外部强制（硬编码检查表、数值门槛），而不是"提醒AI注意"。
