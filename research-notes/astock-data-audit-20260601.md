## 一、核心结论（3句话）

1. **UASS只扫涨停池(`stock_zt_pool_em`)，创业板/科创板10%-19%涨幅完全是盲区** — 这是设计缺陷不是bug
2. **东方财富API全线失败的根因是Shadowrocket VPN的TUN模式DNS劫持** — `*.eastmoney.com`被解析到假IP `198.18.0.171`，修复VPN规则即可恢复
3. **修复方案：`stock_zh_a_spot_em()`全市场扫描筛涨幅>5%，或push2delay按涨幅翻页** — `zt_pool_strong_em`是历史筛选非当日强势，不能用

## 二、UASS盲区诊断

### 根本原因

```
UASS数据入口: ak.stock_zt_pool_em(date) → 只返回封板股
                                          ↓
主板涨停线 ±10%  → 9.8%+的进入 ✓
创业板涨停线 ±20% → 需要19.5%+ → 10%-19%完全不在 ✗
科创板涨停线 ±20% → 同上 ✗
```

全市场涨幅>5%非涨停股：**332只**，TOP100中漏了25只。

### 修复方案

~~**~~`stock_zt_pool_strong_em`不是"今日大涨未涨停"~~** — 它是60日新高/近期多次涨停的历史筛选，润和软件同样不在里面。143只中77只今日涨幅<5%。~~

**正确方案：全量行情筛选**

```python
# 方案A：akshare全市场（VPN修复后可用）
import akshare as ak
df = ak.stock_zh_a_spot_em()  # 5,500+只，3秒
strong = df[df['涨跌幅'] > 5]  # 筛涨幅>5%的，约300-500只

# 方案B：push2delay分页（不依赖VPN修复）
# 按涨幅降序翻页，遇到涨幅<5%停止，前3-5页=300-500只
url = "https://push2delay.eastmoney.com/api/qt/clist/get"
params = {
    "pn": "1", "pz": "100", "po": "1",  # po=1降序
    "fid": "f3",  # 按涨幅排序
    "fs": "m:0+t:6,m:0+t:80,m:1+t:2,m:1+t:23,m:0+t:81+s:2048",
    "fields": "f2,f3,f12,f14,f20,f21"
}
# 翻到涨幅<5%停止
```

两个方案都能覆盖创业板/科创板10-19%的盲区。方案B不依赖VPN修复，更稳健。

## 五、VPN问题修复

### 问题
Shadowrocket TUN模式将`*.eastmoney.com`的DNS解析劫持到`198.18.0.171`（虚假IP），导致：
- akshare的EM源接口全部超时
- efinance实时行情不可用
- UASS板块资金/概念板块数据获取失败

### 修复步骤
1. 打开Shadowrocket → 设置 → 规则
2. 添加规则：`DOMAIN-SUFFIX,eastmoney.com,DIRECT`
3. 同时添加：`DOMAIN-SUFFIX,push2.eastmoney.com,DIRECT` 和 `DOMAIN-SUFFIX,push2ex.eastmoney.com,DIRECT`
4. 保存并重连

### 验证
```bash
# 修复前
curl -s "http://push2.eastmoney.com/api/qt/clist/get?..." → Empty reply
# 修复后应返回JSON数据
```
