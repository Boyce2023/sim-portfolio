# Agent-19: 自主发现机制设计框架
**生成日期**: 2026-05-22
**目标**: 从"用户说才动"升级到70-80%机会自主发现

---

## 核心诊断：当前为什么不够自主

当前模式失败的根本原因不是工具缺失，而是**扫描范围被共识锁死**：
- 只看用户点名的标的（NVDA/AMD/GOOGL）
- 没有主动扫描低于共识雷达的机会
- 回测中最高收益的trade（SNDK +680%、STX +236%、DG +25.6%）全是非共识发现，不是用户指派

目标：建立每日主动扫描协议，让系统自己"嗅到"这类机会。

---

## 一、每日主动扫描SOP（10分钟，BJT 08:00执行）

> 美股收盘后、A股开盘前的信息密度最高窗口。每步有明确的时间上限和输出物。

### Step 1 — 宏观仪表盘（2分钟）
**命令**: `yf macro`
**检查项**:
- VIX水平及日变化：>25 = 防御模式；<18 = 风险偏好开启
- DXY日变化：>+0.5% = 新兴市场压力；<-0.5% = 商品/国际股受益
- 10Y美债yield日变化：>+5bp = 成长股估值承压；大幅下行 = XLRE/XLU受益
- 原油价格日变化：>+3% = XLE买入信号前置；<-3% = XLE减仓信号
**输出**: 一行判断 → `[宏观] 风险偏好ON/OFF | 关注方向: {板块}`

### Step 2 — 隔夜异常个股扫描（3分钟）
**命令**: `yf news` + 盘后涨跌幅筛选
**触发阈值**:

| 信号 | 阈值 | 含义 |
|------|------|------|
| 个股单日涨幅 | >+8% + 成交量>均值3x | 研究触发：有结构性催化剂？ |
| 个股单日跌幅 | <-8% + 成交量>均值3x | 研究触发：做空机会？thesis被证伪？ |
| 盘后财报反应 | >+10% or <-10% | 立即评估：是否影响watchlist同类标的 |

**操作**:
- 发现触发信号 → 记录标的名 + 涨跌幅 + 初步原因分类（财报/宏观/行业/个股）
- 超过3只触发 → 优先处理涨幅最大和跌幅最大各1只
- 2分钟内完成分类，不深入研究（深入留给Step 5）

### Step 3 — SEC Filing快速扫描（2分钟）
**数据源**: SEC EDGAR（[https://www.sec.gov/cgi-bin/browse-edgar](https://www.sec.gov/cgi-bin/browse-edgar)）或 WebSearch "SEC EDGAR 8-K today"
**重点Filing类型**:

| Filing | 触发条件 | 含义 |
|--------|---------|------|
| 8-K | 任意watchlist公司 | 重大事件：并购/管理层变动/guidance更新 |
| S-1/S-11 | 新IPO申请，市值预计>$3B | 新标的候选 |
| SC 13G/13D | 知名机构新建仓（>5%） | Smart money信号 |
| Form 4 | 3人+30天内insider buy | Insider cluster确认信号（配合RULE F-4） |

**操作**: 30秒扫头条，发现相关filing记录标的名，留Step 5深入。

### Step 4 — 财报日历检查（2分钟）
**命令**: `yf news` 检索下周财报
**检查内容**:
- 未来7天哪些watchlist标的有财报？→ 确认催化剂日历是否需要更新
- 未来7天有无超大市值公司财报（>$100B）？→ 市场情绪牵引方向
- 本周财报beat/miss中有无影响组合持仓的同类公司？

**输出**: `[财报] 下周关注: {标的} {日期} {预期vs现实}`

### Step 5 — S&P季度再平衡检查（1分钟）
**触发时机**: 每季度末月（3/6/9/12月）的第一个周五后
**检查方法**: WebSearch "S&P 500 index changes {month} {year}"
**执行规则**: 发现公告立即对照RULE F-1执行机械策略，优先级高于其他所有任务

## 二、异常检测信号清单（什么应该触发"这里有机会"）

### 高确定性信号（立即研究，≤4小时响应）

**I-1: S&P成分股公告（★★★）**
- 触发: 官方公告新增/删除S&P 500成分股
- 响应: 执行RULE F-1，公告后次日开盘买入（新增）或评估做空（删除）
- 历史依据: APP +26.8%、HOOD +24.8%（H1回测）

**I-2: 个股单日8%+异常涨跌 + 3x成交量（★★）**
- 触发: 盘后或盘前扫描发现
- 响应: 5分钟快速评估（是财报beat？是并购？是宏观噪音？）
- 过滤条件: 必须有可识别的基本面原因，纯情绪波动排除
- 历史依据: SNDK/STX的storage supercycle发现就来自异常成交量

**I-3: 板块ETF连续3天跑赢SPY >1%/天（★★）**
- 触发: XLK/XLE/XLV等连续3日相对强势
- 响应: 进入对应板块做选股扫描（不追ETF本身，找其中mid-cap机会）
- 历史依据: Apr 2026 XLK +20%单月，连续领先是板块切换信号

**I-4: VIX单日变化>15%（★★）**
- 触发: VIX单日>15%涨幅或跌幅
- VIX急涨: 切换防御模式，检查组合downside暴露；不新建做多头寸
- VIX急跌: 风险偏好打开，重新评估被压制的成长股
- 历史依据: Mar 2026伊朗战争VIX飙升，4月停火VIX回落对应XLK +20%

### 中确定性信号（48小时内评估）

**I-5: Insider Cluster Buy（★）**
- 触发: 同一公司3人+在30天内买入Form 4，每人>$100K
- 响应: 加入watchlist观察，配合其他信号使用
- 独立开仓标准: 需同时满足I-2或其他高确定性信号

**I-6: 大额回购公告（★）**
- 触发: 新回购授权≥$10B 或 占市值≥3%
- 响应: 对照RULE F-2三项条件（授权规模/实际执行/近期回调≥10%）
- 历史依据: Visa $20B授权后+4.1%；Qualcomm $20B授权

**I-7: 知名机构新建仓13G/13D（★）**
- 触发: 知名机构（Berkshire/Pershing Square/Tiger Global等）新披露>5%持仓
- 响应: 研究该标的是否符合系统评分框架，不盲目跟进
- 过滤: 必须通过Agent-11的5维度评分

**I-8: 同板块龙头财报beat后mid-cap未联动（★）**
- 触发: XLK/XLE内龙头财报beat，但同板块mid-cap标的未跟随上涨
- 响应: 评估mid-cap是否有同样的基本面改善，发现定价滞后机会
- 历史依据: NVDA财报beat后，SNDK/STX storage play滞后定价→H2 +680%/+236%

## 三、信息源优先级（分层执行）

| Tier | 查询频率 | 来源 | 工具 |
|------|---------|------|------|
| **Tier 1** 每日必查 | BJT 08:00 | yfinance价格/新闻、SEC EDGAR当日filings | `yf macro` / `yf news` / `yf price` |
| **Tier 2** 每周一次 | 周一 | ETF fund flows（iShares/SSGA）、13F季报、分析师升降评级 | WebSearch |
| **Tier 3** 触发时查 | 信号触发后 | 公司investor day、产业会议、深度WebSearch | `web-access` skill |

**优先级铁律**: Tier 1数据比Tier 2/3高一个量级，Tier 3只在Tier 1/2信号确认后启动。

## 四、从信号到Trade Idea的决策流程

```
发现异常信号（Step 1-5）
    ↓
[快速评估 5分钟]
  - 信号类型: I-1~I-8哪一类？
  - 基本面有无支撑? (有新闻/Filing/财报？)
  - 当前组合有无暴露该标的的相关头寸？
    ↓
[过滤条件]
  - 2-3个维度有信号？→ 进入深度研究
  - 只有1个模糊信号？→ 加入Watchlist，设定跟踪条件
    ↓
[深度研究 <4小时]
  - 运行5维度评分（Agent-11框架）
  - 完成进场检查表（CLAUDE.md必填项）
  - Bear case downside%（>35%→排除；25-35%→T级试仓）
    ↓
[Trade Decision]
  - 通过所有评分 + bear case ≤25% → 建仓（给用户推荐附完整评分+rationale）
  - 未通过但有潜力 → 加Watchlist，设再次触发条件
  - 未通过 → 记录排除原因（弹药库思维：记录覆盖≠开枪）
    ↓
[时间限制]
  - 从信号发现到build position: ≤48小时
  - 超过48小时 → 机会可能已过，重新评估必要性
```

## 五、每周自主产出目标

**最低目标**: 每周2-3个新trade idea（附完整评分），用户可否决但不需要指派。

**产出格式**（每个idea包含）:
```
标的: [TICKER]
发现来源: [I-1~I-8信号类型]
一句话thesis: [...]
信心等级: S/A/B/C
建议仓位: [对应%上限]
Bear case downside: [%]
催化剂+日期: [...]
Agent-11评分: [5维度得分]
推荐行动: [建仓/Watchlist/排除]
```

**周一例行任务**（ETF流量检查，15分钟）:
- 检查过去一周XLK/XLE/XLV/XLF流量方向
- 更新板块偏好判断（对照agent-sector.md轮动框架）
- 如板块偏好发生切换 → 触发组合再平衡评估

## 六、减少对用户输入的依赖——行为改变要点

| 旧模式 | 新模式 |
|--------|--------|
| 用户说"看看XXX" → 研究 → 建议 | 每日扫描发现异常 → 研究 → 主动推荐给用户 |
| 只看用户watchlist内标的 | 扫描范围覆盖S&P 500全体成分股 + 盘后异动非共识标的 |
| 等财报季才评估仓位 | 每日Step 4实时追踪催化剂日历 |
| 热门名字优先（NVDA/AMD/GOOGL） | Mid-cap非共识机会优先（DG/SNDK型） |
| 用户challenge才验证信号 | 每个信号自动过I-1~I-8分级过滤器 |

**核心认知转变**: 覆盖面广是为了淘汰（弹药库≠开枪）。每周发现20个信号，最终产出2-3个trade idea，18个被过滤掉是系统正常工作，不是失败。
