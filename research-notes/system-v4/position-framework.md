## 一、核心结论（先行）

**旧系统三大仓位致命伤**：散弹枪11只（5只2%试单alpha≈0）、Sizing与Conviction倒挂（SRUUF C级8.1% > FPS B+级4.6%）、最低建仓$3K在$150K里是2%（手续费都回不来）。

**修复逻辑**：AI的"给每个想法一点仓位"是训练偏差，不是风控智慧。强制集中度规则是必需的外部机制——不是建议，是硬约束。

## 三、集中度硬规则

```
多头上限：6只
  - S级：最多1只
  - A级：最多2只
  - B级：最多2只
  - C/T级：最多1只（共享槽位）

空头上限：3只

总持仓上限：9只（消灭11只散弹枪）

现金下限：≥15%（从20%下调；集中度提高后单只质量更高，可适度降低现金缓冲）
```

**槽位管理原则**：所有槽位已满时，新建仓必须先清退最低Conviction的现有仓位。不允许"临时超过上限后再调整"。

## 四、加仓决策树

```
触发问题：是否应该加仓？
│
├─ 催化剂已经确认落地？
│   ├─ 是 → 进入加仓检查
│   └─ 否 → 禁止加仓（不是"涨了所以加"，是"catalyst confirmed才加"）
│
└─ 加仓检查（所有条件必须满足）：
    ├─ 距上次建仓/加仓 ≥ 3个交易日？
    ├─ 本次加仓量 ≤ 初始仓位的50%？
    ├─ 加仓后总仓位 ≤ 该等级上限？
    └─ Conviction等级未降级？
        ├─ 全部满足 → 执行加仓
        └─ 任一不满足 → 不加仓，记录原因到pending_actions.json
```

**加仓禁止场景（明确列举）**：
- 仅因股价上涨加仓（Recency Bias的典型表现）
- 催化剂当天追涨（catalyst当天禁止操作，等1周资金验证）
- 加仓后超出等级上限（无例外）
- 距上次操作不足3个交易日

## 五、减仓决策树

```
触发问题：是否应该减仓/清仓？
│
├─ 止损触发（EOD收盘价跌破止损线）？
│   └─ 是 → 全仓清出，无例外，不等反弹
│
├─ Conviction降级（重新打分后低于当前等级）？
│   └─ 是 → 立即减至新等级上限（超出部分当日清退）
│         示例：A级(15%)降至B级 → 减至10%，当日执行
│
├─ 达到目标价？
│   └─ 是 → 减半（卖出50%），剩余50%转Trailing Stop模式
│         Trailing Stop：从最高价回落12%触发清退（EOD确认）
│
├─ 需要腾位置给更高Conviction新标的？
│   └─ 是 → 清退Conviction最低的现有仓位（整只清，不是减半）
│
└─ 以上均不触发 → 持有（L11铁律：催化剂前默认持有，不因"涨了不少"减仓）
```

**减仓顺序优先级（多仓位同时需要减时）**：
1. 止损触发的无条件先减
2. Conviction最低的先减
3. 持仓时间最长且未出现新催化剂的先减

## 六、H1+H2 数据回测验证

### 测算逻辑
用H2时间窗口内138笔交易（排除Agent 15跨期）做两套Sizing对比：

**情景A：Equal Weight（旧系统散弹枪）**
- 每笔平均仓位 = 组合总仓位 / 持仓只数
- 实盘模拟：138笔均分仓位，平均收益 +6.59%

**情景B：Conviction-Weighted（新框架）**
- Top 10 winners（AMD×3、MU、COIN Short、CEG Short等）→ 给15%仓位（S/A级）
- 中间60笔（Avg PnL +7-12%）→ 给10%仓位（A/B级）
- 后30笔losers（Avg -5.18%）→ 这些标的在新框架下多数不会进入持仓（Conviction评分低于20分 = T级或直接排除）

### 简化测算结果

| 情景 | Top 10仓位 | 其余仓位 | 加权收益估算 |
|------|-----------|---------|------------|
| 情景A Equal Weight | ~7-8%/只 | ~7-8%/只 | +6.59%（实际H2均值） |
| 情景B Conviction Weighted | **15%**（S/A级） | 7-10%（B级） | **+9.2%估算** |

**估算方法**：Top 10 trades（时间窗口版）平均 +41.2%，占组合15%仓位，贡献 +6.2pp；中间trades平均 +6.59%，占组合65%仓位，贡献 +4.3pp；合计 +10.5pp，扣除Losers占剩余20%仓位约 -1.3pp，净增益约 **+9.2%** vs 旧系统 +6.59%。

**结论**：集中Sizing可提升组合回报约+2.6pp（估算，置信度中等）。真正收益不在于精确数字，而在于消灭了"5只2%试单贡献alpha≈0"的沉没成本，将那部分仓位集中到有效trades。

**关键验证点**：Agent 10（91%WR，+18.2%）的11笔trades集中在AMD/GOOGL/AMZN，完全符合新框架逻辑——高Conviction少量标的，大仓位，系统性入场。Agent 09（30%WR，-0.4%）的10笔做空分散到各种标的，正是散弹枪失败的最佳反面案例。

## 七、与现有规则的接口说明

**与strategy.md现行规则的关系**：
- 本框架的Conviction→Sizing表**替代**当前strategy.md §铁律5中的分级上限（旧规则：S≤40%/A≤25%/B≤15%/C≤8%，已过时，仓位上限过高）
- 集中度硬规则（9只上限）**替代**现行11只隐性上限
- $7,500最低建仓**新增**规则，当前strategy.md无此门槛

**T级与旧"试仓"概念的区别**：
- 旧试仓：3%（$4,500），无时限，无升级要求
- T级：5%（$7,500），2周硬时限，必须升级到C级或以上，否则清退
- T级是"资格审查期"，不是"永久小仓位"

**与ABCD下跌分类的接口**：
- D类（Thesis证伪）→ 跳过所有减仓程序，直接全清，优先级高于一切
- C类（叙事切换）→ 触发Conviction重新打分，如降级则执行减仓树
- A/B类 → 不触发任何减仓，Hold
