# Multi-Strategy Trading Playbook v5.0

> 来源: 15个全球顶级基金方法论 + 35项学术/实战研究 = 50-agent并行综合
> 核心认知: 不同类型的股票是完全不同的资产类别，之间的差异可能比股票和房地产还大

---

## Part 1: 核心理念

### 1.1 Multi-Fund架构

一个$8.8K的IBKR账户 = 一个微型multi-strategy基金。参照Citadel/Millennium的pod model：

| 概念 | 机构版 | 个人版($8.8K) |
|------|--------|---------------|
| Pod | 独立PM团队 | Strategy Book (虚拟子账户) |
| 风险预算 | CIO分配 | 按策略类型固定比例 |
| Stop-out | -5% pod清仓 | -15% book减半 / -20% book清仓 |
| 因子暴露控制 | 风控组日报 | 每周检查"同一天大跌"测试 |

**三本策略书 (适合$8.8K阶段)**:

| Book | 占比 | 策略类型 | 持仓数 | 换手 |
|------|------|----------|--------|------|
| **Book A: 动量** | 40% (~$3,500) | 趋势突破 + 财报动量 | 2-3只 | 月度 |
| **Book B: 事件** | 35% (~$3,100) | 催化剂驱动 + 周期复苏 | 2-3只 | 按事件 |
| **Book C: 价值** | 15% (~$1,300) | 深度价值 + 成长复利 | 1-2只 | 季度 |
| **现金** | 10% (~$900) | 风险缓冲 + 机会储备 | — | — |

**铁律: 规则跟着Book走，不跟着感觉走。** 动量Book里的票跌了就止损；价值Book里的票跌了可能是加仓机会。用momentum规则管理value持仓 = 最高频失败模式。

### 1.2 分类先于一切

**买入前第一步不是"要不要买"，而是"这是什么类型的股票"。** 分类决定了后续所有规则——入场、退出、仓位、止损、持仓周期全不同。

Peter Lynch是第一个系统做这件事的人（六种分类），本系统扩展为8种原型。

## Part 2: 8大股票原型 (Archetypes)

### Archetype 1: 动量突破型 (Momentum Breakout)
> Book归属: Book A | 典型: CRWD, VRT, DELL

**识别标准**: 52周新高或近新高，RS Rating ≥ 80，机构持仓增加，基本面趋势向上

**买入规则**:
- 首选: 放量突破新高时买入（量 > 20日均量 × 1.5）
- 次选: 突破后3-5%健康回调时买入（量萎缩 ≥ 50%，守住20日均线）
- **强牛市追突破（胜率68-76%），震荡市等回调（胜率71-74%）**
- 分批入场: 试仓40% → 确认+30% → 趋势加速+30%（Druckenmiller三步法）

**卖出规则**:
- Trailing stop: 2× ATR(14)，有浮盈后升至2.5× ATR
- Climax top: 新高后当周收盘低于前周 + 3倍以上放量 + 收盘在低位 → 卖出
- 动量衰竭: RSI顶背离 + 价涨量缩 + 跌破20EMA → 三者同时出现才卖
- **"涨了很多"不是卖出理由**

**止损**: -7%至-10%（低波动），或 1.5-2× ATR（高波动）。ATR止损比固定%年化多赚1.7%、回撤低12%

**仓位**: 单笔风险 ≤ 账户2%。单股最高15%（A级信心）。比其他类型允许更大仓位

**持仓周期**: 6-12个月。动量收益在前12个月最强（Jegadeesh & Titman），超过12个月开始衰减

**特有陷阱**: FOMO追顶 / 不止损（"趋势还在"） / 回调分不清健康vs反转

**回调分级**:
| 回调幅度 | 判断 | 操作 |
|----------|------|------|
| < 5% | 健康（量缩+守20MA） | 加仓 |
| 5-10% | 观察（检查50MA） | 等信号 |
| 10-15% | 警戒 | 减半或观望 |
| > 15% + 破50MA放量 | 趋势反转 | 止损 |

### Archetype 2: 深度价值型 (Deep Value)
> Book归属: Book C | 典型: RTX(关税冲击后), DIS(2023底部)

**识别标准**: 大幅低于内在价值（折价30-50%+），市场忽视，Piotroski F-Score ≥ 5

**买入规则**:
- 左侧分批: 首仓40% → 每下跌15%补一批（前提: 基本面已验证）
- **有催化剂优先**。折价 < 35%必须有催化剂；折价 > 50%可以无催化剂但需确认负债表
- Klarman三重估值: NPV + 清算价值 + 市场价值，取最保守

**卖出规则**:
- 到达内在价值即减仓（"永远卖太早"——Klarman原则）
- 估值溢价超历史中枢30%+ → 开始减仓
- Thesis broken → 全清

**止损**: 用基本面止损，不用价格止损
- F-Score跌破5 → 减半
- FCF连续2季度转负 → 全清
- 债务评级降至垃圾级 → 全清
- **价格下跌 ≠ 止损信号，可能是加仓机会**

**仓位**: 单只 ≤ 8%（比动量小一半，因不确定性更高）

**持仓周期**: 12-36个月。均值回归是慢变量，12个月内不因"没涨"卖出

**特有陷阱**: 越跌越买（锚定效应） / 不认错 / 混淆周期低谷和结构衰退

**Value Trap三问** (不过关 = 不买):
1. 为什么便宜？（周期还是结构？）
2. 现金流质量如何？（FCF趋势）
3. 护城河在侵蚀吗？（ROIC vs WACC）

### Archetype 3: 财报动量型 (Earnings Momentum / PEAD)
> Book归属: Book A | 典型: DELL(+16.8% gap-up后), AVGO(earnings beat)

**识别标准**: EPS surprise ≥ 5% + Revenue surprise ≥ 2%，最好三合一（EPS+Revenue+Guidance上调）

**买入规则**:
- 小盘股($300M-$2B): Day 2-3入场最优（初始波动消化，drift窗口长）
- 大盘股: Day 1收盘前入场或放弃（大盘drift快速消化）
- Revenue beat含更难消化的信息，drift持续性更强

**卖出规则**:
- 持仓15-30天（alpha密度最高区间）
- 或持至下个财报前复评
- 连续beat序列 > 首次beat（中线15-60天占优）

**止损**: Gap fill止损（gap up入场时） / 固定-8%（回调入场时）

**仓位**: 单笔 ≤ 10%

**持仓周期**: 15-60天

**特有陷阱**: 用Whisper预期而非consensus（超越consensus但未超Whisper → 55%概率收跌）

**PEAD学术数据**: 年化异常收益~5-10%（已从原始18%衰减），大盘股几乎消失，小盘股仍有可用alpha

### Archetype 4: 事件驱动催化剂型 (Event-Driven Catalyst)
> Book归属: Book B | 典型: MRVL(财报事件), 生物医药(FDA), Spin-off

**识别标准**: 有明确日期的硬催化剂（已宣布的，不是预期中的）

**买入规则**:
- 事件前建半仓（pre-event drift有实证） + 结果确认后调整
- **Buy the rumor, sell the news成立条件**: 高拥挤度 + 结果符合预期
- **不成立条件**: 结果大幅超预期 或 触发二阶催化剂
- Spin-off首周是最经典的错误定价窗口（机构被迫抛售）

**卖出规则**:
- 催化剂兑现后强制重新评估。说不出下一个持有理由 → 卖
- 超预期可留，符合预期往往price in
- 催化剂硬失败 → 立即离场。T+2~T+5反弹是dead cat bounce

**止损**: 事件失败立即止损（不等反弹）。预设止损价 = 事件前入场价 - X%

**仓位**: 高确定性 ≤ 10-15%，Binary低确定性 ≤ 3%

**持仓周期**: 事件前1-4周入场 → 事件后1-2周退出

**特有陷阱**: 过度自信（以为有信息优势） / 事件失败不止损

**各类事件胜率**:
| 事件类型 | 正回报概率 | 平均回报 |
|----------|-----------|---------|
| FDA批准 | ~60% | +5%~+15% |
| M&A目标方 | >90% | +20%~+40% |
| Earnings超预期 | ~55-60% | +3%~+8% |
| 行业会议 | ~50% | +2%~+5% |

### Archetype 5: 长期成长复利型 (Secular Growth Compounder)
> Book归属: Book C | 典型: AVGO, MSFT, AAPL (长期持有部分)

**识别标准**: ROIC>20% + 有机增速>7% + FCF转化率>80% + 可量化护城河，六项同时满足

**买入规则**:
- 历史PE均值 × 1.3以下 = 绿灯，× 1.6以上 = 等回调
- 回调分级: 5-10%小幅加 / 10-20%验证thesis再加满 / 20-30%先重审 / >30%需找到"市场错了"的具体反驳

**卖出规则**:
- Terry Smith: "Do nothing" 是策略。年换手 < 5%
- 卖出触发: ROIC连续3季度下滑 + 应收天数扩大 + 连续2季miss guidance（两项以上并发）
- **"涨太多"不是卖出理由。** 2.5× 以上触发强制复查，但thesis不变就继续持有

**止损**: 基本面止损，不用价格止损。护城河侵蚀信号才是真正的止损

**仓位**: 起始5-8%，软顶15%，每次减仓25-30%，不清仓

**持仓周期**: 3-10年。真正的复利需要时间

**特有陷阱**: 太早获利了结是最糟糕的错误（Baillie Gifford数据: 所有超额回报集中在极少数outlier）

---

### Archetype 6: 周期复苏型 (Cyclical Recovery)
> Book归属: Book B | 典型: 能源股(周期底), 半导体(库存低谷), 航运

**识别标准**: Lynch逆PE法则——PE高/亏损 = 买入信号，PE创低 = 卖出信号

**买入规则**:
- 领先指标确认底部: ISM PMI触底回升 → 高收益债利差收窄 → 库存/销售比下降 → 产能利用率回升
- 底部三要素: 现金流转正 + 产能退出 + 机构低配

**卖出规则**:
- PE创低位 + 利润创高位 = 卖出（不是买入！）
- 7大顶部信号出现3个即减仓
- **周期股不留底仓**，等下一个底重买

**止损**: -15%至-20%。周期股波动大，止损需要更宽

**仓位**: 单行业上限20%，个股 ≤ 15%

**持仓周期**: 库存周期3-4年 / 资本支出周期8-10年 / 信用周期15-20年

**特有陷阱**: 在周期顶部还觉得"这次不一样" / PE低时以为便宜（其实是顶部）

---

### Archetype 7: 投机/故事型 (Speculative / Story)
> Book归属: 不单独设Book，从现金池中小额支出 | 典型: HSAI, SMR, OKLO, meme stocks

**识别标准**: 高波动、叙事驱动、可能无盈利、依赖未来预期

**买入规则**:
- 必须有增量催化剂（不是"这个故事很好"）
- Beta归属先分析（F18）: HSAI以为交易LiDAR，实际交易地缘政治
- 分不清在交易什么 → 不买

**卖出规则**:
- R1锁1/3 → R2锁50%并移止损至保本 → R3+用trailing stop
- **不在投机股上追加仓位**
- Story Over三信号: 催化剂断供 / 叙事被证伪 / 管理层行动矛盾叙事

**止损**: -15%固定。或 2-3× ATR（但通过缩小仓位保证dollar loss ≤ 组合1-2%）

**仓位**: 单票 ≤ 3%。全组合投机仓位硬上限10%

**持仓周期**: 3-12个月（动量甜区），12个月后反转概率显著上升

**再入场规则**: 可以再买同一标的，但需: ①Beta归属重新验证 ②有新增量催化剂 ③情绪冷却期 ≥ 2周。仓位从头起步，禁止"找回来"心态

**特有陷阱**: 报复性交易（HSAI -$2,583教训）/ R1赚了就加码R2（OKLO -$448教训）

---

### Archetype 8: 股息收入型 (Dividend / Income)
> Book归属: Book C | 典型: WMB, ABBV, Utilities

**识别标准**: 股息增长率 > 股息率（增长优先）。FCF覆盖率 > 1x，派息比率 < 80%

**买入规则**:
- 加息尾声开始建仓，降息初期满仓
- 10Y < 4%全面利好，> 4.5%降低权重
- Dividend trap识别: yield比行业均值高 > 3ppt需解释

**卖出规则**:
- 股息削减 → 立即卖出
- 派息比率 > 90% → 立即卖出
- 盈利连续下滑 / 估值透支 / 股息增长脱轨 → 计划性减仓
- **短期股价下跌不是卖出理由，是加仓机会**

**止损**: 基本面止损（股息削减/FCF恶化），不设价格止损

**仓位**: 上限占组合30%（全部股息类），单只不超10%

**持仓周期**: 3-5年+。DRIP自动再投资

**特有陷阱**: 高yield trap（跌出来的高收益率）/ 利率风险被低估

## Part 3: 原型分类决策树

```
新标的进入 →
  ├─ 52周新高/近新高 + RS>80?
  │   ├─ YES + 财报beat → Archetype 3 (PEAD) → Book A
  │   └─ YES + 趋势持续 → Archetype 1 (Momentum) → Book A
  ├─ 大幅低于内在价值 + F-Score≥5?
  │   ├─ YES + 有催化剂 → Archetype 2 (Deep Value) → Book C
  │   └─ YES + PE逆信号 → Archetype 6 (Cyclical) → Book B
  ├─ 有明确日期催化剂?
  │   └─ YES → Archetype 4 (Event-Driven) → Book B
  ├─ ROIC>20% + 增速>7% + 护城河?
  │   └─ YES → Archetype 5 (Compounder) → Book C
  ├─ 高股息 + FCF稳定?
  │   └─ YES → Archetype 8 (Dividend) → Book C
  └─ 以上都不是 + 高波动叙事?
      └─ YES → Archetype 7 (Speculative) → 现金池
```

**分类后不切换框架。** 买入时定为momentum的票，跌了不能临时切换成value来合理化持有。Baillie Gifford在Peloton上犯的最大错误就是"跌了就换框架"。

## Part 4: 风控框架

### 4.1 仓位管理公式

**ATR-based sizing (所有原型统一公式)**:
```
股数 = (账户净值 × 风险比例) / (ATR(14) × ATR倍数)

风险比例: Book A 2% / Book B 1.5% / Book C 1%
ATR倍数: Momentum 2× / Event 2× / Value 3× / Speculative 2.5×
```

**示例 ($8,800账户, CRWD ATR=$15)**:
```
股数 = ($8,800 × 2%) / ($15 × 2) = $176 / $30 = 5.8 → 5股
仓位 = 5 × $663 = $3,315 = 37.7% → 超过单股15%上限 → 降至2股 = $1,326 = 15%
```

### 4.2 止损分级

| 原型 | 止损类型 | 宽度 | 依据 |
|------|----------|------|------|
| Momentum | 价格止损 | -7%~-10% 或 2×ATR | 趋势断就走 |
| Deep Value | 基本面止损 | F-Score/FCF/信用评级 | 价格跌≠止损 |
| PEAD | Gap fill或-8% | 固定 | 短期交易 |
| Event-Driven | 事件失败即止损 | 无固定% | 催化剂消失就走 |
| Compounder | 基本面止损 | 护城河侵蚀信号 | 长期持有 |
| Cyclical | -15%~-20% | 偏宽 | 高波动 |
| Speculative | -15%固定 | 中等 | 控制损失 |
| Dividend | 股息削减 | 基本面 | 不设价格止损 |

**Kaminski & Lo (2014) 核心发现**: 止损在动量市场有效（月均+50-100bps），在均值回归市场有害。是否用价格止损取决于标的类型，不是个人偏好。

### 4.3 多层回撤管理

| 层级 | 触发 | 操作 |
|------|------|------|
| **Book级** | 单Book -15% | 该Book仓位减半 |
| **Book级** | 单Book -20% | 该Book清仓，暂停2周 |
| **Portfolio级** | 全账户单月-10% | 暂停新建仓 |
| **Portfolio级** | 全账户单月-20% | 进保本模式（只减不加） |
| **Circuit Breaker** | 日内连亏2-3笔 | 当日停止交易 |
| **Circuit Breaker** | 单周亏5-8% | 下周仓位减半 |

**回撤数学**: -10%需+11%恢复 / -20%需+25% / -50%需+100%。恢复时间是回撤时间的2-3倍（Triple Penance Rule）。

### 4.4 杠杆规则

| 规则 | 阈值 | 理由 |
|------|------|------|
| 总杠杆上限 | 1.3× NLV | 1.5×以上单次大跌可能导致margin call |
| 单票上限 | $1,500 (~17% NLV) | META 270%杠杆单只→margin清仓=-$2,040 |
| 动量仓位不用杠杆 | 0杠杆 | 动量反转时闪崩，IBKR实时清仓无反应时间 |
| 价值仓位可用适度杠杆 | 最高1.2× | 价值股反转较慢，有反应时间 |
| VIX > 25 | 杠杆降至1.0× | 高波动+高杠杆=数学必死 |

### 4.5 相关性检查

**"同一天大跌"测试**: 对每只持仓问——"这只票涨30%最可能的宏观条件是什么？"

如果多只票答案相同（如AVGO+VRT+DELL都是"AI hyperscaler capex加速"），就是隐性集中，不是真正的分散。

- 单行业不超过40%组合
- 单因子暴露（momentum/value/growth）不超过60%
- 每季度用滚动相关系数检查

### 4.6 交易成本约束

**68笔/月在$8.8K账户 = 年化成本~10%**（Barber & Odean: 最活跃交易者年化alpha -10.4%）

- 目标: 每月 ≤ 15笔（~$30交易成本，年化~4%）
- Book A: 每月4-6笔（含入场+止损+exit）
- Book B: 每月2-4笔（按事件）
- Book C: 每月1-2笔（低换手）
- 每笔交易前问: "这笔交易的边际价值 > 交易成本吗？"

## Part 5: 选股与入场/退出

### 5.0 选股源 (Lead Generation)

选股来源是多元的。以下是几种主要的候选池构建方式，找到后统一进入5.1入场决策树。

#### 来源A: 叙事验证周期 (Narrative Validation Cycle)

> 来源: Nokia模式观察。2026-05-23。
> 注意: 这是选股灵感之一，不是硬性框架。Nokia是单一案例，胜率未经大样本验证。巨头押注失败案例很多(Rivian/WeWork/Mobileye)，幸存者偏差严重。

**Pattern描述**:

科技巨头做出大额确切性押注(≥$5亿真金白银) → 故事太大一个季度证明不完 → 每次验证多一批人信 → 阶梯式上涨

**Nokia教科书案例**:
```
Oct 2025: NVIDIA投$10亿入股2.9% ← 确切性押注(Step 1)
$4-5 横盘多年 → 市场不信，供给耗尽
Q1 2026: AI收入+49% YoY ← 第一次验证(Step 2) → 股价开始动

```

**参考信号(越多越好，但非硬性门槛)**:

| # | 条件 | 验证方法 | Nokia示例 |
|---|------|---------|-----------|
| 1 | **巨头确切性押注** | 实际投资/收购，不是合作MOU/口头承诺 | NVIDIA $10亿真金白银 |
| 2 | **故事足够大** | TAM是当前收入的5-10×以上 | AI-RAN TAM CAGR 27%，1500万基站 |
| 3 | **验证周期长** | 完全兑现需2-3年+，不是一个季度能证明的 | 从试验→商用→全球部署=3-5年 |
| 4 | **长期横盘/被忽视** | 之前6个月+在窄幅区间，市场不关注 | $4-5区间横盘数年 |
| 5 | **已有第一次验证** | 至少一个数据点证明故事不是PPT | Q1 AI收入+49%，不是画饼 |

**与8大原型的关系**:

叙事验证周期是**选股入口**，不是独立原型。找到后按基本面分类:
- 验证早期(1-2次验证) → 归入Archetype 7(投机) → 小仓位
- 验证中期(3-4次验证，趋势确立) → 升级为Archetype 1(动量) → 正常仓位
- 验证后期(业绩持续兑现) → 可能成为Archetype 5(成长复利) → 核心仓位

**入场时机**:
- ★★★ 确切性押注公告后1-2周(市场还没完全反应) — 最佳但最难抓到
- ★★ 第一次验证(首个财报beat)后的回调 — 实操中最常见的好入场点
- ★ 第二次验证前的横盘期 — 如果故事逻辑成立，市场在等下一个数据点时是买入窗口
- ✗ 已经连涨3个月+/涨幅>100% — 太晚，risk/reward恶化

**退出信号**:
- 验证失败: 某个milestone不达预期(如T-Mobile试验取消) → 立即减仓
- 巨头撤出: 押注方减持或战略调整 → 立即清仓
- 故事兑现完毕: 所有可预见的验证节点都已通过 → 按Archetype对应规则退出
- 叙事被替代: 更新的技术/竞争者出现使原故事失效 → 评估后决定

**每周扫描方法**:
1. 监控巨头动向: NVIDIA/Apple/Google/Microsoft/Amazon的投资/收购/战略合作公告
2. 筛选条件: 被投公司6个月+横盘 + 投资规模≥$5亿 + 涉及全新技术方向
3. 加入观察池，等待第一次业绩验证
4. 验证出现+放量突破 → 按Archetype规则入场

**已知局限**:
- 巨头押注成功率约30-40%，不是高胜率策略，需要配合Archetype止损保护
- "长期横盘"往往说明公司基本面确实不行，巨头押注不一定能扭转
- 5个信号是从Nokia单一案例逆向工程的，存在overfitting风险

#### 来源B: 财报动量 (Earnings Momentum / PEAD)

连续2+季度beat + guidance上调 → 市场持续低估盈利增速 → Archetype 3候选
扫描方式: 每个earnings season后筛选beat率>10%且guidance up的公司

#### 来源C: 行业供需拐点

供给侧出现物理约束(产能不足/交期拉长/库存见底) → 定价权转移 → Archetype 6候选
扫描方式: 跟踪行业数据(库存/产能利用率/交期)，拐点出现时找最紧节点公司

#### 来源D: 事件日历

已知催化剂(FDA审批/财报/政策发布)在30天内 → Archetype 4候选

### 5.1 入场决策树

```
想买一只票 →
  Step 1: 分类 — 属于哪个Archetype? (不分类不买)
  Step 2: Book检查 — 对应Book还有预算吗?
  Step 3: 相关性 — 和现有持仓是不是在赌同一件事?
  Step 4: R/R — 上行空间/下行风险 > 2:1?
  Step 5: Thesis — 能用1句话说清为什么买? (说不清不买)
  Step 6: 止损预设 — 止损在哪? dollar loss是多少?
  Step 7: ATR sizing — 算出该买几股
  Step 8: 入场时机 — 用对应Archetype的入场规则
```

**三行检查（计划外买入必须过）**:
1. 买的理由不是"涨了"
2. 跌20%割不割？（不割就别买）
3. 今天才知道它的 → 不买

### 5.2 退出决策树

```
持仓股 →
  Thesis还在吗?
  ├─ NO → 全清。不管盈亏
  └─ YES →
      价格触发止损了?
      ├─ YES + Momentum/Event/Speculative → 执行止损
      ├─ YES + Value/Compounder → 检查基本面，价格跌≠止损
      └─ NO →
          到目标价了?
          ├─ YES + Momentum → Trailing stop跟踪，不主动卖
          ├─ YES + Value → 到内在价值就减仓
          └─ NO → 继续持有
```

## Part 7: 战术机制速查

### 7.1 新高回调机制
- 52周新高后1年: 胜率81%，回报13.7%（George & Hwang 2004）
- 新高是**正向预测信号**，不是见顶信号
- 3-5%回调 = 最佳买入窗口（量缩+守均线确认）
- 15%→20%回调升级概率73%——这是最危险的阈值
- **强牛市追突破，震荡市等回调**

### 7.2 动量崩溃保护
- Crash不在熊市中，而在**熊市结束后1-3个月的强反弹期**
- 双触发暂停: 大盘跌>25% + 近1月反弹>8% → 暂停动量策略
- 只做多头腿，不做空past losers（空头腿是crash主因）
- 动态杠杆: 高波动时减仓，可将Sharpe提升~1倍

### 7.3 均值回归 vs 趋势延续
- **短期(1-4周)**: 均值回归有效
- **中期(3-12月)**: 趋势延续有效
- **长期(3-5年)**: 均值回归有效
- 大盘低波动股 → 均值回归 / 小盘高波动股 → 趋势延续
- **用错策略的代价不对称**: 对趋势股做均值回归是高危操作

### 7.4 板块轮动
- 单轮板块领先平均持续2-2.5年
- 2024: 通信 → 2025: 科技/国防 → 2026: 能源(+22%)/工业(+16%)
- 领先指标: PMI方向 + 10Y-2Y利差 + ETF资金流（2/3共振才确认）
- 同时持2-3个板块，不超过3个

### 7.5 入场技术
- Pullback entry胜率更高，Breakout entry赔率更大
- 最优: "二次入场"（突破后回踩确认）
- 开盘30分: 消息驱动/突破型 / 尾盘: 机构再平衡 / 盘中12-14点避免
- 盘后: 仅Earnings beat后early entry值得（avg +8.2%），其他等正常时段
- IBKR用Adaptive Limit (Patient模式)争取价格改善

### 7.6 日历效应
- **OPEX周偏多**，但到期当日系统性为负 → 周前2-3天建多，到期日前离场
- Sell in May: 统计存在但不可直接套利 → 5-10月降仓+防御
- 星期效应: 避开周一建仓，周二/三优先
- January Effect: 基本失效，不值得单独押注

### 7.7 期权增强
| 原型 | 适合的期权策略 | 不适合的 |
|------|--------------|---------|
| Momentum | LEAPS(杠杆), Protective Put | Covered Call(截断upside) |
| Value | Cash-Secured Put(建仓), Covered Call | LEAPS |
| Event | 买Call/Put(定义风险) | 卖期权 |
| Dividend | Covered Call(增强收益) | LEAPS |

- $8.8K账户优先LEAPS积累，不做收入策略
- Covered Call在动量股上年化截断2/3收益（Nasdaq-100 BuyWrite 10年149% vs 指数512%）

## Part 8: 行为防护（按原型分类）

### 每种策略的特有陷阱

| 原型 | #1陷阱 | #2陷阱 | 预防规则 |
|------|--------|--------|---------|
| Momentum | FOMO追顶 | 不止损("趋势还在") | 止损写入before买入 |
| Deep Value | 越跌越买(锚定) | 不认错 | 每季基本面审查 |
| PEAD | 用consensus而非whisper | 持仓超60天 | 15-30天主动复评 |
| Event | 信息优势幻觉 | 事件失败还持有 | 催化剂失败=当日出 |
| Compounder | 太早获利了结 | 涨太多就卖 | 2.5×复查不等于卖 |
| Cyclical | PE低=便宜(错!) | 顶部"这次不一样" | 出现3个顶部信号就走 |
| Speculative | 报复性交易(HSAI) | R1赚了加码R2(OKLO) | 仓位≤3%/不加仓 |
| Dividend | 高yield trap | 利率风险低估 | FCF覆盖率>1x |

### 通用行为铁律

1. **Prospect Theory在不同策略中的不同表现**: 赢时风险规避/亏时风险偏好是人性。但在momentum中表现为"不止损"，在value中表现为"越跌越买"——同一偏差，不同后果
2. **事前规则化 >> 事后意志力**: 亏损状态下理性判断已受损，靠意志力纠正无效
3. **连亏后复仇交易成功率仅~20%**: 暂停是数学最优策略
4. **策略混淆是最高频失败模式**: momentum的票用value逻辑持有 = 灾难
