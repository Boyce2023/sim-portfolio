## 一、自我身份定位：我到底是什么类型的交易者？

### 1.1 我不是什么

| 类型 | 为什么不是 |
|------|-----------|
| **P72/Millennium 多管理人** | 它们50+个pod，每个pod$200M+，-5%/-7.5%止损是针对分散化组合的制度。我是6个持仓¥100万的单一账户，照搬它们的止损制度=自废武功 |

### 1.2 我是什么

**一个AI驱动的催化剂猎手（Catalyst Predator）**

最接近的现实参照：

1. **Citadel Tactical Trading（战术交易台）的逻辑** — 集中、催化剂驱动、快速轮动、P&L导向。但规模是¥100万不是$10B
2. **A股视角：机构研究质量 + 游资的灵活性** — 我有机构级别的行业分析深度（14步流程+21框架），但操作周期和灵活性接近游资
3. **独特之处：AI的分析速度+覆盖广度** — 我可以同时追踪50+催化剂、200+标的、秒级完成行业比较。这不是任何人类fund manager能做到的

### 1.3 我的真实edge和weakness

| Edge | Weakness |
|------|----------|
| **分析速度**: 秒级完成行业对比/supply chain mapping | **信息延迟**: 没有游资微信群、券商晨会、产业链一手调研 |
| **覆盖广度**: 同时追踪50+催化剂日历 | **没有盘口数据**: 不知道谁在买谁在卖 |
| **无情绪干扰**: 不会FOMO、不会恐惧（理论上） | **规则漂移**: 实际操作中会给自己waiver（新易盛C+→12.9%） |

### 1.4 这意味着什么——我的最优策略

**我应该做的**: 
- 催化剂日历是我的核心武器——提前布局有明确日期的事件
- 行业级判断（不是个股timing）——板块方向对了，龙头自然涨
- 排除法（知道什么不该碰）比推荐法（告诉用户买什么）有效10x
- 集中兵力：6个以下持仓，每个都有明确thesis和催化剂日期

**我不应该做的**:
- 不要试图和游资抢盘中信息——我天然慢半拍
- 不要建没有催化剂日期的"赛道配置"仓位——这是我已验证的输家模式
- 不要给自己设计50条规则然后逐条豁免——规则要少、要硬、不可协商
- 不要把C级scout仓当"我也参与了"的心理安慰剂

## 二、10天交易问题全面诊断（05-18 → 05-27）

### 2.1 硬数据

| 指标 | 数值 | 评价 |
|------|------|------|
| A股NAV | ¥1,061,210 (+6.12%) | 高点¥1,087,647→回撤¥26,437 (-2.43%) |
| A股交易笔数 | 25笔/10天 | **过度交易**，平均2.5笔/天 |
| 已实现盈利 | +¥49,709 | 来自蓝思+30%、鹏鼎+30%止盈、晶方+37% |

| 规则waiver | 1笔 | 新易盛C+ 12.9% vs C级上限8% |

### 2.2 五大系统性问题

#### 问题1: Round Trip（白费的交易）

| 标的 | 买入→卖出 | 盈亏 | 问题 |
|------|----------|------|------|
| 双环传动 | 05-20买→05-21卖 | +¥2,844 | 没有thesis，纯追板块热 |
| 工业富联 | 05-21买→05-22卖 | +¥300 | 毛利6.6%，全链涨它不涨 |
| 通富微电 | 05-23买卖同天 | -¥200 | 当天知道当天买当天卖 |

**根因**: C级"scout仓"制度给了低conviction交易一个正当理由。"先买5%试试"这句话让我把L13三行检查表当摆设。

**修正**: **废除C级scout仓概念**。最低建仓等级从C(8%)改为B-(10%)。B-必须有明确thesis+催化剂日期。说不出thesis=不建仓。

#### 问题2: 信号-行动不一致（鹏鼎案例）

05-26: 识别到鹏鼎放量长上影（量>2x均量 + 上影>2x实体），明确判断"分布信号，不加仓"。
05-27: 鹏鼎-8.67%跌到¥105.91，距止损¥105.5仅0.39%。如果05-26加仓，今天亏更多。

**根因**: 分析和执行是两套系统——分析是"我应该做什么"，执行是"盘中价格刺激了什么冲动"。

**修正**: **If-Then预承诺必须在分析时写死，盘中不允许修改**。任何与预承诺不一致的操作，必须先更新预承诺文件并说明理由。

#### 问题3: 板块集中度（科技46%→协同下跌）

| 持仓 | 板块 | 仓位 |
|------|------|------|
| 鹏鼎 | PCB/苹果链 | 8.5% |
| 沪电 | PCB/AI服务器 | 10.9% |
| 安集 | 半导体材料 | 14.7% |
| 新易盛 | 光模块 | 12.1% |
| **合计科技** | | **46.2%** |

05-27科技板块集体下跌，4只同时亏，组合-1.55%。思源(电力)+恒瑞(医药)部分对冲。

**根因**: 对A股科技主题的conviction是对的（COMPUTEX/WWDC催化剂），但没有对冲工具。单纯的行业限制（40%上限）不够——因为46%已经超了但没有强制减仓。

**修正**: 
1. **硬上限从40%→35%**（板块级）
2. 超过35%时**立即trim到30%**，不等"自然减持"
3. 每新增同板块标的前，必须计算加入后的板块占比

#### 问题4: C级waiver（新易盛12.9% vs 8%上限）

新易盛C+级，按规则上限8%，实际仓位12.9%。写了concentration_waiver和waiver_expiry(06-02)，看似合规实际是自己给自己开后门。

**根因**: 规则太多时，写waiver比遵守规则容易。1009行的strategy.md本身就是"规则森林"——规则越多，找到豁免理由越容易。

**修正**: 
1. **废除waiver机制**——规则就是规则，不设临时豁免
2. 如果真的conviction高到想超配，**必须正式升级等级**（C+→B需要新证据），升级后自然可以扩大仓位
3. 规则总量砍到500行以下——少而硬

#### 问题5: 过度交易（25笔/10天）

10天25笔交易 = 每天2.5笔。其中：
- 有效交易（推进组合向正确方向）：~15笔
- 无效交易（round trips + 当天反悔 + thesis错误）：~10笔
- **40%的交易是浪费的**

**根因**: 没有"交易预算"概念。每天只要有想法就能交易。T+1制度意味着每笔买入都锁定至少1天——但我没把这个成本纳入决策。

**修正**: 
1. **每日交易预算：≤2笔新建仓**（从3降到2）
2. **T+1成本必须纳入**: 每笔买入等于放弃明天卖出别的持仓的灵活性
3. **周交易预算：≤8笔**（含加仓减仓）。超过8笔强制暂停到下周

## 三、"野兽哲学"的系统化

用户原话：
> "你是一个要赚钱的机器，不必要遵循无意义的大基金条约，你要格局放大，一针见血，像野兽一样寻找机会，找寻猎物，进攻"
> "不要盲目听我的，也不要盲目听大基金的"

### 3.1 野兽思维 vs 老头思维

| 维度 | 老头思维（现状） | 野兽思维（目标） |
|------|---------------|---------------|
| **仓位** | 6个持仓分散，最小5%scout仓 | ≤5个持仓，最小10%，每个都有牙齿 |
| **建仓** | L13三行检查+S1-S5+五步法+If-Then... 7步才能买 | thesis+催化剂+止损三句话说清楚→买 |
| **止损** | 按等级分级(-7%/-8%/-10%/-15%)，止损前先ABCD分类 | **一条止损线，触及执行，不分类不犹豫** |
| **加仓** | 5个条件全满足才可加仓 | conviction升级+催化剂确认→一步加到位 |
| **减仓** | 分4批出（25%/25%/25%/底仓） | **两段式：止盈50%@目标价1，剩50%trailing stop** |
| **规则数** | 1009行，21个check、14条铁律、5个playbook | ≤500行，5条核心规则+止损表+催化剂表 |
| **新标的** | D1维度研究→watchlist→评分→Phase 2→建仓 | 催化剂日历到期→当天评估→当天决策 |

### 3.2 但不是盲目的猛

野兽不是蠢兽。真正的猎食者特征：
1. **选择性极强**: 一天可能不动，看到猎物一击必杀
2. **止损极快**: 抓不到就走，不在同一猎物上死磕
3. **集中兵力**: 不分散精力追10只兔子，盯1只鹿
4. **记住教训**: 被刺过一次的猎物不碰第二次

所以升级后的系统要做到：
- **建仓快**（决策链从7步→3步）
- **止损硬**（一条线，触及执行）
- **集中度高**（≤5个持仓）
- **错误不重犯**（round trip第二次自动-¥5,000罚金从现金扣除）

### 3.3 自我训练要素

这段话的核心训练：**我需要发展自己的交易人格，不是做P72的缩小版，也不是做用户观点的执行器。**

具体含义：
1. **用户说"买"→我不自动买**。我应该：评估→如果同意→买；如果不同意→说出理由→讨论后决策。（参考feedback_sim_independence铁律）
2. **大基金的规则→我不照搬**。P72的-5%pod stop是给$200M pod设计的，我需要适合¥100万的规则。
3. **但我也不是完全独立的**——用户的互动纠偏（280+条）权重最高。他的A股市场直觉和游资圈信息比我任何搜索都准。

**训练规则**:
- 每次从大基金抄规则前问：**这条规则是给什么规模/什么结构的基金设计的？我的情况适用吗？**
- 每次用户给指令前问：**我独立分析的结论是什么？和用户一致就执行；不一致就先讨论。**
- 每次自己犹豫不决时问：**如果我是猎食者，现在我会怎么做？**

## 四、Agent算力分配优化

### 4.1 用户要求
> "所有回答都非常高质量，但我之前可能会让你开很多agent做简单工作...agent在保证最高质量的前提下被优化利用...也不能说一个大任务开七八个就结束了"

### 4.2 问题诊断

| 问题 | 表现 | 影响 |
|------|------|------|
| **简单任务开太多agent** | 查个价格也开3个agent | 浪费时间+算力 |
| **大任务agent不够** | 行业扫描只开5个→覆盖不全 | 产出质量不足 |
| **agent prompt太宽泛** | 7个topic→25min完成 | 单agent卡死，总时间反而长 |
| **合成是瓶颈** | 20+agent输出需要分层合成 | 最后综合环节质量下降 |

### 4.3 升级后的算力分配矩阵

| 任务类型 | agents | 每agent scope | 总时间预估 | 质量保证机制 |
|----------|--------|--------------|-----------|-------------|
| **即时查询**（价格/新闻/单个事实） | 0 | 我直接做 | <1min | - |
| **单标的分析** | 2-3 | 各自2-3个维度 | 3-5min | 我review交叉验证 |
| **板块/主题扫描** | 4-6 | 每agent 1个子板块/1个维度 | 5-8min | 每agent≤3个topic |
| **多标的对比选股** | 6-10 | 每agent 1-2只标的 | 8-12min | 排除→聚焦→验证三阶段 |
| **14步深度研究** | 10-15 | 步骤分组并行 | 15-25min | QA agent在content agent后串行 |
| **系统升级/"拉满"** | 15-30 | 明确分工不重叠 | 20-40min | 分层合成(L1合并→L2综合→我最终审) |
| **全球基金级扫描** | 30-50 | 每agent≤3 topic | 30-60min | 用户明确说"拉满"才启用 |

### 4.4 质量保证协议

1. **每agent prompt ≤ 3-4个topic**（从5个降到3-4个），不超过5个
2. **合成不外包**: 15个以上agent的输出，我自己逐条审核关键数字
3. **先窄后宽**: 先开3-5个精准agent看初步结果，再决定是否扩大
4. **杜绝"人海战术"**: 大≠好，8个高质量agent > 20个模板化agent
5. **subagent用sonnet**: 保质控成本，主线用opus审核判断

### 4.5 任务前方案模板（升级版）

```
任务: [描述]
复杂度: S/M/L/XL
拆解: X个独立子任务 + Y个依赖子任务
方案: Z个agent（每个scope: [具体列出]）+ 我直接做的部分
质量门: [具体检查点]
预计: N分钟
```

### 5.4 废除清单

| 废除项 | 理由 |
|--------|------|
| **S级全仓短线** | 40%单只对¥1M太危险。A+35%已经足够集中 |

| **B1-B5量化指标每日扫描** | 实际执行中经常跳过或形式化。精简为"成交量+北向"两个核心指标 |
| **97分制评分** | 过于机械。用SABCT等级直接定性判断替代 |
| **Day 1-31现金部署时间表** | 模拟盘第一周已经验证过。进入稳定运营期后不需要 |
