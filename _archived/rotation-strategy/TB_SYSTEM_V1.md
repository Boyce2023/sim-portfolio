## §0 定位与核心假设

### §0.1 Track B身份

Track B是独立于Track A基本面系统的第二套操作逻辑，核心是**板块轮动 + 动量交易**。两套系统共享资金池，但评级框架、入场逻辑、止损锚点完全不同。

| 维度 | Track A（基本面） | Track B（轮动/动量） |
|------|-----------------|---------------------|
| 核心逻辑 | 公司质量 + 催化剂确定性 | 板块热度 + 资金搬家 + 小市值弹性 |
| 止损锚 | 自身ATR + thesis | 板块龙头跌>5% = 全出 |
| 评级上限 | S级（50%） | **B+级（15%）— 硬顶不可突破** |
| 持有周期 | 中长期（催化剂兑现周期） | 短期（1-3周） |
| 升级路径 | 催化剂兑现→升级 | **不可升级为Track A** |
| 记忆文件 | pain_memory.md / victory_memory.md | **tb_pain_memory.md / tb_victory_memory.md** |
| 复盘文件 | playbook.json / scorecard.json | **tb_playbook.json / tb_scorecard.json**（各滚动保留10条） |

### §0.2 核心假设（系统成立的前提条件）

Track B系统在以下条件下有效：

1. **市场存在清晰主线**：有明确龙头主导的板块，且资金在同板块内有序传导
2. **板块内资本轮动有序**：从龙头→次龙头→中小盘逐步扩散，传导链可识别（L1→L6）
3. **情绪周期可观测**：涨停家数/炸板率是可靠的情绪温度计

Track B**已知失效场景**（约占全年40%+的交易日）：

| 失效场景 | 具体信号 | 系统响应 |
|---------|---------|---------|
| 震荡市 | 涨停30-60家，方向不明，主线切换频繁 | 仅做α级催化剂，不做情绪追随 |
| 熊市/呼气市 | F20呼气/深度呼气 | 系统关闭，TB仓位归零 |
| 无主线市 | 无单一板块主导，多板块均匀分布 | 等待，不做Track B |
| 崩溃期 | 跌停>50家 / 炸板率>60% | 紧急清仓，系统暂停 |

### §0.3 评级体系概述

Track B使用独立5维100分评级系统：

- **D1 入场信号等级（30分）**：机构/游资/外资共振强度
- **D2 轮动类型质量（25分）**：Type 1-5轮动类型打分
- **D3 产业链传导位置（20分）**：L1-L6传导链位置
- **D4 时间位置（15分）**：板块生命周期阶段
- **D5 目标流动性（10分）**：标的流动性是否达标

**等级映射**：B+（≥75分）/ B（60-74分）/ B-（45-59分）/ C（<45分或一票否决）

**系统定位总结**：Track B是机会型系统，不是全天候系统。主动运作窗口约占全年15-30%交易日。在呼气市和无主线市休息是正确的选择，不是遗憾。

## §1 市场门 + 硬开关

### §1.1 F20市场呼吸（主控开关）

F20是Track B的**一级门控**。每周一更新状态，日内更新条件见§1.3。

| F20状态 | 判断标准 | TB操作 | 仓位上限 |
|--------|---------|--------|---------|
| **强吸气** | 涨停≥80家 + 炸板率<20% + 北向/成交持续放量 | 全力运作，积极建仓 | ≤40% |
| **吸气** | 涨停60-80家 + 炸板率<30% + 成交>8000亿 | 正常运作，建仓 | ≤30% |
| **中性** | 涨停30-60家 / 炸板率30-50% / 方向不明 | 仅做α级催化剂（S级信号），不做情绪追随 | ≤15% |
| **呼气** | 涨停<30家 + 炸板率>50% / 北向持续净流出≥3日 | **零新仓**，评估存量清仓 | 0% |
| **深度呼气** | 涨停<30家 + 成交<5000亿 / 跌停>30家 | **TB仓位归零**，仅TB-S4监控持仓安全 | 0% |

★★★ **深度呼气 = TB仓位0%。不是减仓，是清仓。**

**F20五状态说明**：
- 呼气期 = 零新仓，不是轻仓（历史教训：Agent-14呼气期强行交易=全年唯一亏损期）
- 深度呼气期 = 评估存量持仓是否触发止损线，主动管理出场节奏
- 中性期不等于暂停，α级催化剂（COMPUTEX/财报日/重磅政策首发）仍可操作，sizing ×0.8

### §1.2 硬开关（任一触发=当日停止全部新入场）

**原版4条**：

| 条件 | 操作 |
|------|------|
| 当日大盘跌≥1.5%（A类下跌） | NO-GO；例外：Type 1事件冲击型受益方与受损方明确分化时豁免 |
| 当日跌停家数 > 50家 | NO-GO + 立即触发全仓TB清仓审查 |
| F20判定呼气/深度呼气 | NO-GO（见§1.1） |
| 板块主龙头跌>5% | NO-GO + 当日所有同板块TB仓位全出（铁律，不分析） |

**V1.2新增紧急关闭（直接跳过F20判断）**：

| 条件 | 操作 |
|------|------|
| 当日涨停家数48小时降幅>40% | TB进入仅监控模式，停止新入场 |
| 批量澄清公告 ≥ 5家同日发布 | 当日全出所有TB仓位（不等5%铁律触发） |

### §1.3 F20日内快速更新协议

★★★ **F20不只是每周一更新。以下条件触发日内临时更新**：

| 触发条件 | 操作 |
|---------|------|
| 炸板率或涨停家数同向变化≥20%，且**连续2个交易日**均如此 | 触发F20临时更新，当日生效 |
| 日内涨停家数较开盘后30分钟下降超过30% | 临时记录，收盘确认是否更新F20 |

**更新格式**：F20临时状态（日期：XXXX）→ 原状态 → 新状态 → 触发原因一句话

**炸板率读取时间规范**：
- **14:00读数**：用于当日GO/NO-GO决策（代表市场情绪稳定状态）
- **收盘读数**：用于Layer 2止损触发判断（更新次日If-Then预承诺）

### §1.4 F20 vs 实时数据冲突解决规则

★★★ **实时数据优先于滞后的F20判断**：

| 冲突情景 | 解决规则 |
|---------|---------|
| 涨停<30家**连续3个交易日**，但F20仍显示吸气 | **强制覆盖为呼气**，不等下周一F20更新。实时数据>滞后指标 |
| 涨停30-60家（模糊区间），F20与当日数据方向不一致 | **以F20为准**（模糊区间F20作为仲裁） |
| F20=强吸气，但当日炸板率突破60% | 触发硬开关（§1.2），不等F20更新 |

**规则总结**：F20只在涨停30-60家的模糊区间作为主要判断依据；涨停<30家连续3日时，实时数据直接覆盖F20。

### §1.5 主线质量指数（TB入场前置条件）

★★★ **主线质量指数 = 主线主题板块成交额 ÷ 全市场成交额**

| 读数 | 含义 | TB操作 |
|------|------|--------|
| >25% | 主线清晰，有资格传导扫描（TB-S6） | 正常运作 |
| 15-25% | 主线偏弱，轮动扩散可信度低 | 仅做L1龙头，不做L2+传导链 |
| <15% | 无主线 / 多板块均匀分布 | **等待模式，不发PRIORITY信号，不建TB** |

**主线质量指数 <25% = TB-S6传导链扫描进入等待模式。不发出PRIORITY建仓信号。**

**计算口径**：以主线主题板块申万二级行业成交额/全市场A股成交额（非申万一级，避免混入非主线子行业成交）。

### §1.6 涨停家数阶段判断表（综合§1.1-§1.5）

| 全市场涨停家数 | F20状态 | TB操作 |
|------------|--------|--------|
| ≥80家 | 强吸气 | 全力，≤40%总仓 |
| 60-79家 | 吸气 | 积极，≤30%总仓 |
| 30-59家 | 中性 | 只做α催化剂 |
| <30家（非连续） | 中性偏呼气 | 谨慎，准备减仓 |
| <30家（连续3日） | **强制覆盖为呼气** | 零新仓 |
| <30家 + 成交<5000亿 | 深度呼气 | **TB仓位0%** |

| Type 1 蹭热点 / 无业务关联 | 无效 | 无一手来源验证业务关联 | 0 — **一票否决** |

★★★ **T2硬标准（v1.2严格版）**：Type 2评分（25分）必须有一手定量数据支撑：

### §2.2 等级判定与仓位

| 等级 | 总分 | 仓位上限 | ATR K值 | 最大硬止损 |
|------|------|---------|---------|-----------|
| **B+** | ≥75分 | **15%** | 2.0 | -12% |

**强制降档（无论总分）**：
- 游资主导（无机构跟进）→ 压至B-（≤10%）

- Type 5游资坐庄 → 最高B-，标准上限≤10%；HP-002（特殊/信息量低）→ ≤5%

### §2.3 一票否决清单（直接C，不计算总分）

以下任一条件触发 → 当日/当笔直接清仓或拒绝入场：

```
□ 维度一：无有效入场信号（D1=0）
□ Type 1蹭热点（无业务验证）
□ L6末端层（首次出现）
□ 北交所超小盘（<15亿）
□ 退潮期（炸板率>50%+涨停<30家）
□ 炸板率>50% → 降级流程（先走§2.4降级）
□ 炸板率>60% → 直接进入一票否决，不经降级流程，当日清仓
```

★★★ **炸板率>60% = 跳过降级，直接一票否决清仓（崩溃期铁律）**

### §2.4 动态升降级规则

**升级条件（B-→B→B+）**：

| 触发事件 | 升级幅度 | 加仓动作 |
|---------|---------|---------|
| 持有第2日龙头继续涨停+全市场≥60家 | +1档 | 加仓至新等级上限 |
| T1→T2确认（D+3至D+7出现具体一手数字） | 重新评分，通常+1到+2档 | 按新等级加仓 |
| S级信号追加确认（三方共振） | 直接B+上限 | 加至15% |

**降级条件（B+→B→B-→C）**：

| 触发事件 | 降级幅度 | 减仓动作 |
|---------|---------|---------|
| 龙头次日高开低走（未封板） | -1档 | 减仓至新等级上限 |
| 板块拥挤度>40%（D2 T1/T3/T4/T5） | -1档 | 减仓至新等级上限×50% |
| 炸板率升至30-50%（分歧期开始） | -1档 | 减仓 |
| 持仓超过类型天数上限无新催化 | 降至C | 清仓 |
| 龙头断板 | 直接C | 清仓（铁律） |
| 龙头跌>5% | 直接C | **所有Track B当日全出，无例外** |
| 炸板率>60% | 直接C（一票否决） | **跳过降级，当日清仓** |

### §2.5 T2连续验证协议

T2评分基于一手定量数据，**不是建仓时一次性核实**：

- **D+1**：复核T2的一手数字来源是否仍有效（无更新=维持T2）
- **D+2**：再次确认，有无新信息修正数字
- **D+3**：如仍无新一手数字补充，且原数字无公司/机构二次确认 → **自动降回T3，调整时间止损至T3上限（5日）**

### §2.6 拥挤度指标计算标准化

**计算公式**：主线主题板块成交额 ÷ 全市场成交额（非申万一级）

| 读数 | 含义 |
|------|------|
| >40% | 拥挤，T1/T3/T4/T5触发降级1档 |
| 25-40% | 正常集中 |
| <25% | 分散/主线不清晰（触发§1.5等待模式） |

### §2.7 30秒速查表（GO/NO-GO决策）

```
Step 1（5秒）：一票否决先过

Step 2（15秒）：5维打分
  D1: S=30 / A=22 / B=14 / C=6 / 无=0
  D2: T2=25 / T4=20 / T1核心=15 / T3=10 / T5=5 / 蹭热点=0
  D3: L1=20 / L2=16 / L3=10 / L4-5=6 / L6独立=6 / L6首次=0
  D4: 启动=15 / 主升早=12 / 主升中晚=6 / 高潮分歧=2 / 退潮=0
  D5: 主板≥10 / 北交=8 / 小北交=4 / 超小=0；萎缩-5

Step 3（5秒）：强制降档
  游资主导? → 压至B-（≤10%）
  北交所? → ≤5%

Step 4（5秒）：等级判定
  ≥75 → B+（15%）
  60-74 → B（12%）
  45-59 → B-（10%）
  <45 → C
```

★★★ **速查表操作时间：30秒GO/NO-GO判断 + 会话内完成完整评级（不再使用"90秒/3分钟"说法）**

## §3 建仓五步法

> Track A五步法 = thesis验证。Track B五步法 = **时机抢占**。

### Step 1：市场门（GO/NO-GO）

**核心检查（09:45-10:00涨停家数稳定后）**：

| 条件 | GO / NO-GO |
|------|-----------|
| 全市场涨停≥60家 | GO |
| 全市场涨停30-59家 | 仅α级催化剂（S级信号）可做 |
| 全市场涨停<30家 | NO-GO，零新仓 |
| 当日大盘跌≥1.5%（A类下跌） | NO-GO（例外：Type 1直接受益方分化时豁免） |
| 跌停家数>50家 | NO-GO + 触发全仓审查 |
| F20=呼气/深度呼气 | NO-GO |
| 硬开关任一触发（§1.2） | NO-GO |
| 主线质量指数<25% | NO-GO（不发建仓信号，仅L1龙头等待） |

**Portfolio Heat检查**：
- Heat >13%：缩减新TB仓位sizing（按等级硬顶×0.7）
- Heat >15%：**暂停所有新TB入场**，先降Heat再建仓

**α级催化剂预确认入场协议（v1.2新增）**：
- 背景：TB-S4识别的α级催化剂（如COMPUTEX/政治局政策首发）在9:30-10:00之间，60家涨停确认前可提前布局
- 适用条件：①前日龙头已封涨停 ②催化剂为机构或官方一手来源 ③建仓规模≤等级上限×50%
- 10:00后确认不满足60家 → 当日出场，不继续持有

### Step 2：龙头门（GO/NO-GO）

**核心逻辑：标的再好，龙头不行=不入场。**

| 龙头信号 | 判断 | 操作 |
|---------|------|------|
| 龙头≥2连板 + 当日仍在涨停 | 传导有效，最优窗口 | GO，T+0至T+1入场 |
| 龙头≥2连板 + 当日高开回落未封板 | 传导弱 | 等明日确认，今日不追 |
| 龙头今日首板涨停 | 传导刚启动 | 布局T+1窗口，今日不追 |
| 龙头断板（昨封今开板） | 退出信号 | NO-GO + 清仓同板块TB仓位 |
| 龙头跌>5% | 行情结束 | NO-GO + **全部TB当日全出（铁律）** |

★★★ **BSE北交所铁律**：北交所龙头跌>3%（非跌停）即触发清仓审查（±30%波幅放大，阈值收紧）

**多龙头区分**：
- 主龙头 = 板块内市值最大+成交额最大（如中际旭创/胜宏科技）
- 主龙头跌>5% → TB全出，铁律不变
- 非主龙头跌>5% → 减仓50%，再检查主龙头是否同向走弱

**催化剂仍在前面检查**：
```
□ 龙头催化剂（会议/财报/发布会）未兑现 → YES = 持续有效
□ 催化剂已兑现 → 高风险，不新建
□ Type 4政策型 → 首发当日起15-75天窗口，方向性政策买入，落地性政策减仓
```

### Step 3：标的门（GO/NO-GO）

**A. 基础资格（全部满足）**：
```
□ 同板块（业务相关，非概念蹭热点）
□ 市值<500亿（主板）或30-100亿（北交所）
□ 今日涨幅<15%（追>15%溢价极高）
```

**B. 轮动类型验证**：

| 类型 | 必须满足 |
|------|---------|
| Type 1 事件冲击 | 一手来源验证直接业务关联（URL/公告/SEC，非传闻） |
| Type 2 产业催化 | 具体数字（BOM/订单/季报数据），定性描述=T3 |
| Type 3 情绪扩散 | 龙头已涨20-50%，此标的今日才出现首板 |
| Type 4 政策驱动 | 方向性政策受益方；落地性政策**不建仓** |
| Type 5 游资坐庄 | 龙虎榜确认游资席位（不超过3板入场） |
| 北交所 | 主板龙头连续2日涨幅>20%或连板≥2板后T+3至T+5 |

**C. 入场时机窗口（按市值分层）**：

| 标的市值 | 最佳入场窗口 | 过了窗口怎么办 |
|---------|-----------|-------------|
| 主板二线（>500亿） | 龙头T+1 | T+2起观察仓≤30%，不满仓 |
| 主板中盘（100-500亿） | T+1至T+2 | T+3不追 |
| 主板小盘（<100亿） | T+2至T+3 | T+4不追 |
| 北交所（30-100亿） | T+3至T+5 | T+6不追（口诀：买在涨停前） |

**D. PlayBook匹配检查（可选，<2分钟）**：
- 打开tb_playbook.json，检查当前情景与历史赢家模式的匹配度
- 匹配评分≥80 → sizing ×1.2，但不超过等级仓位上限
- 未匹配或无记录 → 按标准sizing

### Step 4：Sizing（仓位计算）

**仓位优先级**：等级上限 > 类型上限 > PlayBook乘数（不可超等级上限）

**等级上限**（Strategy.md §2.4b 硬顶）：

| 等级 | 仓位上限 | ATR K值 | 首次建仓 | 加仓条件 |
|------|---------|---------|---------|---------|
| B+ | 15% | 2.0 | 5-8%（分2批） | 龙头次日仍强→补至上限 |
| B | 12% | 2.0 | 4-6%（分2批） | 维持入场逻辑不变 |
| B- | 10% | 1.5 | 3-4%（一次性） | 原则不加仓 |

**类型/特殊情况覆盖**：

| 条件 | 上限 |
|------|------|
| 北交所所有类型 | ≤5%（硬约束） |
| Type 5游资坐庄（标准） | ≤10% |
| Type 5游资坐庄（HP-002特殊/低信息量） | ≤5% |
| 临时子方向（不在L1-L6链） | ≤8% |

**信号等级调节**（在上限内浮动）：

| 信号等级 | 实际sizing |
|---------|-----------|
| S级（三方共振） | 上限×100% |
| A级（机构+游资共振/龙头T+1跟风） | 上限×70-80% |
| B级（残差动量/均线能量） | 上限×50% |

**Portfolio Heat约束**（覆盖以上所有）：
- Heat >13% → 全部新TB仓位sizing ×0.7
- Heat >15% → **暂停新建仓，不执行Step 5**

### Step 5：退出预设（写入才能下单）

★★★ **三个退出触发点必须在建仓前明确写入If-Then预承诺**：

**退出触发1 — 铁律止损（无条件，优先级最高）**：
```
IF 板块主龙头跌>5% THEN 当日市价全出所有同板块TB仓位（不分析，不等自己止损线）
IF 炸板率>60% THEN 当日全出（崩溃期一票否决）
IF 跌停家数>50家 THEN 启动全仓审查，审查结果不通过=清仓
```

**退出触发2 — 时间止损（生命周期强出）**：

| 轮动类型 | 持有天数上限 | 到期操作 |
|---------|-----------|---------|
| Type 1 事件冲击追跟风 | 5日 | 无新催化剂=当日出 |
| Type 2 产业催化 | 跟催化剂日期（BOM/会议） | 催化剂兑现后2-3日出 |
| Type 3 情绪扩散 | 5日 | 强出，不因浮盈延期 |
| Type 4 方向性政策 | 7-15日 | 落地性政策出现=减仓 |
| Type 5 游资坐庄 | 12板 | 过12板=风险陡升，强出 |
| 北交所任意类型 | T+7（从主板龙头启动算起） | 强出 |

**退出触发3 — 价格止损（ATR辅助参考线）**：
```
ATR止损参考 = 入场价 - K × ATR(14)
  B+/B 对应 K=2.0
  B-  对应 K=1.5

注意：Track B的主止损锚是龙头状态（触发1），不是ATR。
      ATR是辅助下限，龙头仍强时不强制执行ATR止损。
```

**If-Then写入格式**：
```
IF [龙头X] 今日跌>5% THEN 当日市价全出 [标的Y] 的Track B仓位
IF 持有超过 [N] 日且无新催化 THEN [标的Y] 当日尾盘出场
IF ATR止损线 [¥Z] 触及 THEN 减仓50%，次日龙头弱确认后全出
```

### §3.1 双轨总仓位约束

★★★ **TB总仓位硬约束（strategy.md §2.4b + v1.2修订）**：

| 状态 | TB总仓位 |
|------|---------|
| 正常运作（吸气/强吸气） | **≤30%**（硬约束） |
| 强吸气+牛市高峰（需用户明确批准） | **≤40%**（绝对上限，不可超越） |
| 中性市 | ≤15% |
| 呼气/深度呼气 | 0% |

**双轨总持仓上限**：总持仓数量≤10只（Track A≤8只 + Track B≤3只，合计≤10只）

**TB总仓位≤30%不是软建议，是硬约束。** 40%绝对上限仅在强吸气牛市高峰期，且需用户明确批准，非自动触发。

### §3.2 五类型分批建仓路径

**路径一：Type 1 事件冲击型（速度优先，窗口D+0至D+1）**

```
Day 0 尾盘（事件当天）：
  龙头已封涨停+成交爆量 → 一次性建70%仓位
  龙头高开未封板 → 建30%观察仓

Day 1（T+1）：
  龙头≥2连板+涨停家数>60家 → 补至100%（等级上限内）
  龙头开板/分歧 → 清30%观察仓出场

Day 2+：不追加，只做减仓管理
```

**路径二：Type 2 产业催化型（可分批，窗口15-45天）**

```
Day 0-1（催化剂发布后）：建40%探针仓（A/S级信号才做）
Day 2-3（确认传导）：龙头未断+≥3只涨停+成交未萎缩 → 补至80%
Day 5-7（二次加仓）：T1→T2升级确认+仍在主升浪 → 补至100%
每次加仓前重走Step 1-2（市场门+龙头门）
```

**路径三：北交所（最保守，T+5前不满仓）**

```
T+3：建30%仓位（北交所5%上限内的30%=总仓约1.5%）
T+4：确认首板后补至60%
T+5：连板+主板龙头未分化 → 补至100%（最大5%总仓）
T+5后不再加仓
禁止：追T+2连板（次日-30%概率极高）
```

**路径四：Type 3/4/5**

| 类型 | 分批策略 |
|------|---------|
| Type 3 情绪扩散 | 一次性70%+次日确认补满（窗口3-7天，不等） |
| Type 4 方向性政策 | 40%探针 → D+3确认补至80% |
| Type 4 落地性政策 | **不建仓**（见光死，发布当日减仓） |
| Type 5 游资坐庄 | 一次性建仓（≤10%/≤5%），不加仓 |

### §3.3 Track A冲突处理规则

| 场景 | 处理 |
|------|------|
| 同日TA和TB都有入场信号 | Track A优先；TA完成后剩余空间≥15%才建TB |
| 剩余空间<15% | TB本日不建新仓；例外：TB为S级信号且TA为B级→当日TB优先 |
| Track A止损当日，同时有TB信号 | 止损日暂缓TB新建仓；止损原因=公司特定利空（D类）→TB不受影响 |
| TB龙头跌>5%，Track A持有同板块 | TB当日全出；Track A独立评估thesis，不受TB铁律约束 |
| Portfolio Heat>15% | TB暂停建仓，先降Heat再考虑 |
| 同标的既有TA仓位，又有TB信号 | 在TA等级上限内加仓，不另开TB独立头寸 |

### §3.4 TB专用行为铁律（6条）

```
TB-L1 龙头信号优先于标的信号
  标的再好，主龙头断板当日不入场不持有。无例外。

TB-L2 时间止损是刚性约束
  到期无新催化就出，不因"还有浮盈"延期。
  Type 3到5天全出，不磨蹭。

TB-L3 不追入场窗口外的标的
  错过北交所T+3-T+5窗口，不追T+6以后的连板。
  口诀：买在涨停前，卖在连板第一天。

TB-L4 Track B不升级为Track A
  无论涨了多少，基本面看起来多好，不自动升级。
  升级必须经过完整Track A五步法重新评级（独立决策）。

TB-L5 不做逆势Track B
  板块已在回调时不抄底小票。
  "龙头跌完了应该能反弹"不是TB逻辑，是猜测。

TB-L6 炸板率>60%当日全出
  不等5%铁律触发。崩溃期一票否决，不分析不等待。
```

## §4 三层退出系统

> **元规则**：退出信号永远优先于入场信号。同日触发时，退出优先，入场取消，无例外（P0）。

### §4.1 Layer 1 — 铁律止损（无条件执行，不分析，不等）

★★★ **三条铁律**：任一触发 = 当日执行 Track B 所有相关仓位全部出场。

| ID | 铁律名称 | 触发阈值 | 执行动作 |
|----|---------|---------|---------|
| L1-A | **主龙头跌幅** | 主龙头单日跌幅 **>5%** | Track B 全部当日市价出场 |
| L1-B | **监管限制措施** | 任何形式：保证金上调 / 账户暂停 / 临停 | 当日市价全出，不过夜 |
| L1-C | **批量澄清公告** | 盘前≥5家 / 盘中累计≥10家 / 盘后≥5家 / 3日累计≥15家 | 盘前→开盘即出；盘中→尾盘减50%；盘后→次日开盘即出 |

**主龙头定义（唯一权威定义，其他章节引用此处）**：

```
主龙头 = 板块内【市值最大 + 成交额最大】的标的（如中际旭创 / 胜宏科技）

多龙头分级规则（V1.2）：
  主龙头跌 >5%  → Track B 无条件全出（铁律，不分析）
  非主龙头跌 >5% → 减仓 50% + 立即检查以下三条：
    ① 主龙头是否同日走弱？         → 是 = 全出
    ② 是否系统性板块回调（≥3只同日-5%）？→ 是 = 全出
    ③ 仅个别非主龙头补跌？         → 减50%后观察1天

北交所特殊阈值（BSE-THRESHOLD，±30%放大效应）：
  北交所标的：龙头跌 >3% 即触发（非 -5%）
  原因：北交所±30%涨跌停，-5%相当于主板-15%，信号已滞后。
```

**铁律执行纪律**：触发 → 当日执行 → 不分类 → 不犹豫 → 不等反弹。同日不回买。

**冲突处理**：同时触发多个信号 → 执行 P 数字最小的那个；相同优先级 → 执行动作更严格的。

#### 拥挤度规则详解（统一计算口径）

**拥挤度计算标准**：`主线主题板块成交额 ÷ 全市场成交额`（非申万一级行业成交额）

| 阈值 | T1 / T3 / T4 / T5 操作 | T2（产业催化）豁免规则 |
|------|------------------------|----------------------|
| >40% | 开始减仓 | 关注但不减（有 BOM / 季报支撑时） |
| >48% | 减仓 >50% | **减至半仓（非清仓）**，T2 豁免上限到此为止 |
| 修正值 >2.0 | 减仓 >50% | **无豁免**，T2 同等执行 |
| 修正值 >2.4 | 即日清仓 | **无豁免**，T2 即日清仓 |

> T2 豁免来源：2025年2月 TMT 成交占比 >40% 持续 8 天行情不崩（DeepSeek 有产业基本面支撑），过早离场错失主升浪。但修正值极端值（>2.0 / >2.4）无任何豁免。

#### 炸板率规则（读取时点规范）

- **14:00 读数**：作为决策参考（当日盘面判断），不触发铁律（盘中数据不稳定）
- **收盘率**：触发 Layer 2 止损铁律（P4 / P5）的正式读数
- **>50%（收盘确认）**：当日减仓 → P5
- **>60%（收盘确认）**：当日清仓，跳过减仓步骤，直接一票否决 → P4，无例外

#### 龙虎榜信号时效规范（ALL T+1 执行）

★★★ **所有龙虎榜信号（含机构专用席位）统一在 T+1 开盘执行，不提前当日执行。**
原因：龙虎榜数据 T+1 才公布，当日信号基于估算，不可作为铁律触发依据。

#### 多仓位同日出场顺序（流动性优先）

```
同日需要出场多个 Track B 仓位时，按以下顺序执行：

```

### §5.2 TB-S1：涨停生态监控（情绪分级）

### §5.3 TB-S2：板块成交占比变化（资金流向 + 拥挤度）

**拥挤度统一计算口径**：`主线主题板块成交额 ÷ 全市场成交额`（同 §4.2 定义）。

**输入**：各大板块（TMT / 消费 / 周期材料 / 金融地产 / 新能源）当日成交额占比，与 5 日 / 20 日均值比较。

**判定规则**：

| 条件 | 信号 | 行动 |
|------|------|------|
| 单板块 5 日成交占比变化 +5pp 以上 | 资金净流入 | 该板块 INVESTIGATE |
| 单板块 5 日成交占比变化 -5pp 以上 | 资金净流出 | 减持信号 |
| 连续 3 日上升 + 今日 >20 日均值 +8pp | 主线确立 | 直接 PRIORITY |
| 连续 3 日下降 + 今日 <20 日均值 -5pp | 风格切换启动 | INVESTIGATE 新板块 |
| 原主线占比 -3pp + 新板块 +3pp + 新板块有涨停 | 资金搬家 | 触发 TB-S3 验证 |

**TMT 特殊规则**（基于 T2 豁免体系）：
- TMT 占比 >40%：非 T2 → 减仓关注；T2 → 仅关注不减（有 BOM / 季报支撑）
- TMT 占比 >48%：T2 也减至半仓
- 修正值 >2.0 / >2.4：无豁免，执行 §4.2 对应规则

**频率**：每日收盘后（W2）；每周一汇总周度趋势。

### §5.4 TB-S3：龙头状态追踪（持仓安全阀）

**唯一功能**：监控板块主龙头（§4.1 主龙头定义）实时状态，是 Layer 1 铁律的信息前端。呼气期这是唯一保留运行的 Scanner。

**龙头状态分级**：

| 状态 | 判定标准 | Track B 动作 |
|------|---------|------------|
| STRONG | 涨停 + 成交量 <2× 均量（缩量封板）+ 封板 <14:00 | 无需操作 |
| HEALTHY | 涨幅 ≥3% + 成交量正常 + 未断板 | 无需操作 |

| L2 | PCB / 覆铜板 | L1 启动后 1-4 周（实测 2026 年 5 月压缩至 10 天）|
| L3 | 液冷 / 散热 | L2 主升后 2-4 周 |
| L4 | 半导体封装 | L1 高点后 2-4 个月（特殊路径）|
| L5 | 上游材料 CCL / 铜箔 | L2 同步或略晚 0-2 周 |
| L6 | 末端扩散（超级电容 / 元器件）| 主线高位回调当日 |

**L6 分级处理（V1.2）**：
- 首次出现（1 天）→ 退潮预警，主线降仓 50%，L6 本身不建仓
- 连续出现 2-3 天 + 有独立产业催化 → 可能已成独立主线，允许试仓 ≤8%（需 TB-S4 确认）
- 首次出现 + 72 小时内无新催化剂 → 退潮确认，主线全出

**T1→T2 升级监测**（引用 §4.3 权威定义）：T1 事件 D+3 至 D+7 内扫描是否出现具体 BOM / 订单验证 → 有 = 标记 T1→T2 升级，触发 PRIORITY。

**Lag 警告**：Lag 天数仅基于 AI 算力链历史，每个新周期需重新校准。入场触发以实时日内资金流向为准，不依赖历史 Lag 预判。

**频率**：盘中 12:00 / 14:30 各扫一次；收盘后（W2）汇总；每周一（W3）周度回顾。

| TB-P4 | 误判轮动类型 | `tb_type_mismatch` | 按T2逻辑持有，实为T3；超期被套 | T2入场必须有可量化数字（订单/出货/财报）；无数字=默认T3处理 |

**TB-P8 北交所流动性陷阱详细说明**：
- 北交所±30%涨跌幅+流动性低，当龙头发出-5%信号时可能无法以合理价格成交
- T+1制度：龙头今天-5%，BSE仓位今天挂单，但若成交不了，亏损继续累积过夜
- 危险场景：龙头连续-5%两天，BSE标的因流动性差无法执行，亏损翻倍
- **铁律**：★★★ BSE持仓发现退出信号时，第一时间挂单，不等确认，不等价格好转

### §7.2 TBL2 Circuit Breaker（TB熔断器）

**关键区别**：CB只统计"规则违反型亏损"，不统计板块退潮时纪律性出场的亏损。

**规则违反型亏损定义** = 亏损出场 + post-mortem显示违反以下任一规则：
- 8个Pain Pattern之一未遵守
- 入场三步检查未完整过
- 铁律止损触发前有明确退出信号被忽略

**CB状态表**：

| 状态 | 触发条件 | 操作限制 | 恢复条件 |
|------|---------|---------|---------|
| GREEN | 默认 | 正常，全部轮动类型可用 | — |
| YELLOW | 规则违反型亏损≥2笔（7日滚动） | sizing×0.5；新建≤1只/日；禁止Type 3和Type 5新建 | 连续5个交易日规则违反型亏损=0 |
| RED | 规则违反型亏损≥3笔（7日滚动）或单笔亏损>5% | 禁止所有TB新建；仅执行已有持仓止损和退出 | 5日无规则违反亏损→YELLOW；再5日→GREEN |

**额外触发**：
- 同一天触发两条铁律止损 → 直接进入RED
- 同一Pain Pattern 7日内出现≥2次 → 该Pattern对应轮动类型进入"类型级别暂停"

**★★★ tb_review.py自动追踪CB状态**：
- 每次出场后，tb_review.py读trade_log中的`rule_violation`字段
- 7日滚动计数写入`tb_scorecard.json`的`cb_state`字段

### §8.2 Phase 0：建仓前确认（每次入场必须执行）

**★★★ PlayBook检查（Phase 0新增，入场前必做）**：
```
1. 读取 tb_playbook.json，匹配当前setup：
   - 轮动类型相同（Type X）
   - 传导层级相同（Lx）
   - 市场环境相近（F20状态）
2. 匹配 → 显示PlayBook模式，建仓信心+1档（最多+1档），不突破B+硬顶
3. 匹配Anti-Pattern → 警告，要求回答"此次与Anti-Pattern的结构性不同"
4. 无匹配 → 按5维评分正常进行，不影响评级
```

**入场确认清单（Phase 0全部通过才进入Phase 1）**：
```
□ §1.4硬开关：全部4条处于"开启"状态
□ CB状态：GREEN或YELLOW（RED禁止新建）
□ 类型级别暂停：目标类型未被暂停
□ 轮动类型识别：T1/T2/T3/T4/T5已明确

□ 双轨总持仓：<10只（TA+TB合计）
□ TB持仓计数：<3只
```

### §8.3 Phase 1：建仓期 D+0~D+2

**标准建仓节奏**：
- D+0：探针仓，40%目标仓位
- D+1：确认仓，追加60%。条件：D+0未跌>1×ATR(K=2.0) + 龙头未断板
- D+2：最后窗口。未建齐 → 放弃，不追高

**D+0当日可一次性全仓的特殊情况**：
- Type 1事件冲击：事件当日T+0尾盘确认
- Type 2产业催化：BOM/季报超预期当日龙头放量确认
- "全仓"上限仍是等级硬顶（最高B+ = 15%组合仓位）

**类型连续验证（★★★ Phase 1专属监控项）**：

| 验证节点 | 动作 | 无一手数据时 |
|---------|------|------------|
| D+1 | 查找板块龙头公告/行业协会/主力订单数据，确认Type判断 | 标注"待确认" |
| D+2 | 重新确认Type，检查是否有可量化基本面数字 | 降级处理 |
| D+3 | 无一手数据 → **自动降档**：T2→T3，调整持有天数上限（T3的5天上限）| D+3无数据=按T3处理 |

Type 2和Type 3的区别决定持有天数（T2最长45天，T3最长5天）。D+3后无验证数据，按更保守的T3处理，持有天数上限即刻调整。

### §8.4 Phase 2：持有期 + 退出准备

**持有期每日监控（优先级从高到低）**：

| 级别 | 信号 | 动作 |
|------|------|------|
| Level 1（即刻响应） | 龙头跌>5%；监管限制措施 | → Phase 3强制退出 |
| Level 2（次日响应） | 炸板率>50%；板块成交占比>48%；龙头换人 | → 进入退出准备，分批减仓 |
| Level 3（观察） | 持仓天数达上限-2天；龙头成交量较峰值缩减>30% | → 开始准备退出计划 |

**退出信号分级**：
- 退出信号A（≥2个同时）：炸板率30-50%；涨停家数从>60回落至30-60；成交量较峰值缩减>30%；持仓天数达上限-2天；板块成交占比>40%（非T2）
- 退出信号B（单个即刻）：炸板率>50%；拥挤度>48%；龙头断板换人；L6末端首次出现+48h无新催化

**分批退出节奏**（进入退出准备后）：
- 信号当日收盘：减仓50%
- 次日判断：龙头仍未断板→留10%观察；已弱化→全出
- 最迟T+2：无条件全出（退出准备期不超过2天）

**★★★ 最多1次加仓**：Phase 2中禁止新增加仓。仅Phase 1（D+2后）满足全部条件时可加仓1次，不做多次金字塔加仓。

**多头寸退出顺序**（同时持有多个TB头寸时）：
1. **北交所/小盘标的优先退出**（流动性差，先挂先跑）
2. 主板大盘并行处理

### §8.5 Phase 3：强制退出（铁律触发，即时清仓）

| 铁律2 | 监管任何形式限制措施 | 当日市价全出 |
| 铁律3 | 批量澄清公告（盘前≥5家/盘中≥10家/3日内≥15家） | 盘前→开盘即出；盘中→尾盘减50%次日清；3日累计→当日全清 |

执行价：市价，不挂限价等成交。强制退出后24h禁止同板块再入场。

### §9.4 双轨执行顺序

★★★ **每日执行铁律**：TB操作先行 → 15分钟间隔 → TA评估。

```
3. 等待15分钟
4. 评估TA操作（按strategy.md流程）
5. Daily Review分两节记录：[Track B小节] + [Track A小节]
```

### §9.5 Round Trip跨轨规则

**同标的跨轨Round Trip**：
- TB持仓清仓后，在TA重建（按§9.6升级协议），不算Round Trip（两轨逻辑不同，间隔≥1交易日）
- TB清仓后<3天，TA新建同标的，需有新α级催化剂才免除Round Trip惩罚
- Track内（TA→TA或TB→TB）Round Trip按各自规则处理

**★★★ SS-001/铁律强制出场的Round Trip豁免**：
由铁律（龙头-5%/监管限制/批量澄清）触发的强制出场（非自愿出场）→ 2个交易日等待 + 出现新催化剂 → 豁免Round Trip惩罚，允许重入。
注：必须是"铁律触发"而非"主动决策出场"。铁律出场后2天内，记录`forced_exit_exempt=true`。

---

### §9.6 ★★★ TB→TA升级：严格禁止直接升级

**唯一合法路径**：
1. **先清仓Track B**（按Phase 2/3退出程序执行，不跳步）
2. 等待冷静期（至少1个交易日）
3. 独立评估Track A建仓条件（重新走五步法）
4. 新建Track A仓位，独立计入TA持仓管理

**严格禁止**：
- 把TB仓位直接调高评级（从B+→A-）
- 在未清仓TB的情况下叠加TA仓位
- 用"TB期间赚了钱"作为TA建仓理由（thesis必须是公司质量+竞争壁垒）

TB在持仓期间发现基本面upgrade信号（超预期季报/新订单），**继续持有TB直到退出信号**，然后评估是否满足TA五步法，而不是在TB持有期间偷偷转换track。

---

### §9.7 SS-002 科技主线全线撤退时的紧急覆盖规则

**触发条件**：科技主线全线撤退（科技板块成交占比骤降>15个百分点）且TA已持有8只。

**覆盖规则**：
- 允许临时建入TB，替换同板块TA（同一板块内：TA标的 → 轮动至二线TB标的）
- **净持仓不可增加**：建入TB之前，同板块TA减仓等量
- 双轨合计维持≤10只
- **5个交易日内恢复≤10只约束**：若TA止损清仓后空出名额，TB应归位至≤3只

**此规则不是常规操作**，是系统性主线撤退时的临时应对，触发后需在daily-review注明`ss_002_triggered=true`。

## §10 应急协议（新增）

> 本节处理"无法正常执行"的特殊市场情况。所有"无法执行"均不计入CB/CA违规。

### §10.1 停牌协议

| 阶段 | 处理规则 |
|------|---------|
| 停牌期间 | 维持持仓，每日在daily-review标记"铁律待执行-停牌中" |
| 复牌当日 | 集合竞价无条件卖出，**不论价格**，不分析，不等待 |
| 停牌>5个交易日 | 标记"不可控风险事件"，记录到tb_pain_memory.md，通知Risk Manager（用户手动处置） |
| 停牌天数计算 | 计入持仓时间（对应类型天数上限，停牌不暂停时间止损） |

**北交所停牌特殊处理**：BSE标的流动性更差，复牌首日更可能出现大幅波动。复牌日：集合竞价直接挂最低可成交价（不挂市价），若仍无法成交，等待开盘后第一个可成交价立即成交。

---

### §10.2 涨跌停协议

**跌停（跌停封死无法成交）**：
- 当日：登记退出意向，次日开盘前5min集合竞价卖出
- 次日仍跌停：继续等开盘，继续集合竞价卖出，不等反弹
- 执行记录：标注`limit_down_exit`，不计为铁律违规

**涨停（龙头断板后标的仍涨停）**：
- 涨停日无法买入（若想加仓），等开板后评估
- 若持有TB标的涨停，等次日判断龙头状态再决定减持

**一字板（开盘即封板无法参与）**：
- 节假日后一字板：无法入场，等开板后30-60分钟确认情绪再评估
- 持有仓位一字跌停：按跌停协议处理

---

### §10.3 窗口指导协议

监管窗口指导（含但不限于：监管约谈、口头/书面指导、通报异常交易）：

- **当日执行**：按指导内容执行，不迟疑
- 记录：trade_log中标注`regulatory_guidance`，写入daily-review
- **不计为铁律违规**：窗口指导强制执行 ≠ 主动违反规则，不触发CB计数
- 窗口指导出场后的Round Trip：适用SS-001豁免规则（强制出场类型）

---

### §10.4 通用应急原则

★★★ **"无法执行"的定义与处理**：

| 情况 | 是否计入CB/CA违规 | 记录方式 |
|------|-----------------|---------|
| 停牌无法成交 | 否 | daily-review标注`forced_halt` |
| 跌停封死无法成交 | 否 | daily-review标注`limit_down_exit` |
| 涨停封死（TB标的涨停中） | 否（等待） | daily-review标注`limit_up_hold` |
| 窗口指导强制执行 | 否 | daily-review标注`regulatory_guidance` |
| 系统/网络故障导致延迟 | 否（需提供截图等记录） | daily-review注明`system_failure` |

**唯一例外**：主观决定不执行铁律（如"再等一天看看"）→ 计入CB违规，写post-mortem。

**★★★ tb_review.py CB自动追踪逻辑**：
```python
def update_cb_state(trade_log, tb_scorecard):
    """读取7日滚动trade_log，统计rule_violation型亏损，更新tb_scorecard.json的cb_state"""
    violations = [
        t for t in trade_log
        if t.get("track") == "B"
        and t.get("realized_pnl", 0) < 0
        and t.get("rule_violation", False)  # post-mortem后手动标记
        and days_since(t["exit_date"]) <= 7
    ]
    count = len(violations)
    if count >= 3:
        new_state = "RED"
    elif count >= 2:
        new_state = "YELLOW"
    else:
        new_state = "GREEN"
    tb_scorecard["cb_state"] = new_state
    tb_scorecard["cb_violation_count_7d"] = count
    tb_scorecard["cb_last_updated"] = today_str()
```
