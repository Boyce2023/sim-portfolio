# Track B 建仓系统设计 v1.0

> 编制日期：2026-05-28
> 适用范围：ROTATION_STRATEGY_V1.2 Track B轮动交易
> 前置文件：strategy.md §2.4b（铁律优先）+ ROTATION_STRATEGY_V1.md §1-§3（分类参考）
> 设计原则：**3分钟内走完。每步有GO/NO-GO。速度是Track B的核心竞争力。**

---

## §1 TB建仓五步法（N=5）

> Track A 的五步法核心是thesis验证。Track B 的五步法核心是时机抢占。逻辑不同，步骤不同。

```
Step 1: 市场门 — 今天能不能做Track B？（30秒）
Step 2: 龙头门 — 板块信号是否有效？（45秒）
Step 3: 标的门 — 这只票能不能买？（45秒）
Step 4: Sizing — 买多少？（30秒）
Step 5: 退出预设 — 止损在哪？何时无条件出场？（30秒）
→ 五步通过才下单。全程≤3分钟。
```

---

### Step 1：市场门（GO/NO-GO，30秒）

**查一个数：全市场今日涨停家数。**

| 涨停家数 | 判断 | 操作 |
|---------|------|------|
| ≥80家 | 高潮期末段 | ⚠️ 谨慎，准备退出，不新建 |
| 60-79家 | 主升期 | ✅ GO，可积极建仓 |
| 30-59家 | 启动期/分歧期 | 🟡 只做α级催化剂（S级入场信号） |
| <30家 | 退潮期/无人区 | ❌ NO-GO，零新仓 |

**附加排除条件（任一触发=NO-GO）：**
```
□ 今日大盘跌幅≥1.5%（A类下跌）→ NO-GO
  例外：Type 1事件冲击型，受益方与受损方明确分化时，A类下跌豁免
□ 今日跌停家数>50家 → NO-GO + 立即触发Track B全仓清仓审查
□ F20判定呼气期（每周一更新） → NO-GO
□ 策略硬开关触发（§1.4硬开关任一条件）→ NO-GO
```

**时间节点**：开盘30分钟后（09:45-10:00）涨停家数才稳定，不用预开盘数字。

---

### Step 2：龙头门（GO/NO-GO，45秒）

**核心逻辑：不看标的本身，先看板块龙头。龙头是Track B的生命线。**

| 龙头信号 | 判断 | Track B操作 |
|---------|------|-----------|
| 龙头≥2连板 + 当日仍在涨停 | 传导有效，最优入场 | ✅ GO，T+0~T+1入场 |
| 龙头≥2连板 + 当日高开回落未封板 | 传导弱，谨慎 | 🟡 等明日确认，今日不追 |
| 龙头今日首板涨停 | 传导刚启动 | 🟡 布局T+1窗口，今日不追 |
| 龙头断板（昨日涨停今日开板）| 退出信号 | ❌ NO-GO + 执行清仓已有同板块仓位 |
| 龙头跌>5% | 行情结束 | ❌ NO-GO + **Track B全出，无条件** |

**龙头催化剂检查：**
```
□ 龙头催化剂是否仍在前面？（会议/财报/产品发布未发生）→ YES=GO
□ 催化剂是否已兑现？→ YES=高风险，不新建Track B
□ 是否是Type 4政策型？→ 特殊处理（见Step 3注释）
```

**多龙头区分（ROTATION_STRATEGY §4.4规则）：**
- 主龙头 = 板块内市值最大 + 成交额最大的标的（如中际旭创/胜宏科技）
- **主龙头**跌>5% → Track B全出，铁律不变
- **非主龙头**跌>5% → 减仓50%，再检查主龙头是否同向走弱

---

### Step 3：标的门（GO/NO-GO，45秒）

**标的必须同时满足全部条件：**

**A. 基础资格（三选二豁免注：均须满足）：**
```
□ 同板块（业务相关，不是概念蹭热点）
□ 市值<500亿（主板）或30-100亿（北交所）
□ 尚未大幅启动（今日涨幅<15%；追10%以上溢价极高）
```

**B. 轮动类型匹配：**

| 轮动类型 | 额外条件 |
|---------|---------|
| Type 1 事件冲击 | 有一手来源验证直接业务关联（不是蹭热点） |
| Type 2 产业催化 | 有具体数字支撑（BOM/订单金额/季报数据）|
| Type 3 情绪扩散 | 主线龙头已涨20-50%，今日才出现首板 |
| Type 4 政策驱动 | 方向性政策的受益方；落地性政策**不建仓** |
| Type 5 游资坐庄 | 龙虎榜确认游资建仓（不超过3板入场）|
| 北交所 | 主板龙头连续2日涨幅>20%或连板≥2板后T+3~T+5 |

**C. 入场时机检查（ROTATION_STRATEGY §3.3时机矩阵）：**

| 标的市值 | 最佳入场窗口 | 今天适合吗？ |
|---------|-----------|------------|
| 主板二线（>500亿）| 龙头T+1当日 | 对照今日是第几天 |
| 主板中盘（100-500亿）| 龙头T+1至T+2 | — |
| 主板小盘（<100亿）| 龙头T+2至T+3 | — |
| 北交所（30-100亿）| T+3至T+5 | — |

> **Type 4政策型特殊规则（V1.2）**：政治局级催化+当日成交爆量（>2x 5日均量）→ 龙头涨幅门槛降至20%（不要求50%），但此规则只在政治局级明确政策信号当日生效。

---

### Step 4：Sizing（30秒）

Track B没有thesis驱动的sizing公式。仓位由**轮动类型**和**入场信号等级**决定：

**仓位硬顶（不可超越）：**

| 轮动类型 | 单只仓位上限 |
|---------|-----------|
| Type 1/2/3/4 主板 | ≤15%（strategy.md §2.4b Track B硬顶） |
| Type 5 游资坐庄 | ≤10%（信息滞后折价） |
| 北交所所有类型 | ≤5%（流动性风险） |
| 临时强势子方向（不在L1-L6传导链）| ≤8% |

**入场信号等级调节（在硬顶内浮动）：**

| 入场信号等级（§3.1）| 实际sizing |
|-----------------|-----------|
| S级（三方共振）| 硬顶 × 100% |
| A级（机构+游资共振 / 龙头T+1跟风）| 硬顶 × 70-80% |
| B级（残差动量 / 均线能量）| 硬顶 × 50% |

> **不使用ATR sizing公式**：Track B仓位不由ATR决定，ATR只用于设止损参考线（见Step 5）。

---

### Step 5：退出预设（30秒）

**每笔Track B入场前，必须明确三个退出触发点：**

```
退出触发1（铁律，无条件）：
  → 主龙头跌>5% = 当日全出，不分析，不等自己的止损线

退出触发2（时间止损）：
  → 持有天数上限（按轮动类型查§2.3）
    Type 1追跟风: 5日 | Type 2产业催化: 跟催化剂 | Type 3情绪: 5日 | Type 5游资: 12板
  → 到期无新催化=当日出

退出触发3（价格止损）：
  → ATR(14) × 2.0 作为参考价格止损线（B+级对应K值）
  → 注：Track B止损锚是龙头状态，不是ATR触发——ATR是辅助下限
```

> 退出预设写入格式（盘前If-Then）：
> `IF [标的] 的龙头[X]跌>5% THEN 当日市价全出`
> `IF 持有超过[N]日无新催化 THEN 当日评估出场`

## §3 TB分批建仓规则

### 3.1 核心原则

Track A的分批逻辑是"探针验证公司thesis"。Track B的分批逻辑是"速度vs风险的动态平衡"——有些机会窗口只有1-2天，等待会错过；但高溢价追板也会被打回来。

**按轮动类型分三条路径：**

---

### 路径一：Type 1 事件冲击型（窗口窄，速度优先）

> 典型：NVIDIA新架构发布、地缘事件、技术突破

```
特点：机会窗口D+0至D+1，等不起

Day 0（事件当天尾盘）：
  → 如果龙头已封涨停+成交爆量 → 一次性建70%仓位
  → 如果龙头高开未封板 → 建30%观察仓，等Day1确认

Day 1（T+1当日）：
  → 龙头≥2连板 + 涨停家数>60家 → 补至100%仓位
  → 龙头开板/分歧 → 清30%观察仓出场

Day 2+：
  → 不追加（已过最优窗口），只做减仓管理
```

**一次性全仓条件（TYPE 1特例）：**
- 龙头涨停且封板坚决（封单>5000万）+ 成交额≥前5日均量2倍
- 有明确一手来源验证（非传闻）
- 今日市场涨停≥80家（高潮确认）

---

### 路径二：Type 2 产业催化型（可分批，窗口较宽）

> 典型：BOM拆解、季报超预期、订单公告

```
特点：主升15-45天，可以按批次建仓，不需要第一天冲满

Day 0-1（催化剂发布后）：
  → 建40%探针仓（入场信号等级A/S才做）

Day 2-3（确认传导）：
  确认条件：龙头未断板 + 板块内≥3只标的涨停 + 成交未萎缩
  → 补至80%仓位

Day 5-7（二次加仓窗口）：
  如果T1→T2升级条件满足（§2.4）且尚在主升浪：
  → 补至100%仓位（本轮最后一次加仓机会）

注：每次加仓前重走 Step 1-2（市场门+龙头门）。
```

---

### 路径三：北交所（最保守，T+5前不满仓）

> 流动性窄、日限涨跌30%，分批是必须的，不是选项。

```
特点：涨跌幅大（±30%），流动性陷阱深，加仓要逐步确认

T+3（主板启动后第3天）：
  → 建30%仓位（北交所硬顶5%内的30%，即总仓1.5%）
  → 观察是否出现首板

T+4（确认首板）：
  → 补至60%北交所仓位上限

T+5（确认连板且主板未出现龙头分化）：
  → 补至100%（最大5%总仓）

T+5后不再加仓（最大涨幅区间是T+5至T+7，此后退出准备）

绝对禁止：
  ☒ 北交所标的日均成交<2000万，不建仓
  ☒ 追T+2连板（溢价极高，次日-30%概率大幅上升）
```

---

### 3.2 Type 3/4/5 分批规则

| 类型 | 分批策略 | 说明 |
|------|---------|------|
| Type 3 情绪扩散 | **不分批，一次性70%+次日确认补满** | 窗口3-7天，每次分批都在缩窄窗口 |
| Type 4 方向性政策 | 40%探针 → D+3确认补至80% | 政策行情持续15-75天，可以分批 |
| Type 4 落地性政策 | **不建仓**（发布当日开始减仓，非建仓时机）| 利好兑现=见光死 |
| Type 5 游资坐庄 | **一次性建仓（≤10%或≤5%），不加仓** | 游资信息延迟，加仓时容易被套 |

## §4 TB入场时机矩阵（§3.3+§3.4整合升级）

### 4.1 按标的市值分层的入场决策

| 标的层级 | 最佳入场窗口 | 今日是否在窗口内 | 错过了怎么办 |
|---------|-----------|--------------|-----------|
| 主板龙头二线（>500亿）| 龙头T+1 | 龙头昨日封板+今日仍强 | T+2观察仓30%，不满仓 |
| 主板中盘（100-500亿）| T+1至T+2 | 龙头T+1或T+2当日 | T+3不追 |
| 主板小盘（<100亿）| T+2至T+3 | — | T+4不追 |
| 北交所（30-100亿）| T+3至T+5 | — | T+6不追（口诀：买在涨停前） |
| 北交所超小盘（<15亿）| **不建仓** | 流动性陷阱 | — |

**"已错过"判断标准：**
- 标的当日已涨≥15%（追板溢价过高）
- 今天是龙头启动后第4天以上且已是主板小盘
- 北交所标的已出现连板（T+3之后首板才是最佳买点）

---

### 4.2 按催化剂类型的入场决策矩阵

| 催化剂类型 | 最佳入场 | 持有上限 | 止损触发 | 特别注意 |
|-----------|---------|---------|---------|---------|
| 美股AI巨头财报/GTC/COMPUTEX | D+0尾盘至D+1 | 10-15天（产业链传导）| 龙头-5% | 光模块先于PCB滞后1-4周 |
| 特斯拉/Optimus视频 | D+0尾盘 | 3-5天 | 龙头-5% | 小市值弹性>大市值 |
| 重磅政策首次写入（方向性）| D-3提前布局至D+1 | 7-15天 | 政策落地时减仓 | 买方向性，卖落地性 |
| BOM拆解/季报超预期 | D+0至D+1 | 10-20天 | 季报爆雷当日全出 | 数字越具体置信度越高 |
| 行业大会（COMPUTEX等）| 大会开幕前5-7天 | 大会结束后2-3天出 | 大会结束当日减仓 | 提前抢跑，不追末日 |
| 管制升级受益方 | 恐慌当日D+0（受益方）| 10-20天 | 龙头-5% | T+1-3后反向做多 |
| 传闻/无URL消息 | **不建仓** | — | — | 等官方确认 |

---

### 4.3 入场时机实时决策流（盘中用）

```
当下时间是？
├── 09:15-09:30 集合竞价
│   → 只看：龙头是否高开（>5%=积极信号）
│   → 不下单：等开盘确认
│
├── 09:30-10:00 开盘30分钟
│   → 观察：涨停家数方向、龙头是否维持高位
│   → 初步判断：今天是哪个市场环境（主升/分歧/退潮）
│
├── 10:00-14:00 主交易时段
│   → 最佳建仓窗口
│   → 顺势入场：标的今日已涨但尚未涨停(<15%)
│   → 涨停入场：涨停封单坚决 → 排队候买（排不上就不买）
│
├── 14:00-14:30 尾盘前
│   → 检查：龙头是否有断板迹象
│   → 如有：减仓，不加仓
│
└── 14:30-15:00 最后30分钟
    → 催化剂当天（Type 1）的最后建仓窗口
    → 看尾盘资金流方向（持续净买=可入；尾盘出货=不追）
```

## §5 每日入场决策流程

### W1 盘前（09:00-09:30）

```
1. 读latest_news.json — 昨晚到今晨有无新催化剂？
   有：确认类型（Type 1/2/3/4/5）+ 直接受益标的
   无：执行既有持仓的If-Then预承诺

2. 读portfolio_state.json — 当前Track B仓位
   Track B持仓数量：___只
   Track B总仓位：_____%（距50%上限还有多少空间？）
   今日到期的时间止损：哪些？

3. 写盘前If-Then预承诺（仅Track B部分）：
   IF [龙头X]今日断板 THEN [标的Y]尾盘减仓
   IF [龙头X]今日跌>5% THEN [所有Track B]当日市价全出
   IF [标的Z]今日触及时间止损第N天 + 无新催化 THEN 尾盘出场
```

### W1 盘中（09:30-15:00）

**三个检查时间点：**

```
10:00 — 市场门确认（涨停家数稳定后）
  → 涨停家数：___家 → GO/NO-GO
  → 今日总体方向确认：主升/分歧/退潮

11:30 — 龙头门确认（半日确认龙头强弱）
  → 主龙头是否维持涨停？
  → 有无断板预警？
  → 板块内蔓延是否扩散（≥3只涨停=传导有效）

14:30 — 建仓/减仓决策窗口
  → 新入场：最后窗口，根据入场时机矩阵决定
  → 既有仓位：龙头状态是否有变化？是否触发退出？
  → 写入If-Then更新（盘后生效的才改，今日已定的不改）
```

**盘中禁止：**
```
☒ 看到涨了就追（追板溢价>15%=不追）
☒ 龙头已断板还试图"抄底小票"
☒ 修改今日已写入的If-Then退出预设
```

## §6 Track A与Track B冲突处理

### 6.1 资金分配原则

Track B的资金从Track A的**空余仓位**中划拨，不占用Track A建仓资金。

```
总仓位上限：100%
Track A 可用空间：由Track A持仓决定（A+35% / A25% / A-20%等）
Track B 可用空间：= 100% - Track A已用仓位 - 现金缓冲（5%）

示例：
  Track A持仓：新易盛12% + 沪电12% + 巨化8% = 32%
  Track B可用：100% - 32% - 5% = 63% 的空间
  Track B上限：15%/只 × N只（但总计不超过可用空间）
```

**Track B绝不超过总仓的30%（硬性建议，非死规则）：**
- 轮动交易本质是动量博弈，超过30%时整体portfolio向短期情绪过度暴露
- 例外：牛市顶峰期（强吸气 + 量能>2万亿），可临时放宽至40%

### 6.2 同日双信号优先级规则

**情景一：同日Track A和Track B都有新入场信号**

```
规则：Track A优先，Track B仓位从余量划拨

操作步骤：
1. 先确认Track A是否需要建仓/加仓（thesis驱动，高优先级）
2. Track A操作完成后，检查剩余仓位空间
3. 剩余空间≥15% → Track B可建仓（按正常流程）
4. 剩余空间<15% → Track B本日不建新仓

例外：Track B是S级信号（三方共振）且Track A信号是B级
  → 当日Track B优先，Track A延至次日
```

**情景二：Track A持仓遭遇停损，Track B同日有入场信号**

```
规则：止损当日暂缓Track B新建仓

原因：止损后需30秒分类（§3.3下跌分类），分类完成后才判断市场环境
如果止损原因是"板块结束"（A类下跌/市场整体回撤）→ Track B同样受影响
如果止损原因是"公司特定利空"（D类）→ Track B不受影响，可正常建仓
```

**情景三：Track B龙头跌>5%，Track A持有同板块标的**

```
Track B铁律：同板块Track B仓位当日全出
Track A：不受Track B铁律约束，独立评估thesis
  → 执行§3.3下跌分类：是A类（大盘原因）？B类（无新闻）？C/D类？
  → Track A有自己的止损线，不因Track B退出而自动清仓
```

**情景四：Portfolio Heat超红线（>15%）且有Track B建仓信号**

```
规则：Heat红线触发时，Track B新建仓暂停
原因：Track B持仓持有期短、频繁换手，先降Heat再建仓
操作：先执行到期的Track B仓位清仓 → Heat降至<12% → 再考虑新Track B
```

## §7 Track B专用行为铁律（6条）

```
TB-L1 龙头信号优先于标的信号
  → 标的再好看，主龙头断板当日不入场不持有。无例外。

TB-L2 时间止损是刚性约束
  → Track B到期无新催化就出，不因"还有浮盈"而延期。
  → 延期一次=下次更难出。Type 3到5天全出，不磨蹭。

TB-L3 不追入场窗口外的标的
  → 错过北交所T+3-T+5窗口，不追T+6以后的连板。
  → 口诀：买在涨停前，卖在连板第一天。

TB-L4 Track B不升级为Track A
  → 即使标的涨了很多，基本面看起来不错，也不升级。
  → 升级必须经过完整Track A五步法重新评级（thesis → catalyst → stop → size → scale）。

TB-L5 不做逆势Track B
  → 板块已在回调时不抄底小票。
  → "龙头跌完了应该能反弹"的逻辑不是Track B逻辑，是猜测。

TB-L6 澄清公告≥5家当日全出
  → 不管浮盈多少，批量澄清公告是最高退出信号。
  → 不等公告数量增加，5家就够。
```
