# TB Pain/Reward Architecture — Track B轮动交易行为反馈系统

> 版本：v1.0 | 2026-05-28
> 对标基础：strategy.md §10 Pain/Reward Architecture（Track A版）
> 适用范围：Track B（A股板块轮动，动量接力，持有1-3周，评级上限B+）
> Claude分析意见，非用户投资结论。

---

## 设计原则与Track A的核心差异

Track B的行为反馈系统不是Track A的缩小版，而是针对完全不同的失败模式重新设计。

| 维度 | Track A | Track B |
|------|---------|---------|
| 失败根因 | thesis判断错/催化剂错估/处置效应 | 入场时机错/产业链传导层级搞错/过夜信息滞后 |
| 止损触发 | ATR触发 / D级48h清仓 | 铁律止损三条 + 拥挤度退出 + 龙头断板 |
| "正确出场"边界 | 催化剂兑现出场 = 不触发Pain | 板块热度自然退潮出场 = 不触发Pain |
| 评级上限 | 无上限（A+可用） | B+硬顶（动量交易不给A） |
| 交易频率 | 中频（周级操作） | 高频（日级入场判断） |
| 连续亏损概率 | 低（每次都有thesis支撑） | 中（市场整体退潮时全线亏损是正常） |
| Pain Memory重点 | 判断质量（thesis对不对） | 执行纪律（哪条规则没遵守） |

核心校准：**Track B的Pain几乎全部来自"违反了已知规则"，而非"遇到了未知情况"**。这意味着Pain Memory的3问不是"判断哪里错了"，而是"哪条规则没遵守，为什么没遵守"。

## PART A — PAIN SYSTEM

### TBL1 Pain Memory（轮动痛觉记忆）

**触发定义**：止损出场（铁律止损1/2/3触发）或亏损>2%出场。

**不触发**：板块热度自然退潮，按§6退出信号纪律性出场（正确执行不惩罚）。龙头断板按规则T+1出场，即使亏损，也是纪律性出场而非失误。

**3问Post-Mortem（止损当天完成，5-10min）**：
1. 违反了哪条具体规则？（不能答"市场不好"——必须指向ROTATION_STRATEGY_V1.md的具体规则编号）
2. 入场前是否过了入场三步检查？哪步没过还是强行入场的？
3. 下次同类情况的if-then是什么？（必须是可执行的具体条件，不能是"以后更小心"）

**文件**：`pain_memory_tb.md`（rolling 5条）。建仓前扫描match → 必须回答"这次和上次有什么结构性不同"。

#### TB Pain Pattern 详细定义（7种）

**TB-P1：追末端扩散标的**

```
Tag: tb_late_diffusion
触发条件: L6信号已出现（远端行业小票涨停，如AI算力链主线已见顶后超级电容出现）
            且我仍入场（认为"还有机会"）
识别信号: 产业链层级 = L5/L6（来自taxonomy_lifecycle.md传导层级图）
           主线拥挤度 > 35%（从成交占比监测）
           龙头已无新高3天以上

Post-mortem 3问:
1. 入场时我认为是哪一层（L几）？实际是哪一层？
2. L6信号（taxonomy_lifecycle.md §传导层级）入场前有没有出现过，我看到了吗？
3. "主线见顶后末端弹性"这个判断，有没有任何一手数据支撑？

预防规则: 入场前确认传导层级 ≤ L4（L5出现=警戒/减仓，L6=禁止入场）
           L6出现 = 自动进入主线退出评估，不是入场信号
```

**TB-P2：蹭热点无关涨停**

```
Tag: tb_unrelated_hype
触发条件: 入场标的与当日主线的业务关联 = 无一手来源验证
           （如DeepSeek主线日，股票名含"智"字或历史上做过AI项目但主营无关）

识别信号: 龙虎榜买方 = 纯游资席位，无机构席位
           公司公告/年报无关键词匹配（非Google搜索匹配，要求公司文件来源）
           价格前一日+0%，当日跟板 = 纯搭便车

Post-mortem 3问:
1. 入场时我引用的业务关联来源是什么？（公告/年报/还是新闻标题？）
2. 如果业务关联无效，当时的入场逻辑是什么？（诚实回答是否是"涨停了，感觉有关联"）
3. 下次"感觉有关联"时，最低验证标准是什么？（要求：公司文件中找到关键词，非媒体报道）

预防规则: 无公司文件级来源验证业务关联=禁止入场
           "历史曾经参与XX"不算关联（需要当前主营包含）
           连板三天以上才注意到 + 无关联 = 直接跳过，不入场
```

**TB-P3：游资票过夜（T+1延迟信息陷阱）**

```
Tag: tb_hot_money_overnight
触发条件: Type 5游资票持有过夜，次日龙虎榜显示核心席位T+1已大额净卖出
           （信息天然T+1延迟，今天的龙虎榜看的是昨天的行为）

识别信号: 入场日龙虎榜未公布（当日入场后龙虎榜延迟1天）
           次日开盘异常弱势（-3%以上低开）
           次日龙虎榜确认：核心席位净卖出 > 净买入50%

Post-mortem 3问:
1. 入场时能看到的最新龙虎榜是几天前的？实际参考的有效性是否已过期？
2. 持有过夜决策时，有没有以下任何一个减仓信号：连板≥8板/当日成交量萎缩/板块拥挤度>35%？
3. Type 5游资票持有上限是"龙虎榜确认买，龙虎榜出现卖立即走"，我在哪一步偏离了这个规则？

预防规则: Type 5游资票不超过3板（§7.3仓位表注释）
           连板≥8板无条件减仓50%（§7.5铁律，不等龙虎榜确认）
           当日拿到的龙虎榜数据是T-1，持有过夜必须认可此信息滞后风险
           梭哈魔咒检查（§7.5）：章盟主/陈小群出现 + ≥8板 = 次日必须出
```

**TB-P4：误判轮动类型（T3当T2处理）**

```
Tag: tb_type_mismatch
触发条件: 入场时判断为Type 2（产业催化，可享受A类下跌豁免+持有更久）
           实际为Type 3（情绪扩散，无基本面支撑，快速衰减）
           结果：按T2逻辑持有太久，被T3的快速衰减套住

识别信号（事后识别）: 后续未出现基本面数据验证
                       板块热度在启动后5天内即开始衰减
                       龙头无业绩/订单公告支撑

Post-mortem 3问:
1. 入场时判断Type 2的依据是什么？（是否有具体数字：订单金额/产品出货量/会议纪要中的可量化指标）
2. taxonomy_lifecycle.md §轮动类型特征表的哪几项我没过就入场了？
3. 下次遇到类似情况，Type 2 vs Type 3的鉴别标准是什么？（要有可量化的区分点，而非"感觉上"）

预防规则: Type 2入场必须有可量化的基本面数字支撑（订单/出货/财报数据）
           无数字支撑的"产业趋势"统一归Type 3处理（更保守的持有逻辑）
           Type 3上限5天出场，不等基本面验证（验证周期比Type 3生命周期长）
           A类下跌豁免仅适用Type 1（§7.3注释），Type 2/3大盘跌不豁免
```

**TB-P5：连板股追高（≥5板追入次日断板）**

```
Tag: tb_chasing_high_board
触发条件: 连板≥5板时追入，次日断板（炸板或跌停）
识别信号: 入场当日连板计数 ≥ 5
           振幅递减规律：第三波涨幅为第一波25-40%（quant_rotation_rhythm.md §四）
           炸板率历史数据在连板≥5后显著上升

Post-mortem 3问:
1. 入场时连板计数是多少？有没有超过§7.5的连板警戒线？
2. 入场时我对振幅递减规律（每波涨幅递减50-70%）的判断是什么？预期收益是多少？
3. 连板≥5后的统计分布是怎样的（历史断板概率）？我的仓位是否反映了这个概率？

预防规则: 连板≥5板 = 仓位减半（即使仍在上涨）
           连板≥8板 = 无条件减仓50%（§7.5铁律）
           第三波入场检查（§信号速查手册注释）：第三波涨幅期望<第一波40%，要求收益预期相应调低
           入场连板计数 ≥ 5 + 无机构席位（纯游资）= 直接跳过
```

**TB-P6：节假日一字板追涨**

```
Tag: tb_holiday_gap_chase
触发条件: 节假日后首日，标的跳空高开（一字板或大幅高开），立即追涨买入
识别信号: 节假日后首日时间标记
           开盘价 > 前收盘 × 105%（高开≥5%）
           买入时间在开盘后30分钟内（无充分判断时间）

Post-mortem 3问:
1. 买入时对于"节假日高开通常是情绪高点"这一规律，有没有考虑？
2. 节假日前最后一个交易日的仓位准备（if-then预承诺）是否已做，还是临时反应？
3. 如果节假日后不追涨，等开盘后30-60分钟确认盘口再入场，对结果有何影响？

预防规则: 节假日后首日开盘30分钟内不追涨（等情绪稳定）
           节假日前制定if-then预承诺（非盘中决策）
           一字涨停日 = 无法参与（无有效入场价格），等开板后再评估
           节假日前预期利好已Price In的标的（节前已有连板）= 节后开盘减仓而非加仓
```

**TB-P7：梭哈魔咒未识别**

```
Tag: tb_allIn_trap
触发条件: 超大单游资买入（章盟主/陈小群等）作为入场信号跟入，
           且已连板≥8板，次日或T+2被套

识别信号: 龙虎榜出现超大单知名游资营业部（历史记录在signal_dragon_tiger.md §核心结论）
           连板天数 ≥ 8（§7.5警戒线）
           成交量异常放大（超前10日均量300%+）= 出货信号非买入信号

Post-mortem 3问:
1. 入场时有没有检查过§7.5的梭哈魔咒定义（≥8板 + 超大单游资 = 顶部信号）？
2. "超大单游资出现"在signal_dragon_tiger.md中被定义为入场信号还是退出信号？（应为退出信号）
3. 如果反向操作（见到梭哈信号减仓而非加仓），成本/收益比是多少？

预防规则: 梭哈魔咒识别公式（§7.5）：≥8板 + 超大单游资（章盟主/陈小群等） = 100%次日出
           超大单游资出现 = 出货信号，不是背书信号
           见到梭哈信号时持仓的处理：当日减仓50%，次日无论涨跌全出
```

### TBL2 Circuit Breaker（TB熔断器）

**设计逻辑**：Track B交易频率更高，全面连续止损概率更大。但"板块整体退潮"是市场原因，不应触发个人行为惩罚。因此需要区分"系统性亏损（市场退潮）"和"规则违反型亏损"，只惩罚后者。

**TB vs Track A熔断器对比**：

| 参数 | Track A | Track B | 原因 |
|------|---------|---------|------|
| YELLOW触发 | 周组合跌>5% 或 连续止损≥2 | 规则违反型亏损≥2笔（7日内）| Track B市场退潮时全线亏损是正常，不惩罚正确执行的亏损 |
| RED触发 | 周组合跌>8% 或 连续止损≥3 | 规则违反型亏损≥3笔（7日内）| 同上 |
| 恢复条件 | 连续2周无止损 | 规则违反型亏损=0（连续5个交易日）| Track B高频，2周太长 |

**关键定义：规则违反型亏损**
= 亏损出场 + post-mortem显示违反了以下任一规则：
- 7个Pain Pattern之一未遵守
- 入场三步检查未完整过（§附录入场三步检查）
- 铁律止损触发前已有明确退出信号被忽略

注：板块退潮按规则纪律性出场的亏损，即使连续5笔，**不计入规则违反型亏损计数**。

**熔断器状态表**：

| 状态 | 触发条件 | 效果 | 恢复条件 |
|------|---------|------|---------|
| GREEN | 默认 | 正常操作，全部轮动类型可用 | — |
| YELLOW | 规则违反型亏损 ≥ 2笔（7日滚动）| sizing ×0.5；每日新建Track B ≤ 1只；禁止Type 3和Type 5新建（高风险类型停用） | 连续5个交易日规则违反型亏损 = 0 |
| RED | 规则违反型亏损 ≥ 3笔（7日滚动）或 单笔亏损 > 5%（单只仓位 × 5%）| 禁止所有Track B新建仓；仅执行已有持仓的止损和退出 | 5个交易日无规则违反亏损 → YELLOW；再5日 → GREEN |

**额外触发（不依赖亏损笔数）**：
- 同一天触发两条铁律止损（§7.2）= 直接进入RED，原因：系统性风险已至，非个人判断问题，但仍需强制停止新建
- 同一Pain Pattern 7日内出现 ≥ 2次 = 该Pattern对应的轮动类型进入"类型级别暂停"（见TBL3）

**板块退潮的特殊处理**：
市场环境硬开关触发（§1.4）→ 策略自动关闭 → 此期间的亏损出场**不计入Circuit Breaker**。硬开关关闭是正确的市场响应，不是系统失灵。

### TBL3 Conviction Credit — 按轮动类型分层（TB信念信用）

**基础设计**：Track B评级上限B+，不存在Track A的"A+止损30天禁止A+"逻辑。但引入轮动类型级别惩罚，防止同类错误反复。

**全局信念信用规则**：

| 情形 | 处罚 | 解除 |
|------|------|------|
| 同一Pain Pattern Tag第2次触发（30日内）| 该Pattern对应的轮动类型降至B-级，sizing自动×0.7，持续14天 | 到期自动解除 |
| 同一Pain Pattern Tag第3次触发（60日内）| 该轮动类型暂停2周（不新建）| 暂停期满后重新从B-开始 |
| Grade Accuracy < 40%（≥5笔规则违反型亏损）| 全Track B降至B-级，sizing ×0.7 | 规则违反型亏损连续5笔 = 0后恢复 |

**按轮动类型的信念信用规则**：

| 轮动类型 | Pain Pattern主要来源 | 同类连亏2次的惩罚 | 同类连亏3次的惩罚 |
|---------|-------------------|----------------|----------------|
| Type 1 核心受益方 | TB-P4（轮动类型误判）| 该类型sizing ×0.8，14天 | 禁止Type 1新建，2周 |
| Type 2 产业催化 | TB-P4（T3当T2处理）| 要求提供≥2条可量化数字才可入场（额外验证门槛），14天 | 禁止Type 2新建，2周 |
| Type 3 情绪扩散 | TB-P1（末端扩散）/ TB-P6（节假日追涨）| sizing ×0.5（从≤15%降至≤7.5%），14天 | 禁止Type 3新建，2周 |
| Type 4 政策方向 | TB-P2（蹭热点）| 要求公司文件验证政策关联，14天 | 禁止Type 4新建，2周 |
| Type 5 游资坐庄 | TB-P3（过夜陷阱）/ TB-P5（连板追高）/ TB-P7（梭哈魔咒）| sizing压至≤5%（原5-10%上限进一步压缩），入场上限2板（原3板），14天 | 禁止Type 5新建，2周 |
| 北交所标的 | TB-P5 / TB-P1 | 单只上限降至≤3%（原≤5%），持有上限压至3天（原5天），14天 | 禁止北交所新建，2周 |

**信念信用积分（正向）**：
- 同一轮动类型连续3笔规则遵守（含盈利和纪律性亏损）= 该类型信念信用+1级，可至B+（不可突破）
- 正向积分不累加（上限B+，动量交易不给A）

## PART B — VICTORY PROTOCOL

### TBL4 Victory Memory（TB胜利记忆）

**触发定义**：盈利出场 + 满足以下任一条件：
- 盈利 ≥ +5%（单笔）
- 正确识别退出信号（如拥挤度>48%后第1天退出，之后价格下跌验证）
- 轮动类型识别正确且持有周期在预期范围内结束

**3问Victory Memo（盈利出场当天，5min）**：
1. 哪条具体的轮动规则帮我赚到了这笔钱？（具体到信号名称，如"L1→L2传导在第3天确认"）
2. 退出时机的MFE检查：我的出场时间，相比信号速查手册的最优退出点，早/晚了多少？
3. 这个setup在下次可以直接复用的条件是什么？（3个可量化的entry条件）

**文件**：`victory_memory_tb.md`（rolling 5条）。建仓前match →如果当前setup与某条Victory完全匹配（轮动类型相同 + entry层级相同 + 市场环境相似），sizing可至当前等级上限（不超B+硬顶）。

**特殊Victory Pattern（高价值记忆，额外标注）**：

*VP1：龙头T+1精准入场二线*
```
成功条件: 龙头T+0连板确认 + T+1次日二线标的在开盘1小时内以5%以内溢价入场
           且最终盈利≥5%
3问重点: 时机窗口确认（T+1开盘1小时是"传导确认期"，超过则溢价过高）
         同类二线标的范围（哪种市值/哪种业务关联是最佳入场）
记忆价值: 中-高（2-3案例后pattern稳定性验证，可进入playbook_tb.json）
```

*VP2：产业链传导时机正确（L1→L2切换成功）*
```
成功条件: 识别传导层级从L1切换到L2的时间节点 + 在L2峰值前退出
           传导Lag参数参考: PCB 1-4周（V1.2修正从4-8周）
3问重点: L2峰值识别信号是什么？（拥挤度具体数值 / 量能变化 / 龙头动作）
         Lag时间的预判精度（预期x天vs实际y天）
记忆价值: 高（AI May案例已验证，1个完整案例后即可入playbook）
```

*VP3：拥挤度信号准确退出*
```
成功条件: 拥挤度>48%信号出现后T+0或T+1出场，之后价格下跌≥5%验证
3问重点: 拥挤度信号的提前量（退出时vs最高点差距多少%）
```

```
           且符合持有上限5天 + 单只≤5%规则
3问重点: 北交所入场时间（T+3~T+5的实际执行是否准确）
```

### TBL5 Conviction Amplifier（TB信念放大器）— 仅GREEN状态生效

**参数保守设计原因**：
- Track B单只仓位已是≤15%，加成空间有限
- 轮动高频意味着连续成功有时是市场环境好而非技能（过热警示）
- B+评级上限决定加成天花板

**放大器状态表**：

| 状态 | 触发条件 | Sizing倍率 | 额外条件 |
|------|---------|-----------|---------|
| NORMAL | 默认 | ×1.0 | — |
| ELEVATED | 同类轮动类型连续盈利≥3笔 + 无规则违反型亏损（10日内）| ×1.15 | 加成后单只仍不超B+级对应仓位上限 |
| PEAK | 跨类型连续盈利≥5笔 + Grade Accuracy > 70%（规则遵守率）+ 无规则违反型亏损（15日内）| ×1.2 | PEAK状态最长持续5个交易日后强制回到ELEVATED重新评估 |

**反转信号检查（PEAK时必做）**：
连续成功5笔以上 → 强制检查市场环境是否过热（涨停家数/拥挤度趋势/F20评分）。
如市场呼吸评分连续下降 + 涨停家数开始回落 → PEAK不适用，反而触发减仓提醒（"市场可能即将呼气，你的连续成功是市场beta，不是alpha"）。

**放大器不适用情形**：
- Type 5游资票（风险太高，加成可能导致梭哈魔咒加速触发）
- 北交所标的（流动性风险，不加成）
- 连续成功集中在同一板块热度周期内（非独立样本，不算真正的放大器触发条件）

## PART C — FOMO判定系统（TB版）

### TB-FOMO vs 正常轮动 vs Timing错误的区分标准

**核心前提**：Track B的"我没买到那波上涨"默认不是FOMO。轮动策略覆盖的是"资金搬家"逻辑，主线涨完后才看到是正常的——没有人能每次都抓到L1第一波。

**三步判定流程（遇到"我是不是FOMO"的问题时必过）**：

```
Step 1 — 入场时有没有完整的轮动逻辑？
  有（轮动类型明确 + 传导层级确认 + 入场三步检查通过）→ 不是FOMO
    → 进入Step 2判断是"正常轮动"还是"Timing错误"
  没有（看着涨了想买/感觉会继续涨）→ 是FOMO → Tag: tb_fomo_impulse

Step 2 — 入场层级有没有超过L4？（有完整逻辑但层级太晚）
  层级 ≤ L4，时间在合理传导窗口内 → 正常轮动（即使亏了也是市场判断问题）
  层级 ≤ L4，但时间超过传导窗口（如T+5以后才追主板）→ Timing错误
  层级 ≥ L5，或完全是L6末端 → 是FOMO变体（TB-P1：追末端扩散）→ Tag: tb_late_diffusion

Step 3（仅FOMO确认后）— 是主动FOMO还是被动FOMO？
  看到别的票涨了，放弃已有持仓去追 → 主动FOMO（最危险，和Track A的L10关注点漂移一样）
  新板块启动，从零开始追入但层级太晚 → 被动FOMO（次危险，通过规则能防）
```

**四种模式的完整区分表**：

| 模式 | 特征 | Tag | 处罚 | 正确处理 |
|------|------|-----|------|---------|
| 正常轮动（含亏损）| 逻辑完整 + 层级≤L4 + 时间窗口正确 | 按实际Pain Pattern标注（如timing_error）| 无额外处罚，分析技术问题 | 3问post-mortem优化入场时机 |
| Timing错误 | 逻辑完整但入场太晚（超出传导窗口）| `tb_timing_late` | 无额外处罚，但记录Lag时间误判 | 优化传导Lag参数 |
| 被动FOMO | 无完整逻辑 + 看到涨停追入 + 层级≤L4 | `tb_fomo_impulse` | TBL3降档（按对应轮动类型），15日 | 强制入场三步检查 |
| 追末端FOMO | 无论有无逻辑 + 层级≥L5 | `tb_late_diffusion` | TBL3降档（Type 3/4），15日 | 立即识别传导层级，拒绝入场 |
| 主动FOMO换仓 | 放弃有催化剂持仓去追新涨停 | `tb_fomo_rotation` | TBL2 + TBL3双重处罚，视为严重违规 | L10检查：thesis变了吗？关闭行情等1小时 |

**"板块涨了我没买"的正确认知**：
Track B的覆盖目标是有完整轮动逻辑的入场机会，不是追逐所有上涨板块。每个板块热度周期中：
- L1第一波（龙头涨停）：Track B入场窗口，可能已过
- L2传导（二线标的）：Track B核心目标
- L3+：逐层降低参与度，L5/L6=不参与

"L1第一波没抓到"≠ FOMO，是Track B的正常局限性。应对方法是下次更早识别催化剂，不是追L5/L6末端。

## PART D — Scorecard（每session开始 < 2分钟快速检查）

**设计原则**：Track B Scorecard是"今天能不能出手"的快速判断，比Track A的Scorecard更关注市场环境（因为Track B是市场环境依赖型策略）。

**TB-Scorecard（每session开始，手动检查，标准化格式）**：

```
=== TB Scorecard [日期] ===

[0] 市场环境（硬开关前置，30秒）
  □ §1.4硬开关：全部4条是否处于"开启"状态？
    - 沪深300过去20日 > -8% ？
    - 全市场成交额 ≥ 1.5万亿（连续5日）？
    - F20市场呼吸 = 吸气/中性？
    - 涨停家数（昨日）≥ 30家？
  → 任一否 = 今日Track B不新建。仅执行已有持仓止损。停止检查。

[1] CB状态（10秒）
  Circuit Breaker = GREEN / YELLOW / RED
  当前规则违反型亏损计数（7日滚动）= _笔
  → RED = 今日不新建。YELLOW = sizing ×0.5，仅1只/日。

[2] 类型级别暂停检查（10秒）
  当前暂停的轮动类型 = [Type X，到期日：YYYY-MM-DD]
  → 确认今日目标不在暂停类型内。

[3] Pain Memory扫描（20秒）
  最近1条规则违反亏损：[Tag / 日期 / 轮动类型]
  今日目标与上次失误的相似度：高/中/低
  → 高相似度 = 必须回答"这次有什么结构性不同"再入场。

[4] Victory Memory（10秒）
  最近1条Victory Pattern：[Tag / 日期]
  今日目标是否match Victory条件：是/否
  → 是 = sizing可至当前等级上限（不超B+）。

[5] CA状态（10秒）
  Conviction Amplifier = NORMAL / ELEVATED / PEAK
  → PEAK时 + 市场F20连续下降 = 反向信号警示。

[6] 今日入场候选（30秒）
  目标轮动类型 = Type X
  传导层级确认 = L_
  入场三步检查 = 通过/未通过
  → 未通过 = 今日不入场此标的。

=== 输出：今日Track B状态 = 可入场/限制入场/不入场 ===
```

**Scorecard输出逻辑**：

| 状态 | 条件 | 操作权限 |
|------|------|---------|
| 可入场 | 硬开关全绿 + CB=GREEN + 无类型暂停 | 按正常规则操作 |
| 限制入场 | 硬开关全绿 + CB=YELLOW | 仅1只/日，sizing ×0.5，禁止Type 3和Type 5 |
| 不入场 | 硬开关任一否 或 CB=RED | 仅执行已有持仓的止损和退出 |
