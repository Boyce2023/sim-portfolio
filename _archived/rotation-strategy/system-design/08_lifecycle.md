## 1. Track B 持仓生命周期阶段

### 1.1 四阶段定义

Track B持仓周期1-3周，远短于Track A的催化剂兑现周期。四阶段对应不同的行为逻辑：

```
阶段0 → 建仓期 (D+0 ~ D+2)
阶段1 → 持有期 (D+2 ~ 出场信号)
阶段2 → 退出准备期 (信号出现后 ~ 清仓完成)
阶段3 → 强制退出 (铁律触发 → 即时清仓)
```

---

### 1.2 阶段0：建仓期 (D+0 ~ D+2)

**判定标准（全部满足才进入建仓期）：**
```
□ ROTATION_STRATEGY_V1.md §3.2 入场检查清单全部通过
□ 市场硬开关未触发（§1.4 全部为否）
□ 轮动类型已识别（Type 1/2/3/4/5，§2.1确认）
□ 龙头仍创新高 + 成交放量（不做龙头已见顶的接力）
```

**建仓行为：**
- D+0：探针仓，40%目标仓位。按§3.3入场时机表选对应标的类型。
- D+1：确认仓，追加60%。条件：D+0持仓收盘未跌>1×ATR(K=2.0) + 板块龙头未断板。
- D+2：最后入场窗口。D+2仍未建齐 → 放弃，不追。超过最优入场窗口的建仓=追高。

**观察龙头确认（Track B独有步骤）：**

| 建仓日 | 必须观察的龙头信号 | 不满足 → 动作 |
|--------|-----------------|------------|
| D+0 | 龙头当日涨幅≥5% + 成交放量（>5日均量×1.3） | 不建探针仓 |
| D+1 | 龙头未断板 + 板块涨停家数仍≥30家 | 放弃确认仓，只持探针仓 |
| D+2 | 龙头仍创新高（T+2新高=强势延续信号） | 若未创新高，仓位不超过探针仓 |

**D+0当日可一次性全仓的特殊情况（复用strategy.md §2.7逻辑）：**
- Type 1事件冲击：事件当日T+0尾盘确认，一字板开盘次日等开板建仓，可全仓。
- Type 2产业催化：BOM/季报超预期当日，确认龙头放量后，可全仓。
- 注意：Track B的"全仓"上限仍是等级硬顶（最高15%组合仓位），不是portfolio全仓。

---

### 1.3 阶段1：持有期 (D+2 ~ 出场信号)

**持有行为：**
1. 每日开盘前：确认龙头状态（未断板 = 继续持有）
2. 每日盘中：按If-Then预承诺执行（§7预承诺格式），盘中不修改
3. 持有上限：按轮动类型（ROTATION §2.3表格）执行时间止损
4. 棘轮止损：每日更新，仅单向上移（复用strategy.md §3.1 Layer 2逻辑，但K值固定=2.0）

**持有期监控项（每日收盘后，优先级从高到低）：**
```
Level 1（即刻响应）: 龙头断板/跌>5% → 触发阶段3强制退出
Level 1（即刻响应）: 监管限制措施 → 触发阶段3强制退出
Level 2（次日响应）: 炸板率>50% → 进入阶段2退出准备
Level 2（次日响应）: 板块成交占比>40%（非T2） → 进入阶段2退出准备
Level 3（观察）: 持仓天数达上限的前2天 → 开始准备退出计划
Level 3（观察）: 龙头成交量较峰值缩减>30% → 进入阶段2退出准备
```

**持有期棘轮止损参数（Track B专用，K=2.0固定）：**

| 盈利阶段 | 止损调整 | 说明 |
|---------|---------|------|
| 建仓后 | Entry - 2×ATR(14) | 初始止损，硬止损-10% |
| +10%浮盈 | 移至保本 | Track B持有周期短，保本线早移 |
| +20%浮盈 | High - 2×ATR | 棘轮开始 |
| +35%浮盈 | High - 1.5×ATR | 收紧跟随 |
| 持仓达类型天数上限 | 无论盈亏，计划出场 | 时间止损优先于利润 |

Track B不使用strategy.md §3.1的时间衰减（Layer 3）——时间上限由轮动类型§2.3决定，已内含时间逻辑。

### 1.4 阶段2：退出准备期 (信号出现 ~ 清仓完成)

**进入退出准备期的触发信号：**
```
退出信号A（≥2个→立即进入）：
  - 炸板率30-50%（分歧开始）
  - 全市场涨停家数从>60回落至30-60（分歧期）
  - 主线成交量较峰值缩减>30%
  - 持仓天数达类型上限前2天（时间预警）
  - 板块成交占比>40%（非T2标的）

退出信号B（单个即刻进入）：
  - 炸板率>50%
  - 板块成交占比>48%（T2亦不豁免）
  - 龙头"换人"（原龙头断板，新龙头出现）
  - L6末端信号首次出现+48h无新催化
```

**分批减仓计划（退出准备期标准节奏）：**

| 节点 | 操作 | 条件 |
|------|------|------|
| 信号当日收盘 | 减仓50% | 进入退出准备期 |
| 次日判断 | 再减至留10%或全出 | 若龙头当日仍未断板：留10%观察；若已弱化：全出 |
| 最迟T+2 | 全出 | 退出准备期内无条件完成清仓 |

**Track B退出准备期不等待：** 不像Track A的3天决策窗口，Track B退出准备期最长2天。原因：动量交易的特性是快进快出，信号出现后犹豫的代价极高（见ROTATION §1.5元规则）。

---

### 1.5 阶段3：强制退出 (铁律触发，即时清仓)

**三条铁律（复用ROTATION §7.2，在阶段1/2随时可触发）：**

```
铁律1：板块龙头（主龙头）单日跌>5% → Track B当日市价全出，无例外
  非主龙头-5%：减仓50%后按§4.4多龙头规则判断（不是无条件全出）

铁律2：监管任何形式限制措施（保证金/账户限制/临停） → 当日市价全出

铁律3：批量澄清公告（盘前≥5家，或盘中≥10家，或3日内≥15家） → 立即清仓分级执行
  盘前≥5家 → 开盘即出
  盘中≥10家 → 尾盘减仓50%，次日开盘清余仓
  3日内累计≥15家 → 等同铁律2，当日全清
```

**强制退出执行要求：**
- 执行价：市价，不挂限价单等成交
- 时间：当日，不过夜
- 后续：强制退出后24h内禁止同板块再入场（冷静期）
- 记录：触发原因写入Trade Journal（用于Pattern Tag统计）

## 2. Track B 加仓规则

**设计原则：** Track B加仓比Track A更保守——因为动量信号的可信度比基本面thesis低，加仓应跟随更强的市场确认，而不是跟随自己的conviction。

### 2.1 加仓触发条件（全部满足）

```
条件1：当前仓位 < B+级硬顶（15%）
条件2：持仓已进入阶段1（D+2之后，探针+确认均已完成）
条件3：龙头继续创新高（不是反弹，是新高）+ 成交放量（>5日均量×1.5）
条件4：板块产业链传导确认（L1→L2验证完成，§5.1传导序列）
  → 具体标准：同板块第二层标的已出现涨停，验证资金从L1搬到L2
条件5：持仓距类型天数上限≥3天（接近上限时不加仓，避免高位加仓后时间止损触发）
条件6：全市场涨停家数仍≥60家（主升期判断）
```

**产业链传导确认细则（条件4展开）：**

| 持有的是 | 传导确认信号 | 无传导 |
|---------|-----------|-------|
| L1光模块 | L2 PCB出现涨停 | 不加仓 |
| L2 PCB | L5 CCL同日跟涨≥3% | 不加仓 |
| L3液冷 | L2 PCB仍强势（不是回调） | 不加仓 |
| Type 3情绪扩散标的 | **禁止加仓**（情绪型不确认） | — |

### 2.2 加仓上限

- 单笔加仓量：不超过初始仓位的50%（若初始7%，加仓不超3.5%，总仓位≤10.5%）
- 累计上限：总仓位不超B+级硬顶15%，无论加几次
- 加仓次数：最多1次。Track B不做多次金字塔加仓（持有周期太短，加仓窗口有限）

### 2.3 加仓成本处理（平均成本 vs 新入场价）

**使用平均成本法，不重置入场价。**

理由：Track B的止损锚定于龙头（龙头跌>5%全出），不依赖自身ATR止损线作为主要出场信号。棘轮止损基于初始入场价建立的High基准。加仓后用平均成本计算P&L，但棘轮止损的"保本"基准仍锚定初始入场价（更保守，避免平均成本效应放宽保本线）。

实操示例：
```
初始入场：7% @ ¥10.00，棘轮保本线 = ¥10.00
加仓：3.5% @ ¥11.00，平均成本 = ¥10.37
棘轮止损保本线：仍维持 ¥10.00（不因加仓重置为¥10.37）
```

## 3. Track B Trade Score（TB-TS）

**设计原则：** Track B的价值不在于公司研究深度，而在于动量交易的执行质量——识别对、进得早、出得快、赚板块涨幅的合理份额。评分维度与Track A §4.6完全不同。

### 3.1 TB Trade Score 评分维度

| 维度 | 权重 | 满分标准 | 0分标准 | 计量方式 |
|------|------|---------|---------|---------|
| **轮动类型识别准确度** | 25% | 入场前正确识别Type（事后验证吻合） | 识别错误（如把Type 3认成Type 2，导致超期持有） | 对/错二元，对=满分，错=0分 |
| **入场时机（龙头启动Lag）** | 25% | D+1~D+2入场（按§3.3最优窗口） | D+4以后入场（错过最优窗口追高） | Lag天数评分：D+1=100/D+2=80/D+3=50/D+4+=0 |
| **出场纪律（信号→执行速度）** | 25% | 信号出现当日T+0执行 | 信号出现后超过2天未执行 | T+0=100/T+1=70/T+2=40/T+3+=0 |
| **相对板块捕获率** | 15% | 个股涨幅 ≥ 板块平均涨幅的80% | 个股涨幅 < 板块平均涨幅的40% | (个股涨幅/板块涨幅)×100，cap 100 |
| **持有纪律（类型天数上限执行）** | 10% | 在类型上限内出场，未超期 | 超过类型天数上限仍未出场 | 合规=100/超期1天=50/超期2天+=0 |

**总分→等级：**
- A（≥80）：标准动量交易，执行规范
- B（60-79）：可接受，有1-2个维度需改善
- C（40-59）：执行不达标，月末≥2笔C需系统检讨
- D（<40）：严重执行失误，写post-mortem + Pattern Tag

**TB Trade Score特别说明：**
1. 类型识别错误（维度1=0分）是最严重的单维度失误——说明入场前没有做§2.2类型检查清单。
2. 出场纪律（维度3）是最影响结果的维度——动量交易最大的亏损来源是该走没走。
3. 相对板块捕获率（维度4）<40%时，说明标的选择有问题（选了板块里最弱的跟风票），需复盘选股逻辑。

## 4. Track B Round Trip 规则

**核心问题：** 轮动策略本质上是频繁进出同一板块，不能简单套用Track A的3日Round Trip惩罚。但也不能完全免除——防止无效来回交易。

### 4.1 Round Trip 判定标准（Track B专用）

**以下情况不构成Round Trip（正常轮动操作）：**

| 操作模式 | Round Trip？ | 理由 |
|---------|------------|------|
| 同板块不同标的：卖PCB买CCL | **否** | 产业链内资金搬家，是strategy.md §5.2规定的正常操作 |
| 同板块同标的：上次出场已盈利+出场后>3天重新入场+有新催化剂 | **否** | 新的交易机会，不是Round Trip |
| 同标的：卖出再买同一类型（如两次Type 3情绪扩散同一票）| **是** | 同类型重复操作，无新催化剂支撑 |
| 同标的：3天内卖出且盈亏<3% | **是** | 与strategy.md §4.7标准一致 |
| 同标的：出场后<3天重入（无论盈亏，无新催化剂） | **是** | 3天冷静期，无新催化剂=无效操作 |

**Round Trip判定三步检查（Track B版）：**
```
Step 1：是否同一标的？
  否（同板块不同票） → 不是Round Trip，正常记录
  是 → 进入Step 2

Step 2：上次出场到本次入场间隔是否>3个交易日？
  否（≤3天） → Round Trip（无论理由，惩罚触发）
  是 → 进入Step 3

Step 3：是否有新催化剂（上次出场后出现的、入场前不存在的α/β级催化剂）？
  否 → Round Trip
  是 → 不是Round Trip，允许重入（但TB Trade Score维度1须重新验证类型）
```

### 4.2 Round Trip 惩罚（Track B专用，比Track A更轻——因轮动策略本身需要快进快出）

| 触发次数 | 惩罚 | 解除 |
|---------|------|------|
| 第1次 | 警告，记录Pattern Tag `tb_round_trip` | 下一笔Track B Trade Score ≥ B级自动解除 |
| 第2次（30天内） | 该板块3个交易日禁止新建Track B仓位 | 到期自动 |
| 第3次（30天内） | 全体Track B降仓位上限50%，持续5个交易日 | 到期自动 |

注：Round Trip惩罚不影响Track A的仓位操作，两轨独立。

### 4.3 轮动过程中卖A买B（同板块换票）

**明确答复：正常操作，不是Round Trip。**

卖L1光模块买L2 PCB = strategy.md §5.2资金搬家规定的标准动作。条件：
- 必须在同一个产业链传导序列内（§5.1）
- 不是因为看涨了才换（不是FOMO换票）
- 换票前检查：是否有具体的传导确认信号（§2.1条件4）

同板块换票的Round Trip豁免适用于**不同标的**。同一标的卖了再买回，仍按§4.1三步检查判断。

## 5. 双轨组合管理

**设计原则：** Track A和Track B持有不同目的的仓位，但共用现金池，因此需要明确的组合约束，防止同时满仓导致无法执行止损。

### 5.1 持仓数量上限

| 轨道 | 最大持仓数 | 说明 |
|------|-----------|------|
| Track A | 8只 | strategy.md §3.2规定 |
| Track B | **3只** | 动量交易同时追踪3只以上难以有效监控龙头信号 |
| 双轨合计 | **10只** | 超过10只时：禁止新建Track B，优先Track A 止损清仓后释放名额 |

**Track B持仓数3只的依据：**
- 每只Track B标的每日需要盯龙头、炸板率、成交量3个实时信号
- 3只以上同时触发阶段2时，人工判断容量不足，容易漏执行
- 历史回测：单次主升浪中，Track B优质标的通常只有1-2只（过度分散反而摊薄弹性）

### 5.2 总仓位约束

| 状态 | Track A仓位 | Track B仓位 | 合计上限 | 现金底线 |
|------|------------|------------|---------|---------|
| 强吸气（F20） | 无独立上限 | ≤30% | 无硬约束（strategy.md §3.2移除现金底线） | 无 |
| 吸气（F20） | 无独立上限 | ≤25% | 无硬约束 | 无 |
| 中性 | 无独立上限 | **≤15%** | **≤70%** | ≥20% |
| 呼气 | 无独立上限 | **0%（全关）** | strategy.md决定 | — |

**Track B仓位约束附加规则：**
- Track B 3只合计上限30%（强吸气），平均每只10%。单只仍受B+级15%硬顶。
- Portfolio Heat红线（strategy.md §3.4）：Track B标的ATR较大，加入组合后Heat上升更快。Track B建仓前必须检查Heat是否接近红线15%。
- 若加入Track B后Heat预计超过13%（红线的86%）：降低Track B单只仓位，而非拒绝入场。

### 5.3 现金分配（共用现金池）

**共用现金池，不分配。**

理由：分离现金池会导致"Track A现金满，Track B在等机会"的低效情况，且实际操作中难以维护两个独立的现金账户。

**共用池下的优先级规则：**
```
资金使用优先级（冲突时按序执行）：
1. 执行止损（最高优先级，不因现金不足延迟）
2. Track A建仓（conviction主导，基本面thesis）
3. Track B加仓（已持有仓位，龙头强势确认中）
4. Track B新建仓（新入场）
```

现金不足时：不建Track B，不压缩Track A。Track B是机会性策略，应在Track A有充足现金的前提下执行。

### 5.4 同板块双轨同时持仓的出场规则冲突处理

**场景：** Track A持有中际旭创（光模块龙头），Track B持有天孚通信（同板块二线）。
某日主龙头跌>5%，Track B铁律触发全出。Track A是否同时出场？

**规则：**

| 情形 | Track B | Track A | 依据 |
|-----|--------|--------|------|
| 主龙头（Track A持有标的本身）跌>5% | 全出（铁律2） | 按strategy.md §4.3处理（板块结束信号→同板块出场） | 两者均触发，动作一致 |
| 非主龙头跌>5%，主龙头（Track A持有）未跌 | 减仓50%后观察（§4.4多龙头规则） | 不出场，监控主龙头 | Track A和Track B判断独立 |
| Track B的止损信号（自身ATR触发）出现，龙头未跌 | 先按龙头信号判断：龙头仍强 → 只减仓50%，不全出 | 不影响Track A | Track B止损锚定龙头，非自身ATR |

**关键规则：Track B没有"自己的ATR止损线"作为主要出场信号。** 止损锚定于龙头，而非自身价格走势。自身价格的棘轮止损（§1.3）是次要防线，龙头信号优先。

### 5.5 Daily Review 双轨展示格式


## 6. Track B → Track A 升级协议

**strategy.md §2.4b原文：**
> "Track B不升级为Track A（除非公司本身thesis验证后重新评级）"

**解读：** 这不是简单的"禁止升级"，而是要求**清仓Track B → 重新走Track A五步法建仓**。两者不是同一持仓的状态升级，而是完全不同的两笔交易。

### 6.1 升级协议（禁止直接升级，只允许重建）

**严格禁止：**
- 把Track B的仓位直接调高评级（如从B+改为A-）
- 把Track B的仓位直接扩大
- 在未清仓Track B的情况下叠加Track A仓位

**允许的路径（唯一正确路径）：**

```
Step 1：先清仓Track B（按正常阶段2退出程序执行）
Step 2：等待冷静期（至少1个交易日）
Step 3：独立评估Track A建仓条件：
  - 重新走§2.1五步法（thesis/catalyst/stop/size/scale）
  - Track A thesis必须是"公司质量+竞争壁垒+业绩验证"，不能是"之前Track B赚了钱"
  - 若Track B盈利出场后公司thesis仍在，满足Track A五步法 → 允许新建Track A仓位
Step 4：新建Track A仓位，独立计入Track A持仓管理
```

### 6.2 升级路径的触发条件（什么情况值得评估）

**如果出现以下信号，Track B出场后可以评估Track A重建：**
```
□ 持有期间发布了超预期季报（营收+净利beat + 毛利改善，§4.4扩张型）
□ 有新的α级催化剂出现（具体日期，非γ级主题）
□ 管理层公告了实质性业务变化（新客户/新产品线/重大订单）
□ 公司从"蹭热点"变为"核心受益方"（有具体业务数字验证）
```

**如果以下情况出现，不考虑升级Track A：**
```
✗ 仅仅因为Track B期间价格涨了很多
✗ 板块仍在轮动中（说明动量逻辑仍有效，可以重建Track B，不需要Track A）
✗ 没有独立的、基本面层面的thesis（说不出Track A五步法中的thesis就不建）
```

### 6.3 升级评估频率

- 每笔Track B盈利出场后：在写Track B Trade Score时增加一行"Track A评估"（是/否+原因）
- 不需要每次Track B都评估，只在**出场理由是"时间上限"而非"信号退出"**时才值得评估（时间上限出场=主动退出，可能thesis仍在；信号退出=市场否定，不值得升Track A）

## 7. Track B If-Then 预承诺格式

**设计原则：** Track B的预承诺比Track A更短期、更具体、更依赖龙头状态。必须在收盘后非交易时间写入，盘中只执行。

### 7.1 预承诺模板格式

```
[TB] 标的/Ticker: ___
轮动类型: Type [1/2/3/4/5]
入场日: D+0 = YYYY-MM-DD
龙头绑定: ___（主龙头名称+关键价格水位）
类型天数上限: D+___（绝对退出日期）

--- 板块层面 If-Then ---
IF 主龙头[___] 当日跌>5%
THEN T+0收盘前市价全出（铁律，不分析，不等）

IF 主龙头[___] 断板（涨停后开板）
THEN 当日清北交所仓位，主板仓位减50%，次日再判断

IF 炸板率超过[50%]，连续2天
THEN 次日开盘减仓至留[5%]，3天内清仓

--- 催化剂层面 If-Then ---
IF [具体催化剂事件+日期，如"COMPUTEX 2026-06-01 NVIDIA AI架构发布"]
THEN 升至[B+级]，加仓至[12%]（目标仓位）
ELSE 发布后价格不涨（利好兑现）→ T+0减仓至[5%]，T+1清仓
ELSE 发布后跌（负面验证）→ T+0全出

--- 时间止损 ---
IF 持仓达 D+[X]（类型天数上限）
THEN 无条件出场，不论盈亏，不论龙头状态

--- 产业链传导 If-Then（如有）---
IF [L1光模块]冲顶信号（主力净卖出>10亿 或 量价背离+次级涨停）
THEN 减持[持有标的]50%，同日评估[L2 PCB]建仓机会

CURRENT_STATUS: [阶段0/1/2/3] | 龙头状态: [创新高/走弱/断板] | 浮盈: ___
```

### 7.2 预承诺示例（实际标的，Track B蘅东光/北交所光连接）

```
[TB] 标的/Ticker: 蘅东光(835508)
轮动类型: Type 2（产业催化，光连接BOM需求）
入场日: D+0 = 2026-05-XX
龙头绑定: 中际旭创(300308)，关键支撑价¥XXX（前期量能平台）
类型天数上限: D+12（北交所Type 2最长参考值）

IF 中际旭创当日跌>5%
THEN 当日市价全出蘅东光（铁律，北交所±30%，不能等次日）

IF 中际旭创断板
THEN 当日清仓蘅东光（北交所流动性风险，快出）

IF 炸板率全市场>60%，连续出现
THEN 当日清仓（崩溃期，铁律级）

IF Rubin架构BOM数据验证光连接增量（季报或公告，有具体数字）
THEN 加仓至8%（北交所上限），持有至D+12上限或新退出信号

ELSE 至D+12无新催化 → D+12收盘前全出

CURRENT_STATUS: 阶段1（持有期） | 龙头状态: 创新高 | 浮盈: +X%
```

### 7.3 预承诺写入纪律

- 写入时间：每日收盘后（15:00-16:00），次日生效
- 格式要求：必须有龙头绑定（不写龙头名称的预承诺无效）
- 修改限制：盘中不可修改（strategy.md §1 R5）。想修改→收盘后改→次日生效。
- 归档：每笔Track B出场后，将预承诺和实际执行情况对比记入Trade Journal。偏差大的写post-mortem。
