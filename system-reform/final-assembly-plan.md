## 1. 结构选型决策

**推荐: Option C — 单文件，三个显式 "COPY THIS TO USE" 分区**

理由如下（逐项 vs Option A/B）：

| 维度 | Option A (当前) | Option B (3文件) | Option C (推荐) |
|------|---------------|-----------------|----------------|
| 使用摩擦 | 三节混在一起，无明确边界 | 3文件分开，但信号协议游离 | 每节有明确 COPY 边界，0摩擦 |
| 交叉污染风险 | 高（共享节存在，易误读） | 低，但信号协议依赖另外两个文件 | 低，每个大脑是独立封闭块 |
| 版本同步 | 1个文件改，直接生效 | 3个文件必须同步更新，容易遗漏 | 1个文件，3节同步，无漂移风险 |
| 信号协议独立性 | 不独立（共享节混入规则） | 独立但孤立（引用关系断裂） | 独立封闭，与两个大脑平行放置 |
| 适配现有系统 | playbook/quickref已分文件 | 过度拆分，与 playbook 文件产生重复 | 与 playbook 互补，prompt是"大脑规则"，playbook是"操作流程" |

**最终结论**: Option C。单文件便于整体版本控制，三个 `=== SECTION START / END ===` 边界使每个区块可以独立 copy-paste 进 Claude 对话，与 sim-portfolio 现有 playbook 体系不重复（prompt 是规则大脑，playbook 是执行流程，两层分工清晰）。
