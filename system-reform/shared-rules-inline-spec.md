# Shared Rules Inline Specification
> 目的: 告诉rewrite agents，共享铁律内联进各市场prompt时，每条规则的具体文字内容
> 权威来源: rule_authority.md + strategy.md (A股) + US_TRADING_SYSTEM_V4.md (美股)
> 生成日期: 2026-05-23

---

## 背景与问题

当前 `/tmp/trading-system-prompts.md` 在两个market prompt之外，设有独立的"共享铁律"章节（第255-284行）。这导致：
1. 两个prompt都必须引用第三节，增加agent读取负担
2. 无法表达市场差异阈值（如ABCD A类触发点）
3. 认知负担：agent需同时维持3段上下文而非1段自洽prompt

**解决方案**: 将共享铁律完全内联进每个market prompt，消除第三节。

## 规则逐条内联规范

### Rule 1: ABCD 下跌分类

**差异**: A类触发指数不同，阈值不同。B/C/D类文字完全一致。

| 字段 | A股版本 | 美股版本 |
|------|---------|---------|
| 参考指数 | 沪深300 | SPY |
| A类触发阈值 | ≥1.5% | ≥2.5% |
| 分类名称 | A类/B类/C类/D类 | 与A股相同 |

**A股内联文字**:
```
### ABCD下跌分类（分类前禁止卖出）
A类：沪深300同步跌≥1.5%，无个股新闻 → Hold（Beta跌，不是你的问题）
B类：沪深300稳/涨+无新闻+个股跌<3% → Hold 1-2天，等市场澄清
C类：有新闻但不涉及核心thesis → 先写结论再操作，评估减仓
D类：硬数据证伪thesis（财报miss/核心客户流失/政策否决）→ 48h内清仓，无例外

执行顺序：收到跌的消息 → 先完成分类（60秒）→ 然后才能操作
S级例外：从入场价跌7%立即EOD全退，不等ABCD分类
```

**美股内联文字**:
```
### ABCD下跌分类（分类前禁止卖出）
A类：SPY同步跌≥2.5%，无个股新闻 → Hold（Beta跌，不是你的问题）
B类：SPY稳/涨+无新闻+个股跌<3% → Hold 1-2天，等市场澄清
```

### Rule 2: Conviction 等级与仓位上限

**差异**: 两市场的仓位上限显著不同。A股仓位更高（A股集中度策略），美股更分散（流动性/做空组合需要）。

**权威来源**: rule_authority.md 裁定 C1/C2 + strategy.md §3.1（A股）+ US_TRADING_SYSTEM_V4.md §3.1（美股）

| 等级 | A股上限 | 美股上限 | 差异说明 |
|------|---------|---------|---------|
| S级 | 目标30% / 上限40% | 初始15% / 上限25% | A股允许集中押注；美股分散+空头需预留空间 |
| A级 | 目标15% / 上限25% | 初始10% / 上限15% | 同上 |
| B级 | 目标10% / 上限15% | 初始7% / 上限10% | 同上 |
| C级 | 目标5% / 上限8% | 初始5% / 上限8% | 两市场一致 |
| T级 | 目标4% / 上限8% | 5%固定 / 上限5% | T级定义略有差异 |

**A股内联文字**:
```
### 建仓前必须定Conviction等级（A股仓位上限）
S级（讲清5条S1-S5 + bear case<10%）→ 目标30%，上限40%，最多1只，止损-7%硬止损
A级（能讲清3层逻辑）→ 目标15%，上限25%，止损15-20%
B级（1-2层逻辑）→ 目标10%，上限15%，止损10-15%
C级（直觉+单一数据点）→ 目标5%，上限8%，止损7-10%
T级（事件驱动/超短期）→ 目标4%，上限8%，止损-5%

说不出thesis → 不建仓，无例外
```

**美股内联文字**:
```
### 建仓前必须定Conviction等级（美股仓位上限）
S级（5维度评分S + Momentum≥8）→ 初始15%，上限25%，最多1只，止损-7%硬止损EOD，持仓≤10交易日
A级（5维度达A）→ 初始10%，上限15%，最多2只，止损trailing -12%（从高点），时间止损60天
B级（5维度达B）→ 初始7%，上限10%，最多2只，止损trailing -10%（从高点），时间止损30天
C级（5维度达C）→ 初始5%，上限8%，最多1只，止损-8%硬止损，时间止损20天
T级（事件驱动/超短期）→ 5%固定，上限5%，最多1只，止损-8%，时间止损10天

说不出thesis → 不建仓，无例外
最低建仓金额：$7,500（低于此门槛不新建）
```

### Rule 3: L10 关注点漂移检查

**差异**: 行为规则完全相同，但示例应配合各市场语境。A股用A股标的名，美股用美股标的名。

| 字段 | A股版本 | 美股版本 |
|------|---------|---------|
| 核心行为规则 | 相同 | 相同 |
| 触发示例 | "看到茅台/宁德涨了想换" | "看到NVDA/TSM涨了想换" |
| 等待行为 | 关闭行情等1h | 关闭行情等1h |

**A股内联文字**:
```
L10 关注点漂移铁律
触发：看到其他A股涨了（比如茅台/宁德/比亚迪）想换仓
强制两问：
  ① 我现在的持仓thesis变了吗？（不是价格变了，是thesis变了吗）
  ② 我想卖是因为看了别的票在涨吗？
如果②是 → 立即关闭行情软件，等1小时，冷静后再判断
不允许直接"那个更好"的直觉换仓
```

**美股内联文字**:
```
L10 关注点漂移铁律
触发：看到其他美股涨了（比如NVDA/TSM/META）想换仓
强制两问：
  ① 我现在的持仓thesis变了吗？（不是价格变了，是thesis变了吗）
  ② 我想卖是因为看了别的股在涨吗？
如果②是 → 立即关闭行情/Webull，等1小时，冷静后再判断
不允许直接"那个更好"的直觉换仓
```

### Rule 4: L11 催化剂前默认持有

**差异**: 规则文字完全相同，无需市场差异化。

**两市场统一内联文字**:
```
L11 催化剂前默认持有
"已经涨了不少"不构成减仓理由
减仓需要以下之一的充分理由：
  - 止损触发（达到预设止损线）
  - 有更好的A级机会且资金有限
  - 有明确资金需求
  - D类：thesis被证伪
催化剂还在前面 → 持有，不因"涨了"减仓
```

### Rule 5: L12 卖出标准

**差异**: 核心逻辑完全相同，无需市场差异化。

**两市场统一内联文字**:
```
L12 卖出标准（与盈亏无关）
唯一正确的卖出问题："现在这个价格，我会新买吗？"
  → 会买 → 持有，不卖
  → 不会买 → 卖出，与盈亏无关
禁止因为亏损了不甘心持有，也禁止因为盈利了不舍得卖
```

### Rule 6: L13 三行检查表（买入前）

**差异**: 规则逻辑相同，但第③条示例可以配合市场语境。

**两市场统一内联文字（语言无需差异化，规则够清晰）**:
```
L13 三行检查表（每笔买入必须全部通过）
① 买的理由是什么？（不能是"涨了"或"别人推荐了"）
② 跌20%我割不割？（说不出→不买；说得出但真的会割→不买）
③ 是不是今天才知道它的？（是→不买，你是信息链末端）

第③条最关键：今天才知道=别人早就知道了=你来接盘
全部通过才能建仓，一条不过即止
```

### Rule 7: L14 Conviction-仓位配对

**差异**: Conviction等级映射的仓位上限不同（沿用Rule 2的市场差异），核心行为规则相同。

**A股内联文字**:
```
L14 建仓前定信心等级（A股）
必须先定等级，仓位不能超对应上限：
  A级（3层逻辑讲清楚）→ ≤25%
  B级（1-2层逻辑）→ ≤15%
  C级（直觉+单点）→ ≤8%
说不出thesis是什么 → 仓位=0，不建仓
等级定了不能因为"感觉很好"临时加仓超上限
```

**美股内联文字**:
```
L14 建仓前定信心等级（美股）
必须先定等级，仓位不能超对应上限：
  A级（5维度评分达A）→ ≤15%
  B级（5维度评分达B）→ ≤10%
  C级（5维度评分达C）→ ≤8%
说不出thesis是什么 → 仓位=0，不建仓
等级定了不能因为"感觉很好"临时加仓超上限
```

### Rule 8: L15 Thesis验证（搜"最近为什么涨"）

**差异**: 核心逻辑完全相同，ticker示例格式不同。

**A股内联文字**:
```
L15 Thesis验证（买前必做）
建仓前搜索："[A股名称或代码] 最近为什么涨"（例："宁德时代 最近为什么涨"）
对比搜索结果与你的买入理由：
  → 市场的实际驱动力 = 你的买入理由 → 可以买
  → 市场的实际驱动力 ≠ 你的买入理由 → 不买（你在交易错误的thesis）
典型错误：圣泉605589——因"钠电"买入，实际涨是因为PPO树脂
```

**美股内联文字**:
```
L15 Thesis验证（买前必做）
建仓前搜索："[股票代码] why is it going up"或"[公司名] stock rally reason"
（例："MSTR why is it going up"）
对比搜索结果与你的买入理由：
  → 市场的实际驱动力 = 你的买入理由 → 可以买
  → 市场的实际驱动力 ≠ 你的买入理由 → 不买（你在交易错误的thesis）
典型错误：HSAI——以为在交易LiDAR渗透率，实际主beta是中美关系/ADR监管风险
```

### Rule 9: 价格查询命令（yf price）

**差异**: ticker格式不同。A股需要交易所后缀，美股直接用ticker。

| 字段 | A股版本 | 美股版本 |
|------|---------|---------|
| 示例命令 | `yf price 000002.SZ` 或 `yf price 600519.SS` | `yf price AAPL` 或 `yf price NVDA` |
| 交易所后缀 | 深交所=.SZ / 上交所=.SS | 无后缀 |

**A股内联文字**:
```
### 数据规则
- 所有A股价格用yf实时查询，不用记忆/旧数据
  格式：yf price {代码.交易所} （深交所=.SZ，上交所=.SS）
  例：yf price 000002.SZ / yf price 600519.SS
- 引用任何数字必须有来源（公告/财报/一手媒体），不用"据估算"包装猜测
- 搜索引擎结果≠今日数据：搜到的数字必须加日期校验，超过1个交易日的价格数据不引用
```

**美股内联文字**:
```
### 数据规则
- 所有美股价格用yf实时查询，不用记忆/旧数据
  格式：yf price {TICKER}
  例：yf price AAPL / yf price NVDA / yf price SPY
- 引用任何数字必须有来源（SEC filing/财报/Bloomberg/Reuters），不用"据估算"包装猜测
- 搜索引擎结果≠今日数据：搜到的数字必须加日期校验，超过1个交易日的价格数据不引用
- VIX/SPY/TNX用yf macro命令拉取
```
