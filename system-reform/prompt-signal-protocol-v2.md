# 跨市场催化剂信号协议 v2.0

> 设计原则：两个市场大脑完全独立运作。此协议是唯一跨市场通信通道。
> 信号只传事实，不传结论，不传建仓建议。接收方独立判断。

---

## 信号 Schema

```json
{
  "from":         "<市场代码，如 US | CN>",
  "timestamp":    "<ISO 8601，含时区>",
  "event":        "<一句话事实，不含判断>",
  "chain":        "<受影响供应链路径，用→分隔>",
  "first_mover":  "<一阶已动标的 + 幅度>",
  "type":         "earnings | policy | product | commodity | ipo",
  "urgency":      "4h | 24h | 1w"
}
```

字段规则：
- `event`：只写可观测事实（价格、数字、公告），禁止含"利好/利空"等判断词
- `chain`：从事件源头写到接收方市场的末端节点
- `first_mover`：已经反应的龙头标的，帮接收方判断时效
- `urgency`：发送方预估的时效窗口，接收方可自行覆盖

---

## 路由规则

| # | 规则 |
|---|------|
| R1 | 信号存入 `signals/pending/` 目录，每条独立文件 |
| R2 | 催化剂发生后 **4h 内**发送有效；>24h 废弃，不发 |
| R3 | 龙头已连板第3天才发 = 废信号，不发 |
| R4 | 接收方读取信号后**独立**判断，不受发送方观点约束 |
| R5 | 接收方处理后将文件从 `pending/` 移至 `processed/`，标注 `action_taken` |

---

## 触发条件（发送方检查清单）

以下任一事件发生时，发送方必须生成信号：

- [ ] 龙头标的单日涨跌 > 5%
- [ ] 财报 EPS/Revenue 显著 beat 或 miss
- [ ] 重大政策/监管变化（影响跨境供应链）
- [ ] 供应链关键节点供给突变（减产/停产/新产能）
- [ ] 大宗商品价格单日变动 > 3%
- [ ] 重大 IPO 上市（定价 / 首日涨跌）

## 嵌入各市场 Prompt 时的差异化示例

### A 股 Prompt 使用（接收来自 US 的示例）

```json
{
  "from": "US",
  "timestamp": "2026-05-20T08:30:00-04:00",
  "event": "NVDA Q1 FY2027: EPS $0.96 vs $0.93e, Data Center revenue $39.1B +93% YoY",
  "chain": "AI算力→PCB→CCL→电子布→铜箔",
  "first_mover": "NVDA +3.2%",
  "type": "earnings",
  "urgency": "4h"
}
```
接收方动作示意：沿 chain 找还未动的国内节点（鹏鼎/沪电/生益），验证是否在今日行情中滞涨。

### 美股 Prompt 使用（接收来自 CN 的示例）

```json
{
  "from": "CN",
  "timestamp": "2026-05-21T10:15:00+08:00",
  "event": "工信部发布通知：氢燃料电池重卡2026年补贴延续，单车上限18万元",
  "chain": "政策→燃料电池→质子交换膜→铂催化剂→FCEL/BE",
  "first_mover": "亿华通 +7.4%",
  "type": "policy",
  "urgency": "24h"
}
```
接收方动作示意：检查 FCEL/PLUG/BE 是否有联动滞后，结合 F21 Earnings 节奏判断是否值得入场。

## 与主系统的集成说明

- 此文件是**唯一信号标准**，两个市场 Prompt 均引用本文档，不在各自 Prompt 内重复定义 schema
- 各市场 Prompt 只需嵌入"本市场作为接收方"的示例（见上方差异化示例），不嵌入对方市场的内部规则
- 协议变更需同步通知两个市场 Prompt 维护者，版本号升位
